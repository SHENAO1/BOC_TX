# 单时隙 BOC(15,3) 基带组合验证——技术实施文档
---

## 1. 文档目的与适用范围
本文档是 BOC-TDMA 系统分阶段实施计划中的 **P3 阶段技术实施文档**，承接 P0（USRP 链路验证）、P1（BOC 子载波分析）、P2（Gold 码扩频验证），并为 P4（单周期 TDMA 帧生成验证）提供可直接继承的"单时隙 BOC 基带模板"。

P3 阶段的核心定位是：**第一次将 P1 锁定的 BOC 子载波 **`sc[n]`** 与 P2 锁定的扩频码序列 **`c_up[n]`** 进行真正的乘法组合，生成一个完整的、长度为 9000 点的单时隙 BOC(15,3) 基带信号 **`s_slot[n]`**，并对其频谱与自相关结构进行定性与定量验证。**

P3 必须独立存在，原因如下：

+ P1 仅验证了 BOC 子载波 `sc[n]` 本身的方波结构与频域特征；
+ P2 仅验证了扩频码 `c_up[n]` 的码域结构与扩频后的近似单峰频谱；
+ 二者都未经过组合，BOC 调制的标志性特征——**频域双峰结构**与**自相关 M 形特征**——尚未真正出现；
+ 若不在 P3 单独验证组合后的基带，直接进入 P4 做 32 时隙拼接，一旦频谱异常将无法快速定位是 P1 错、P2 错、还是组合关系错；
+ P3 是从"独立模块验证"走向"组合信号验证"的关键阶段，也是整个工程链路中**第一次真正看到"BOC 双峰"的阶段**。

本阶段验证对象限定为：**单时隙、单 bit、单载荷、9000 点 BOC 基带信号**，不涉及 TDMA 多时隙拼接、多载荷复用或连续循环发射。

---

## 2. 本阶段在总工程中的位置
依据【总规划已明确】的分阶段路线：

```plain
P0  USRP 链路基线验证
        │
        ├──> P1  BOC 子载波分析（锁定 sc[n]）
        │
        └──> P2  Gold 码扩频验证（锁定 c_up[n]）
                                │
                                ▼
                        P3  单时隙 BOC 基带组合验证   ← 本阶段
                                │
                                ▼
                        P4  单周期 TDMA 帧生成验证
                                │
                                ▼
                        P5 / P6  ……
```

依赖关系明确：

+ **P3 必须以 P1、P2 都通过验收为前提**，不允许在 P1 或 P2 仍有未锁定项的情况下启动 P3 的频谱与自相关验收；
+ **P3 的输出 **`s_slot[n]`** 将直接成为 P4 时隙拼接的基本单元**，P4 不会重新生成单时隙信号，而是直接对 P3 的输出做时隙级编排；
+ P3 是工程链路上**第一次将 P1 和 P2 组合起来**的阶段，也是 BOC 调制本质特征第一次出现的阶段。

## 3. 本阶段目标
本阶段只引入一个新的复杂性维度：**将 BOC 子载波与扩频码进行乘法组合**。具体目标如下：

+ **G1**：确认单时隙 BOC 基带的离散时间模型 `s_slot[n] = A · b_m · c_up[n] · sc[n]` 在工程实现中正确成立；
+ **G2**：将 P1 锁定的 `sc[n]` 与 P2 锁定的 `c_up[n]` 按统一采样率正确对齐并组合；
+ **G3**：生成长度严格为 9000 点的单时隙基带信号 `s_slot[n]`，覆盖 250 μs 时长；
+ **G4**：验证组合后的频谱由 P2 阶段的"扩频单峰谱"演变为 BOC(15,3) 标志性的"双峰谱"，主峰中心位于 ±15 MHz 附近；
+ **G5**：验证组合后的自相关函数由 P2 阶段的近似单峰演变为 BOC 标志性的 **M 形结构**；
+ **G6**：形成一个稳定、可复用的"单时隙基带模板"，供 P4 阶段直接继承用于时隙级拼接。

10. **不做最终 USRP 中心频率 / 增益的选型**，P3 若涉及硬件验证仅为定性观测。

## 6. 单时隙 BOC 基带理论模型
### 6.1 连续时间模型
在一个单时隙内（时长 `T_slot = 250 μs`，码片数 750），携带 1 bit 信息的 BOC(15,3) 基带信号连续时间模型为：

$ s_\text{slot}(t) \;=\; A \cdot b_m \cdot \left[ \sum_{k=0}^{749} c_k \cdot p_{T_c}(t - k T_c) \right] \cdot \mathrm{sign}\!\left[\sin(2\pi f_\text{sc} t)\right],
\quad t \in [0,\, T_\text{slot}) $

变量说明：

+ `A`：基带幅度，P3 中取 `A = 1`（最终归一化策略【待确认】）；
+ `b_m ∈ {+1, -1}`：当前时隙携带的 bit 映射后的符号；
+ `c_k ∈ {+1, -1}`：Gold 码截短序列的第 `k` 个码片，来自 P2；
+ `T_c = 1/f_c = 1/3 μs`：单码片时长；
+ `p_{T_c}(t)`：码片矩形脉冲，`p_{T_c}(t) = 1, t ∈ [0, T_c)`，否则为 0；
+ `f_sc = 15 MHz`：BOC 子载波频率；
+ `sign[sin(·)]`：方波形式的 BOC 子载波。

### 6.2 离散时间模型（工程实现模型）
在采样率 `fs = 36 MSPS` 下离散化，P3 使用以下统一离散模型：

$ \boxed{\; s_\text{slot}[n] \;=\; A \cdot b_m \cdot c_\text{up}[n] \cdot sc[n], \quad n = 0, 1, \ldots, 8999 \;} $

其中：

+ `c_up[n]`：**来自 P2**，长度 9000 点，每个码片重复 12 次，取值 ±1；
+ `sc[n]`：**来自 P1**，长度 9000 点，取值 ±1（在 P1 的 `sign(0)` 方案下），每码片内呈现固定的 12 点 BOC 图样；
+ `b_m`：bit 符号，取 ±1；
+ `A`：幅度，P3 取 1。

**工程约束**：

1. P3 **不重新定义** `sc[n]` 与 `c_up[n]`，均直接载入 P1、P2 的锁定输出；
2. bit=0 与 bit=1 只影响 `b_m` 的符号，**整个时隙的 **`s_slot[n]`** 仅发生全局极性翻转**；
3. `s_slot[n]` 的取值完全由 `c_up[n] · sc[n]` 决定，若 P1 锁定的 `sc[n]` 严格为 ±1，则 `s_slot[n]` 也严格为 ±1；若 P1 保留了 `sign(0) = 0` 的处理（【待确认，以 P1 锁定版本为准】），则 `s_slot[n]` 在对应样点处也为 0。

## 7. 组合后的关键物理变化分析
本节是 P3 的工程核心。**P1 与 P2 的组合不是简单的数值相乘，而是引入了 BOC 调制的本质特征。** 必须在进入 P4 之前完整理解下述变化。

### 7.1 从 P2 到 P3 的频谱变化
+ **P2 阶段的 **`s_dsss[n] = c_up[n]`：频谱为矩形码片脉冲谱的基本形态，sinc² 包络，**主瓣集中在基带零频附近，主瓣宽度约为 ±3 MHz（双边 6 MHz）**，整体呈"单峰结构"。
+ **P3 阶段的 **`s_slot[n] = c_up[n] · sc[n]`：在时域上等效于对扩频基带进行了 15 MHz 方波子载波的调制，在频域上等效于**将 P2 的单峰谱搬移到 ±15 MHz 附近**，形成 BOC(15,3) 标志性的**双峰结构**：
    - 主峰中心：约 `±f_sc = ±15 MHz`；
    - 每个主峰的主瓣宽度：仍由码速率 `fc = 3 MHz` 决定，约为 ±3 MHz；
    - 峰间距：约 **30 MHz**；
    - 由于方波子载波含有奇次谐波分量（3 f_sc, 5 f_sc, ...），理论上在 ±45 MHz、±75 MHz 等处仍有副峰，但在 `fs = 36 MSPS` 采样下，**奈奎斯特带宽为 ±18 MHz**，更高次谐波已被折叠或超出观测带宽，主要观测对象仍是 ±15 MHz 的主双峰。

### 7.2 从 P2 到 P3 的自相关变化
+ **P2 阶段 **`c_up[n]`** 的自相关**：近似为一个以零延迟为中心的**单峰三角形**，峰宽约为 ±12 个样点（即 ±1 码片），这是典型的 DSSS 扩频码自相关特征。
+ **P3 阶段 **`s_slot[n]`** 的自相关**：由于 BOC 子载波在每个码片内包含 `f_sc / f_c = 5` 个完整的方波周期（等价地：`2 · f_sc / f_c = 10` 个**半周期**，这是标准 BOC(m,n) 术语中 `2m/n` 的本义），自相关函数在零延迟附近会呈现**标志性的 M 形结构**：
    - **主峰**位于零延迟；
    - 紧邻主峰两侧各出现一个**负的副峰**；
    - 更外侧还有若干个幅度递减的次级副峰；
    - 过了 ±1 码片延迟后迅速衰减到接近零。

> **工程重点**：**M 形自相关是 P3 阶段最重要的结构性验收特征之一。** 它也是后续接收端"主峰模糊问题"（peak ambiguity）的根源——任何 BOC 接收机的跟踪环设计都必须直面这一 M 形结构。在 P3 阶段必须把这条基线图样记录清楚，供后续阶段引用。
>

### 7.3 时域取值分析
+ 若 P1 锁定方案为 `sign(sin(0)) = +1`（或 `-1`），即 `sc[n] ∈ {±1}`，则 `s_slot[n] ∈ {+1, -1}`；
+ 若 P1 保留 `sign(0) = 0`【待确认，以 P1 锁定版本为准】，则 `s_slot[n]` 在每个子载波过零点样点处取 0，整体取值集合为 `{-1, 0, +1}`；
+ `s_slot[n]` 的取值 **完全等于 **`c_up[n] · sc[n]`，bit 的影响体现在整体符号上。

### 7.4 若本阶段不先验证，会给 P4/P5 带来什么工程风险
若跳过 P3 直接进入 P4 的 32 时隙拼接：

1. 一旦 P4 频谱异常，无法区分问题来自"单时隙基带错误"还是"时隙拼接错误"；
2. P4 的多时隙拼接本身会在频谱上引入 `1/T_frame = 125 Hz` 量级的细谱结构（主时钟周期性），这会进一步**掩盖**单时隙层面的双峰异常；
3. 若 P1 或 P2 的继承在 P3 中未经一次"组合验证"，则其任何边界错误（例如相位对齐、采样相位偏移、sign(0) 处理不一致）将直接污染 P4 输出；
4. 接收端在 P5/P6 捕获跟踪时将无法知道"正确的单时隙基带模板"到底应该长什么样，失去最重要的工程基线。

**结论：P3 是 P4 之前不可绕过的最后一道组合验证闸门。**

## 8. MATLAB 侧验证路径
### V1：输入继承与一致性检查
+ 载入 P1 锁定的 `sc[n]`，确认 `length(sc) == 9000`；
+ 载入 P2 锁定的 `c_up[n]`，确认 `length(c_up) == 9000`；
+ 确认两者都在 `fs = 36 MSPS` 采样率下生成；
+ 确认两者索引对齐（`n=0` 对应同一物理时刻）；
+ 打印两者的取值集合（`unique(sc)`、`unique(c_up)`），与 P1、P2 锁定结论一致。

### V2：单时隙 BOC 基带生成
```matlab
A  = 1;
bm = +1;                            % bit=+1 情况
s_slot_pos = A * bm * c_up .* sc;   % 逐点乘

bm = -1;                            % bit=-1 情况
s_slot_neg = A * bm * c_up .* sc;
```

验收项：

+ `length(s_slot_pos) == 9000`；
+ `s_slot_neg == -s_slot_pos`（全样点逐点相等）；
+ 取值集合与 §7.3 分析一致。

### V3：时域图分析
至少输出以下图：

1. `c_up[n]` 完整 9000 点时域图（参考 P2）；
2. `sc[n]` 完整 9000 点时域图（参考 P1）；
3. `s_slot[n]` 完整 9000 点时域图；
4. **前 100 点细节图**，同时叠加 `c_up[n]`、`sc[n]`、`s_slot[n]`，清晰展示"扩频码 × 子载波"的乘法关系；
5. **前 24 点（即前 2 码片）细节图**，验证每码片内 12 点的 BOC 图样。

### V4：频谱分析（P3 核心验收项）
```matlab
Nfft = 2^nextpow2(9000);            % 或 16384
S_slot = fftshift(fft(s_slot_pos, Nfft));
f_axis = (-Nfft/2:Nfft/2-1) * (fs / Nfft);
P_slot_dB = 20*log10(abs(S_slot) + eps);
```

验收项：

+ **双峰中心位置**：主峰应出现在 `f ≈ ±15 MHz`，容差不超过 ±1 个 FFT bin；
+ **峰间距**：约 30 MHz；
+ **主瓣宽度**：每个主峰在 -3 dB / -10 dB 水平下的主瓣宽度应与码速率 3 MHz 相当；
+ **对称性**：双峰幅度应基本一致（≤ 1 dB 差异）；
+ **与 P2 对比图**：将 P2 的 `s_dsss[n]` 频谱与 P3 的 `s_slot[n]` 频谱绘制在同一张图（或上下子图），直观展示"基带主瓣被搬移到 ±15 MHz"；
+ **异常分量检查**：
    - 零频附近是否有残留直流分量（若 `sc[n]` 严格过零对称，应为极低）；
    - 是否存在 ±3 MHz 处未被压制的基带残留（若有，需回查 `sc[n]` 的平衡性）；
    - 奈奎斯特边缘（接近 ±18 MHz）是否出现可疑能量。

### V5：自相关分析（P3 核心验收项）
```matlab
[R, lags] = xcorr(s_slot_pos, 'normalized');
```

验收项：

+ **零延迟主峰**：`R(lag=0) = 1`（归一化后）；
+ **M 形副峰**：主峰两侧紧邻位置应出现**负值副峰**，是 BOC(15,3) 的标志性特征；
+ **对比 P2**：将 P2 的 `c_up[n]` 自相关与 P3 的 `s_slot[n]` 自相关绘制在同一图上，清晰展示"单峰 → M 形"的演变；
+ **零延迟附近细节图**：放大至 `lag ∈ [-36, +36]` 样点（即 ±3 码片），标出主峰与副峰位置；
+ **记录副峰相对主峰的幅度比**（理论值与 BOC(15,3) 参数有关），供后续接收端设计引用。

### V6：bit 极性验证
+ 绘制 `s_slot_pos` 与 `s_slot_neg` 的前 100 点时域图，确认二者互为相反数；
+ 分别做 FFT，绘制幅度谱，确认**幅度谱完全一致**（bit 翻转仅改变相位，不改变功率谱）；
+ 分别做自相关，确认自相关函数完全一致（自相关对整体符号翻转不敏感）。

## 9. USRP + 频谱仪验证建议
**前提：MATLAB 侧 V1–V7 必须全部通过。** USRP 硬件验证仅作为定性确认手段，不作为算法正确性的判据。

### 9.1 发射配置
+ 发射内容：`I[n] = s_slot[n]`，`Q[n] = 0`；
+ 发射方式：将 9000 点单时隙基带以 **循环发射** 方式写入 USRP（本阶段允许简单循环，不等价于 P4 的 TDMA 帧结构）；
+ 采样率：`fs = 36 MSPS`（继承 P0）；
+ 中心频率 / 发射增益：继承 P0 基线配置【待确认最终值】。

### 9.2 频谱仪观测参数（参考总规划）
+ Span：中心频率 ±40 MHz；
+ RBW：300 kHz（参考总规划）；
+ VBW：≤ RBW；
+ 参考电平：根据 P0 记录调整，避免过载。

### 9.3 关键观测现象
+ **BOC(15,3) 双峰结构**：以发射中心频率为对称中心，左右约 ±15 MHz 处各出现一个主峰；
+ **峰间距**：约 30 MHz；
+ **两主峰幅度**：应基本对称；
+ **中心频率处**：应为凹陷（null），而非主峰——**这是区分 BOC 双峰谱与 P2 扩频单峰谱的最关键目视特征**。

### 9.4 与 P2 扩频谱的区分
| 观测位置 | P2（扩频基带） | P3（BOC 基带） |
| --- | --- | --- |
| 中心频率处 | 主峰（最大值） | 凹陷（null） |
| ±15 MHz 处 | 旁瓣，能量低 | 主峰 |
| 整体形态 | 单峰 sinc² | 双峰 |


### 9.5 若双峰不明显，故障树
| 现象 | 可能原因 | 定位方向 |
| --- | --- | --- |
| 中心频率处仍为主峰 | 未乘 `sc[n]` 或乘错采样率 | **算法问题**，回查 V2 乘法实现 |
| 双峰出现但中心不在 ±15 MHz | `sc[n]` 生成频率错误 | **算法问题**，回到 P1 |
| 双峰不对称（幅度差 > 3 dB） | DC offset、IQ 不平衡 | **硬件问题**，回到 P0 |
| 主峰宽度明显宽于 6 MHz（双边） | 码速率错误或 `c_up[n]` 重复点数错误 | **算法问题**，回到 P2 |
| 出现镜像峰（关于 DC 对称） | USRP IQ 不平衡 | **硬件问题**，P0 基线 |
| 完全无信号 | USRP 链路未开 | **硬件问题**，回到 P0 |

## 11. 常见误判与风险
1. **将"双峰不明显"误判为硬件故障**——实际可能是 P1 的 `sc[n]` 采样相位继承错误，或 P2 的 `c_up[n]` 上采样方式错误。**排查顺序应为：先 MATLAB，再 USRP**。
2. **将 P2 扩频谱与 P3 BOC 双峰谱混淆**——P2 中心频率处为主峰，P3 中心频率处为凹陷，这是最直接的区分特征。
3. `sc[n]`** 与 **`c_up[n]`** 长度不一致**——可能由于重新生成时误用不同码片数或不同采样率，必须在 V1 强制检查。
4. **bit 映射方向前后不一致**——P3 本阶段不固化映射方向，但必须记录本次验证使用的映射，供 P4 继承时不发生翻转。
5. **频谱不对称时跳过 P0 基线排查**——IQ 不平衡、DC offset、镜像干扰应优先在 P0 基线中解决，而非在 P3 中反复调整。
6. **未在 P3 单独验证就进入 P4**——P4 的时隙拼接会掩盖单时隙层面的双峰与 M 形异常，一旦出错定位成本极高。
7. **将自相关副峰误判为"错误信号"**——M 形副峰是 BOC 调制的固有特征，不是缺陷，必须在 P3 阶段明确记录其形态作为基线。
8. **fs=36 MSPS 下的频谱混叠误读**——奈奎斯特 ±18 MHz，15 MHz 主峰距边界仅 3 MHz，若主瓣展宽异常需警惕混叠影响。

## 12. 完成判据 / 进入下一阶段条件
P3 验收必须**同时满足**以下条件方可进入 P4：

1. `s_slot[n]`** 已稳定生成**，长度严格为 9000 点，采样率 36 MSPS；
2. **时域乘法关系正确**：`s_slot[n]` 与 `c_up[n] · sc[n]` 逐点相等；
3. **FFT 中明确观察到 BOC(15,3) 双峰结构**：
    - 主峰中心位于 ±15 MHz（±1 bin）；
    - 峰间距约 30 MHz；
    - 双峰幅度对称性 ≤ 1 dB；
    - 中心频率处为凹陷；
4. **自相关 M 形特征已确认**：主峰两侧出现负副峰，副峰位置与相对幅度已记录；
5. **bit=+1 / bit=-1 的关系已验证**：时域互为相反数，功率谱一致；
6. **P2 vs P3 频谱对比图、P2 vs P3 自相关对比图已完成并归档**；
7. **（若执行 USRP 侧验证）频谱仪上已确认 BOC 双峰，中心为凹陷**；
8. **单时隙基带模板已导出为可复用数据文件（如 **`s_slot_template.mat`**）**，供 P4 直接 load。

只有上述全部完成，才允许进入 P4 的 32 时隙拼接。

## 15. 应锁定的参数与遗留待确认项
### 15.1 P3 结束后必须锁定的内容
以下内容在 P3 验收通过后**必须锁定**，P4 及之后阶段不得再行修改：

1. **单时隙 BOC 基带生成公式**：`s_slot[n] = A · b_m · c_up[n] · sc[n]`，`n = 0, 1, ..., 8999`；
2. `sc[n]`** 的使用方式**：直接继承 P1 锁定版本，不重新生成、不重新对齐；
3. `c_up[n]`** 的使用方式**：直接继承 P2 锁定版本，不重新截短、不重新上采样；
4. `s_slot[n]`** 的长度**：严格 9000 点；
5. `s_slot[n]`** 的采样率**：严格 36 MSPS；
6. `s_slot[n]`** 的取值集合基线**（±1，或依 P1 的 sign(0) 方案含 0）；
7. **BOC 双峰频谱基线**：中心位于 ±15 MHz，峰间距 30 MHz，双峰对称；
8. **自相关 M 形基线**：主峰 + 两侧负副峰的形态与位置已记录；
9. **bit 极性关系**：bit 翻转仅导致整体符号翻转，功率谱不变；
10. **单时隙基带模板文件**：作为 P4 唯一可信输入。

### 15.2 仍然【待确认】但不阻碍 P3 → P4 推进的内容
1. **Gold 码多项式与初始状态**——继承 P2 的【待确认】项，不影响 P3 结构性验收；
2. **不同载荷是否使用不同 Gold 码**——属于 P4 多载荷编排的问题；
3. **bit=0 / bit=1 → ±1 的具体映射方向**——P3 仅验证结构性翻转关系；
4. **最终幅度归一化策略**（峰值归一化 / RMS 归一化 / 固定幅度）——P3 取 `A = 1` 足以完成验收；
5. **USRP 最终中心频率 / 发射增益**——P3 若做硬件验证采用 P0 基线，最终值待系统联调敲定；
6. **接收端的 BOC 跟踪环结构**（早减晚、bump-jump 等）——属于 P5/P6 范畴，P3 仅提供 M 形自相关基线作为设计输入。
