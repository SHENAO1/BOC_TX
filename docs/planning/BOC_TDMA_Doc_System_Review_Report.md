## 维度 2：参数一致性

**核心参数（fs=36 MSPS、fc=3 MHz、fsc=15 MHz、Tc、Tslot=250μs、Nslot=9000、Nframe=288000、Tframe=8ms、32 时隙、125 bps、扩频增益 28.75 dB）在全部 10 份文档中一致。自洽验证在多处重复出现，无矛盾。**

**但发现如下严重不一致：**

### P1 和 P5 额外关注

**P1（用户要求额外关注）：** P1 是本套文档中**质量最高的一份**。§6 的数学推导、§7 的分层分析（半周期/单周期/单码片）、§8 的 sign(0) 方案比较（含方案 A/B/C 对比、DC 分量 1/6 的定量推导）、§9 的 V1–V7 任务、§16.1 的 6 项锁定清单都非常扎实。**唯一瑕疵**是：§10.3 表中说"3 次谐波折叠到 9 MHz，5 次谐波折叠到 3 MHz，7 次谐波折叠后也在 3 MHz，与 5 次叠加"——此处描述大体正确，但若读者严格追究"叠加是同相还是反相"，P1 没给结论。对方案 B 频谱基线的细节影响有限，不阻碍 V3–V4 出图。

**P5（用户要求额外关注）：** P5 对 underflow 的工程对策（§8：带宽计算、缓冲区 ≥2 周期、UHD async_metadata 监控、主机负载建议、USB autosuspend 禁用）**覆盖非常全面**。§10.4 的 11 次定时截图方案、§10.6 故障树、§11 的"不要混改参数"纪律、§12 的 10 条常见误判，工程实用性强。**唯一可改进点**：§8.5 CPU 调度建议没有提到 Linux CPU governor 设为 `performance`（节能调度器会引入毫秒级抖动）。这是一个小的风险遗漏。

### 潜在的技术理解偏差（用户要求指出）

| 项 | 说明 |
|---|---|
| **P3 §7.2 把"2·fsc/fc = 10"解读为"每码片 10 个完整方波周期"** | 这是技术理解偏差。标准 BOC(m,n) 术语中 `2m/n` 代表"每码片的**半周期数**"，不是完整周期数。完整周期数是 `m/n = 5`。这不仅是文档写作问题，也反映了对 BOC 术语的误用。 |
| **系统规划 §6 D-6 说"M 形峰值间距 ≈ 1/fsc"** | 这是技术理解偏差。BOC 自相关 M 形的相邻主副极值间距是 `Tsc/2 = 1/(2·fsc)`，不是 `Tsc`。P3 和 P7 的理解是对的。 |
| **P6 §7.2 C-05 说"两个正副峰"** | M 形的副峰是负的（valleys），不是正的。这也是 BOC 自相关结构的基本事实误用。 |

这三个错误彼此相关，都指向对 BOC(15,3) 自相关形态的细节理解不稳。建议在 P1 §16.1 L-03 图样的基础上，补一个"单时隙 BOC 自相关解析表达式 + 图示"作为权威参考，所有下游文档引用此节而不重复推导。
