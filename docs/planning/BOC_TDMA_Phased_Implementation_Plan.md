## 1. 分阶段设计原则

### 1.1 为什么不能一次性完成整个工程

本系统是一个**多层叠加的复杂体制**，从底层到顶层依次包含：

```
采样率与硬件链路
    └── BOC 子载波离散化
         └── Gold 码生成与截短
              └── 扩频调制（码片 × 子载波）
                   └── 单时隙信号成形
                        └── TDMA 帧结构（32 时隙拼接）
                             └── 连续循环发射
```

如果一上来就实现完整链路，存在以下工程风险：

**风险一：问题难以定位。** 当在频谱仪上观测到异常时，无法判断是 BOC 离散化问题、Gold 码生成问题、时隙拼接问题还是 USRP 链路问题。

**风险二：调试代价极高。** 完整系统有 288000 点/周期的数据量，任何环节出错均需从头追查。

**风险三：BOC 子载波离散化问题被淹没。** $36\,\text{MSPS} / (2 \times 15\,\text{MHz}) = 1.2$ 这一关键非整数采样问题，如果不单独分析，在完整系统联调时的影响难以量化和隔离。

**风险四：硬件链路从未单独打通。** 若 USRP 基础配置存在问题（时钟、增益、采样率等），会被误认为是信号生成算法问题。

### 1.2 分阶段的核心思想

> **每一阶段只引入一个新的复杂性维度，其余维度保持已验证状态。**

具体体现为：
- 先硬件后算法：先保证 USRP 能正常发射已知信号，再逐步加入算法
- 先单元后系统：先验证 BOC 子载波、Gold 码各自独立正确，再组合
- 先单帧后循环：先保证单周期波形正确，再做连续发射
- 先定性后定量：先在频谱仪上看到正确"形状"，再做精确参数测量

### 1.3 阶段间的依赖关系

```
Phase 0 ──► Phase 1 ──► Phase 2 ──► Phase 3 ──► Phase 4 ──► Phase 5 ──► Phase 6
  硬件         BOC子        Gold码      单时隙       整周期        循环         参数
  链路         载波         扩频        组合         TDMA帧        发射         优化
  打通         验证         验证        验证         验证          验证         验证
```

**原则：前一阶段不通过，绝不进入下一阶段。**

## 3. 推荐的阶段划分方案

### 3.1 阶段划分的调整说明

相对于"先打通 USRP → 验证 BOC → 验证扩频 → 组合 → TDMA → 循环 → 优化"的原始思路，本方案做了以下调整：

**调整 0：在 P0 内部显式增加无硬件依赖的 `P0-A：参数级自洽检查`。**  
理由：系统规划 §6 阶段 A 已明确存在 `check_params.m`，但此前未在阶段总规划中被正式纳入执行顺序。将其落位为 P0 的前置热身关卡后，可以先验证参数自洽，再进入 P0-B 的 USRP 联调，避免把“公式理解错误”带入硬件阶段。

**调整 1：P1（BOC 子载波）与 P2（Gold 码扩频）设计为可并行推进。**  
理由：两者均以 P0（USRP 链路）为前提，但彼此无依赖。在工程时间充裕时可并行；资源受限时先做 P1（因为 BOC 离散化问题的不确定性更高，是本体制最大的技术风险）。

**调整 2：P1 中增加 MATLAB 内部的理论分析阶段，不急于上硬件。**  
理由：BOC 子载波的离散化问题需要先在 MATLAB 中彻底搞清楚（包括 `sign(0)` 问题的处理方案），再上 USRP 验证。否则硬件测试结果无法解读。

**调整 3：P4（帧生成）先在 MATLAB 内部完全验证，再上 USRP。**  
理由：288000 点的帧数据调试成本高，如果 MATLAB 仿真本身有误，USRP 实验无意义。

## 4. 每个阶段的详细说明

---

### Phase 0：USRP 基础链路打通

#### 4.0.1 为什么需要这一阶段

在做任何信号体制验证之前，必须先确认"硬件链路本身是通的"。这包括：
- USRP 能被主机识别并通信
- USRP 能以 36 MSPS 的速率持续发射
- IQ 文件的格式（sc16、字节序、采样率配置）是正确的
- 频谱仪能接收到信号并读数准确

如果跳过此阶段，后续所有"信号不对"的问题都无法区分是算法问题还是硬件链路问题。

#### 4.0.2 本阶段应该做什么

**任务 0（P0-A）：参数级自洽检查**
- 在不接任何硬件的前提下运行 `matlab/check_params.m`
- 校验 `Tc`、`Nc`、`Nslot`、`Tslot`、`Nframe`、`Tframe`、数据率、处理增益、`fs/fsc`、`fsc/fc`、`fsc*Tslot`
- 允许出现 1 个预期警告：`fs/fsc = 2.4` 为非整数采样比；其余检查项必须全部通过
- 明确输出 `PASS / WARN / FAIL` 汇总，并写明“当前仍不触碰硬件”

**任务 1（P0-B）：USRP 基本配置验证**
- 确认 USRP 型号支持 36 MSPS 采样率（主时钟频率配置）
- 确认主机与 USRP 的接口（USB3 / GigE / PCIe）带宽满足 36 MSPS × 4 bytes = 144 MB/s

**任务 2：发射已知 CW（连续波）信号**
- 在 MATLAB 中生成一个纯单频余弦波：$x[n] = \cos(2\pi f_0 n / f_s)$
- 选取 $f_0$ 为一个容易在频谱仪上识别的频率（例如 1 MHz），使得射频中心频率 + 1 MHz 处可见一个单峰
- 将 IQ 数据（`complex(cos(...), sin(...))`）以 sc16 格式写入文件并用 USRP 工具发射

**任务 3：IQ 格式确认**
- 验证 I、Q 字节序、交织方式（先 I 后 Q）
- 验证归一化系数（int16 满量程 = 32767）

**任务 4：采样率设置确认**
- 明确记录 USRP 主时钟与发射采样率的配置关系，防止后续出现系统性频率误差

#### 4.0.3 本阶段不应该做什么

- 不要试图发射 BOC 信号
- 不要试图验证任何扩频参数
- 不要同时调整多个 USRP 参数

#### 4.0.4 最适合观察的现象

- 频谱仪上的 CW 信号峰是否在预期频率（偏差应 < 采样率误差）
- 信号功率是否随 USRP 发射增益设置线性变化
- 是否存在 DC 分量（零频处的尖峰，USRP 模拟前端 DC offset 问题）
- 是否有明显的镜像频率分量（IQ 不平衡）

#### 4.0.5 如果不在本阶段解决，后续会被放大的问题

| 问题 | 放大方式 |
|---|---|
| IQ 字节序错误 | 导致后续所有信号频谱镜像翻转 |
| 采样率配置错误 | 导致所有频率关系系统性偏移 |
| DC offset 严重 | 干扰 BOC 双峰结构观测 |
| USRP 与主机带宽不足 | 后续 36 MSPS 连续发射时频繁 underflow |

#### 4.0.6 本阶段结束时应形成的记录

- USRP 型号、主机配置、接口类型的记录
- CW 测试的频谱仪截图（注明中心频率、分辨率带宽、参考电平）
- IQ 文件格式确认记录
- 采样率配置参数记录

**进入 P1/P2 的条件：** `matlab/check_params.m` 无 FAIL，且频谱仪上稳定看到预期 CW 峰，无明显 DC offset 和镜像分量，主机连续发射 2 分钟无 underflow 报告。

### Phase 1：BOC 子载波离散化验证

#### 4.1.1 为什么需要这一阶段

这是**本系统最大的技术不确定性来源**。

$f_s / f_{sc} = 36\,\text{MHz} / 15\,\text{MHz} = 2.4$ 意味着：
- BOC 方波子载波的半周期 = 1.2 个采样点（非整数）
- 每隔 12 个采样点（= 1 码片），BOC 子载波完成 5 个完整周期（= $15\,\text{MHz} \times 1/3\,\mu\text{s} = 5$），**这是整数**
- 因此非整数采样不是全局混乱，而是在码片级别上"刚好收敛"

这个微妙的关系必须独立分析清楚，才能确保：
1. MATLAB 生成的离散 BOC 子载波与理论预期的偏差量是已知且可接受的
2. 接收端的相关模板基于正确的离散版本而非理想连续版本
3. 频谱仪上观测到的谱形符合离散化后的预期而非理想 BOC(15,3) 理论谱

#### 4.1.2 本阶段应该做什么

**任务 1：MATLAB 内部理论分析（先做，不上硬件）**

分析以下关键数值关系：

| 关系 | 计算 | 结论 |
|---|---|---|
| 子载波周期内采样点数 | $f_s / f_{sc} = 2.4$ | 非整数 |
| 子载波每时隙完整周期数 | $f_{sc} \times T_\text{slot} = 3750$ | 整数 ✓ |
| 每码片内子载波周期数 | $f_{sc} \times T_c = 5$ | 整数 ✓ |
| 每码片内子载波采样点数 | $N_c = 12$，子载波 5 周期 × 2.4 点 = 12 点 | 恰好整合 ✓ |

**关键结论**：虽然子载波半周期不是整数，但每个码片（12点）内，BOC 子载波恰好完成 5 个整周期。因此在码片级别上，非整数问题不会累积漂移。但在子载波半周期内，采样点的分布是不均匀的（1点或2点交替）。

**任务 2：`sign(sin(...))` 过零点处理方案确认**

验证在 $n = 0, 1, \ldots, 11$（一个码片）中，$\sin(5\pi n/6)$ 是否有零值，并确定处理方案。

**任务 3：在 MATLAB 中生成纯 BOC 子载波并分析**
- 生成 9000 点离散 BOC 子载波序列 `sc[n]`
- 绘制时域图（前 50 点），观察实际方波形状
- 绘制 FFT 频谱，与理想 BOC 子载波频谱对比
- 计算 `sc[n]` 的自相关，分析主峰宽度和旁瓣

**任务 4：USRP 发射单独的 BOC 子载波信号**
- 将 `sc[n]` 作为 I 分量，Q 分量设为 0，写入 IQ 文件
- 用 USRP 发射，用频谱仪观测

#### 4.1.3 本阶段不应该做什么

- 不要引入 Gold 码
- 不要引入数据 bit 调制
- 不要急于做单时隙完整信号

#### 4.1.4 最适合观察的现象

**MATLAB 内部：**
- 离散方波是否在每个码片（12点）边界处对齐
- 频谱中子载波频率处是否有明显功率集中（基波在 15 MHz，奇次谐波在 45、75 MHz 等）
- 与理想 $\text{sign}(\sin(2\pi \cdot 15\,\text{MHz} \cdot t))$ 的连续谱相比，离散化引入了哪些差异

**频谱仪上：**
- 在射频中心频率两侧 15 MHz 处是否出现明显功率集中
- 谱形是否对称（若不对称，提示 IQ 或归一化问题）
- 奇次谐波是否按预期衰减

#### 4.1.5 如果不在本阶段解决，后续会被放大的问题

| 问题 | 放大方式 |
|---|---|
| `sign(0)` 未处理 | 9000 点中出现若干"空洞"，影响扩频相关性能，且难以在完整信号中定位 |
| 对离散谱形预期不正确 | P3 组合后频谱仪显示"异常"时不知道是正常的离散化效果还是真正的错误 |
| 未确认码片边界与子载波相位的关系 | 接收端相关模板设计错误 |

#### 4.1.6 本阶段结束时应形成的记录

- 离散 BOC 子载波的时域波形图（MATLAB 生成，标注关键采样点）
- 离散 BOC 子载波的 FFT 频谱图（对比理论连续谱）
- `sign(0)` 问题的出现频率和处理方案确认
- 每码片内子载波相位关系的分析结论
- 频谱仪实测图（若已上 USRP 发射）

**进入 P3 的条件：** MATLAB 中 BOC 子载波的频谱形状已理解并接受，过零点处理方案已确定并锁定。

### Phase 2：Gold 码扩频基带验证

#### 4.2.1 为什么需要这一阶段

Gold 码的生成、截短和扩频调制需要独立验证，理由：
- Gold 码初始多项式和初相是关键参数，必须确认无误
- 1023 截短为 750 的操作会影响相关性能，需要独立量化
- 扩频后的功率谱（连续谱展宽到 3 MHz 码率对应的 sinc 形状）是接收端设计的基础
- 此阶段不引入 BOC 子载波，问题定位更干净

#### 4.2.2 本阶段应该做什么

**任务 1：Gold 码生成与验证**
- 确认 10 级 Gold 码的两个 m 序列生成多项式
- 生成完整 1023 chips 的 Gold 码序列
- 验证自相关性：峰值 = 1023，旁瓣 ≤ $\{-1, -65, 63\}$（10级Gold码的理论值）
- 截短至 750 chips，重新计算自相关，记录旁瓣电平变化

**任务 2：扩频基带信号生成（BPSK-DSSS，不含BOC）**
- bit = 1（固定），Gold 码 BPSK 扩频，上采样 12 倍
- 生成 9000 点基带信号 `s_dsss[n]`
- 分别用 bit=0 和 bit=1 生成，验证符号极性

**任务 3：频谱分析**
- 理论功率谱主瓣宽度 ≈ $2 \times 3\,\text{MHz} = 6\,\text{MHz}$（矩形脉冲 sinc 谱）
- 验证 MATLAB FFT 结果与理论一致

**任务 4：USRP 发射扩频基带（不含BOC）**
- 将 `s_dsss[n]` 作为 I 分量发射
- 频谱仪观测扩频谱展宽效果

#### 4.2.3 本阶段不应该做什么

- 不要引入 BOC 子载波
- 不要做多时隙拼接

#### 4.2.4 最适合观察的现象

- Gold 码自相关函数的峰值和旁瓣（MATLAB）
- 截短前后自相关性能的变化（MATLAB）
- 频谱仪上看到以射频中心频率为中心、宽度约 6 MHz 的扩频谱
- 改变 bit 值时，频谱不变（只有相位翻转），验证 BPSK 特性

#### 4.2.5 如果不在本阶段解决，后续会被放大的问题

| 问题 | 放大方式 |
|---|---|
| Gold 码初相错误 | 接收端永远无法捕获，但在 P2 阶段容易通过自相关检验发现 |
| 截短方式错误（不是前750） | 相关性能与设计值不符，影响接收灵敏度 |
| 上采样方式错误（非 repelem） | 频谱形状异常，但混入 BOC 后极难诊断 |

#### 4.2.6 本阶段结束时应形成的记录

- Gold 码初相参数记录（锁定值）
- 完整 1023 chips vs 截短 750 chips 的自相关对比图
- 扩频基带信号 FFT 频谱图
- 频谱仪实测图（标注主瓣宽度）

**进入 P3 的条件：** Gold 码自相关符合预期，截短前后的性能差异已量化，USRP 发射扩频谱形状正确。

### Phase 3：单时隙 BOC 基带组合验证

#### 4.3.1 为什么需要这一阶段

这是**整个发端体制的核心信号**。将 P1 和 P2 的成果相乘，得到完整的单时隙 BOC(15,3) 基带信号：

$$s[n] = b_m \cdot c[n] \cdot sc[n]$$

相乘操作看似简单，但实际上引入了新的特性：
- BOC 调制后功率谱从扩频谱（单峰，宽约 6 MHz）变为双峰谱（双峰，中心在 ±15 MHz）
- 自相关函数从三角单峰变为"M 形"双峰
- 这些变化必须在单时隙级别先确认正确，才能进入帧结构

#### 4.3.2 本阶段应该做什么

**任务 1：生成单时隙 BOC 基带信号**
- 使用 P1 确定的 `sc[n]`（含过零点处理）
- 使用 P2 确定的 `c[n]`（Gold 码截短 + 上采样）
- 相乘：`s_slot = b_m * c_up .* sc`

**任务 2：时域分析**
- 验证 `s_slot` 取值在 $\{-1, 0, +1\}$ 中（0 仅来自过零点处理，应极少或无）
- 观察时域波形与单独的 `c[n]`、`sc[n]` 的关系

**任务 3：频域分析（关键）**
- 计算 FFT，验证出现 BOC(15,3) 双峰结构
- 理论双峰中心约在 $\pm 15\,\text{MHz}$（基带），主瓣宽度由码速率 3 MHz 决定
- 对比：纯扩频谱（P2）vs BOC 调制后谱（P3），观察谱形变化

**任务 4：自相关分析**
- 计算 `s_slot` 的自相关函数
- 验证"M 形"双峰特征，两个主峰的间距 $\approx 1/(2 \times 15\,\text{MHz}) = 33.3\,\text{ns}$，对应 $\approx 1.2$ 个采样点

**任务 5：USRP 发射单时隙 BOC 基带**
- 将 9000 点的 `s_slot` 写入 IQ 文件并循环发射
- 频谱仪观测双峰结构

#### 4.3.3 本阶段不应该做什么

- 不要做多时隙拼接
- 不要引入 TDMA 帧结构

#### 4.3.4 最适合观察的现象

- 频谱仪上 BOC(15,3) 的标志性双峰结构（这是最重要的定性验证）
- 双峰间距是否符合 $2 \times 15\,\text{MHz} = 30\,\text{MHz}$
- 峰值对称性（若不对称，可能有 DC offset 或 IQ 不平衡）
- 带外功率是否合理（BOC 谱的带外滚降特性）

#### 4.3.5 如果不在本阶段解决，后续会被放大的问题

| 问题 | 放大方式 |
|---|---|
| BOC 双峰不对称 | 进入 TDMA 帧后，多时隙拼接会叠加多个单时隙的不对称性，频谱分析变得复杂 |
| 自相关 M 形未确认 | 接收端设计的相关模板可能基于错误的形状 |
| 信号幅度未归一化 | USRP DAC 过载或量化噪声严重，在单时隙阶段即可发现并修正 |

#### 4.3.6 本阶段结束时应形成的记录

- 单时隙 BOC 基带信号的时域图（完整 9000 点，以及前 200 点细节）
- BOC 调制前后的 FFT 频谱对比图（P2 扩频谱 vs P3 BOC 谱）
- 自相关函数图（标注 M 形双峰位置和间距）
- 频谱仪实测双峰截图（标注双峰中心频率和峰间距）

**进入 P4 的条件：** 频谱仪上确认 BOC(15,3) 双峰结构，双峰间距约 30 MHz，MATLAB 自相关呈 M 形。

### Phase 4：单周期 TDMA 帧生成验证

#### 4.4.1 为什么需要这一阶段

从单时隙到完整 TDMA 帧引入了以下新复杂性：
- 32 个时隙的拼接与索引管理
- 非发送时隙的静默（零填充）处理
- 帧内时隙排布（固定时隙位置）
- 占空比效应对平均功率谱的影响（4/32 = 12.5% 占空比会使平均功率谱低约 9 dB）

这些问题必须独立验证，才能在 P5 连续发射时自信地说"帧结构是正确的"。

#### 4.4.2 本阶段应该做什么

**任务 1：建立时隙分配表**
- 明确 32 个时隙中哪些是各载荷的发送时隙
- 建立 `slot_config` 数据结构（时隙编号 → 载荷编号 or 静默）

**任务 2：生成完整单周期帧**
- 初始化 288000 点全零向量
- 按时隙分配表填充各发送时隙的 `s_slot`
- 验证帧结构：统计非零时隙数量，检查时隙边界索引

**任务 3：帧结构时域分析**
- 绘制完整 288000 点时域波形（绘制包络或分段绘制）
- 清晰识别发送时隙（高功率段）与静默时隙（零段）

**任务 4：帧结构频域分析**
- 对完整帧做 FFT，对比单时隙 FFT
- 预期：BOC 双峰结构保持，但平均功率降低约 $10\log_{10}(4/32) \approx -9\,\text{dB}$（若4个时隙发送）
- 分析时隙边界处的突变对频谱的影响（旁瓣增加）

**任务 5：USRP 发射单周期帧（单次，非循环）**
- 先验证单次发射的帧数据是否正确
- 用频谱仪的"平均"或"Max Hold"模式观测帧内功率谱

#### 4.4.3 本阶段不应该做什么

- 不要在帧结构验证之前就做循环发射
- 不要在时隙分配表确认之前生成帧数据

#### 4.4.4 最适合观察的现象

- 时域上发送时隙与静默时隙的交替（用频谱仪的时域触发功能）
- 频域上的 BOC 双峰形状是否在帧占空比降低后仍然可识别
- 时隙边界处的射频尖峰（突变引起的宽带辐射）

#### 4.4.5 如果不在本阶段解决，后续会被放大的问题

| 问题 | 放大方式 |
|---|---|
| 时隙索引偏移（MATLAB 1-based vs 0-based混淆） | 循环发射时帧结构持续错位，导致所有时序测试失效 |
| 帧功率谱分析未做 | P5 频谱仪测试时发现功率"偏低"但不知道是正常的占空比效应 |
| 时隙边界突变未量化 | 后续接收端设计的保护时隙长度无依据 |

#### 4.4.6 本阶段结束时应形成的记录

- 时隙分配表（明确每个时隙号对应的载荷或静默）
- 完整帧时域图（标注发送时隙位置）
- 单帧 FFT 频谱图（与单时隙频谱对比）
- USRP 发射单帧的频谱仪截图

**进入 P5 的条件：** MATLAB 帧结构验证完整无误，USRP 单次发射的频谱仪截图与 MATLAB 仿真吻合，时隙索引管理经过代码审查。

### Phase 5：USRP 连续循环发射验证

#### 4.5.1 为什么需要这一阶段

单次发射与连续循环发射之间有本质差异：
- 循环发射要求帧边界处无缝拼接（第 288000 点与下一帧第 1 点之间的过渡）
- 主机需以 144 MB/s 的速率持续向 USRP 供数，任何 underflow 都会导致帧结构破坏
- 实际系统发端需要长时间稳定运行，必须验证 USRP 平台的时序稳定性

#### 4.5.2 本阶段应该做什么

**任务 1：帧边界过渡分析**
- 分析连续两个周期之间的拼接点处，信号是否有相位跳变或幅度突变
- 特别注意：末尾时隙（若为静默）到下一帧首时隙（若为发送）的过渡

**任务 2：IQ 循环缓冲区设置**
- 建议缓冲区大小 ≥ 2 个周期（576000 点），监控 underflow 事件

**任务 3：长时间稳定性测试**
- 连续发射至少 5 分钟（37500 个周期）
- 记录 underflow 事件次数
- 频谱仪每 30 秒截图一次，观测谱形稳定性

**任务 4：帧同步标志验证（若有）**
- 若系统要求帧同步，验证帧头时隙发送内容的正确性

#### 4.5.3 本阶段不应该做什么

- 不要在稳定性测试中同时修改信号参数

#### 4.5.4 最适合观察的现象

- 频谱仪上 BOC 双峰是否在长时间发射中保持稳定（无频谱漂移、无功率闪烁）
- 主机 CPU 占用率与 underflow 率的关系
- 帧边界处是否有周期性射频毛刺（可用频谱仪触发功能捕获）

#### 4.5.5 本阶段结束时应形成的记录

- 5 分钟稳定性测试记录（underflow 次数、频谱仪截图时间序列）
- 帧边界过渡的时域分析图
- USRP 最终配置参数表（采样率、增益、中心频率、缓冲区大小）

**进入 P6 的条件：** 5 分钟连续发射无 underflow，频谱仪截图显示谱形稳定，帧边界无明显毛刺。

### Phase 6：参数扫描与性能优化（可选）

#### 4.6.1 为什么需要这一阶段

当 P5 完成后，系统已具备完整发端能力。P6 的目标是：
- 锁定最优参数配置（为接收端设计提供确定性输入）
- 量化关键参数（Gold 码截短长度、过零点处理方式等）对信号质量的影响
- 为后续接收端设计提供发端参数规格书

#### 4.6.2 本阶段应该做什么

- 扫描 USRP 发射增益，确定不失真工作区间
- 对比不同过零点处理方案（加 epsilon vs 不处理）对频谱的影响
- 分析 Gold 码不同截短位置对自相关旁瓣的影响
- 生成完整的"发端参数规格书"

#### 4.6.3 本阶段结束时应形成的记录

- 参数扫描结果汇总表
- 最终发端参数规格书（供接收端设计团队使用）

## 7. 风险最高的阶段识别

### 7.1 风险等级排序

| 阶段 | 风险等级 | 主要风险来源 |
|---|---|---|
| **P1** | 🔴🔴🔴 极高 | BOC 子载波非整数采样的理论分析难度高；`sign(0)` 问题处理不当会在后续所有阶段引入误差 |
| **P5** | 🔴🔴🔴 极高 | USRP 连续发射的 underflow 问题与主机系统配置强相关，调试难度大 |
| **P3** | 🔴🔴 高 | 两个独立信号相乘后的频谱变化是否符合预期，是对 P1+P2 工作质量的综合检验 |
| **P0** | 🔴🔴 高 | USRP 硬件配置问题（主时钟、IQ 格式）若未在此阶段解决，后续所有测试数据均不可信 |
| **P4** | 🟡 中 | 技术风险低，但索引管理的细节错误容易在此阶段引入 |
| **P2** | 🟡 中 | 技术成熟，风险主要来自 Gold 码初相参数未锁定 |
| **P6** | 🟢 低 | 纯优化阶段，无新增技术风险 |

### 7.2 P1 风险详析

P1 是**全程最关键的阶段**，原因如下：

**技术风险 1：非整数采样的影响需要被正确理解和接受。**  
$f_s/f_{sc} = 2.4$ 产生的离散方波，其频谱与理论连续 BOC(15,3) 谱有偏差。这种偏差是系统性的、已知的、可接受的——但必须在 P1 中量化并在文档中明确。否则后续每次看到"异常"的频谱，都会怀疑是 P1 的问题而不是后续叠加的问题。

**技术风险 2：`sign(0)` 的处理方案影响所有后续阶段。**  
一旦确定处理方案（如加 epsilon），必须在所有后续阶段保持一致。若 P1 不明确，P3、P4 可能使用不同处理方式，导致比较基准混乱。

**技术风险 3：离散 BOC 子载波的相位起始点影响接收端设计。**  
若不在 P1 中明确"每个码片的第 0 个采样点对应子载波的哪个相位"，接收端的相关模板设计将无从下手。

### 7.3 P5 风险详析

P5 是**工程实现风险最高的阶段**：

- 主机 USB3/GigE 接口的实际可用带宽受操作系统调度影响，可能在高负载时达不到 144 MB/s
- 36 MSPS 是部分 USRP 型号的接近上限，时钟配置不当会导致数据丢失
- Underflow 问题在 P0 阶段（发射 CW）不会暴露，因为 CW 的数据量极小

## 8. 推荐的执行顺序

### 8.1 总体执行路线

### 8.3 最小可行验证路径（资源受限时）

如果时间或资源受限，建议的最小验证路径为：

```
P0（必须）→ P1 MATLAB 部分（必须）→ P2（必须）→ P3（必须）→ P5（必须）
```

跳过 P1 的 USRP 验证和 P4 的帧结构详细分析，直接从 P3 进入 P5，但风险是：P4 的帧索引问题可能在 P5 中才暴露，调试代价更高。**不推荐，仅作为进度压力下的备选方案。**
