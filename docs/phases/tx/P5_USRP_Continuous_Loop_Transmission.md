## 1. 文档目的与适用范围

### 1.1 文档目的

本文件服务于 **P5：USRP 连续循环发射验证** 阶段。其核心目标是：**在 P4 已经生成并锁定了正确的单周期帧 `s_frame[n]` 的前提下，验证该帧作为 USRP 循环发射模板，在长时间连续播放条件下能否保持信号稳定、帧边界无缝、频谱不变、且不出现主机-硬件供数中断。**

P5 必须独立存在，原因如下：

+ P4 阶段验证的是"单周期帧的生成正确性"，其 USRP 硬件验证仅采用"短时重复播放"形式——播放 10～100 个周期足以让频谱仪稳定出谱形，但并不构成真正意义上的长时间连续发射；
+ P5 是工程链路上**第一次要求 USRP 以 36 MSPS 的速率持续、不中断地向 DAC 供数至少 5 分钟（约 37500 个周期）的阶段**；
+ 在 P4 的短时重复中，underflow 问题不会暴露或影响极其有限（因为数据量小、每次发射间可能有隐含间歇），而 P5 必须以"零 underflow"为验收标准；
+ 帧边界处从 `n = N_frame-1` 到下一周期 `n = 0` 的拼接关系，在 P4 阶段从未被真正测试过——P4 只看"一帧之内"，P5 才需要看"帧与帧之间"；
+ 如果不在 P5 单独做连续发射验证就直接进入 P6（参数优化），P6 的参数扫描结果将不可信——因为尚未确认基线发射本身是否稳定。

### 1.2 适用范围

本文件覆盖以下工作：

+ 连续循环发射的离散时间模型建立
+ 帧边界连续性的理论分析与 MATLAB 侧验证
+ IQ 循环缓冲区设计与 underflow 风险评估
+ USRP 长时间连续发射（≥ 5 分钟）的测试方案
+ 频谱仪长期稳定性观测的执行方案
+ 最终发射参数的记录与锁定

本文件 **不覆盖** 以下工作：

+ BOC 子载波离散化分析（已在 P1 完成并锁定）
+ Gold 码生成与扩频验证（已在 P2 完成并锁定）
+ 单时隙 BOC 基带组合验证（已在 P3 完成并锁定）
+ 单周期 TDMA 帧结构拼接与索引管理（已在 P4 完成并锁定）
+ 参数扫描与性能优化（属 P6）
+ 接收端捕获、跟踪、解调设计（超出发端验证范畴）

### 1.3 P5 的本质定位

> **P5 不重新生成 `s_slot[n]`，不重新生成 `s_frame[n]`，不重新分析 P1/P2/P3/P4 的算法正确性。**
>
> **P5 只验证"已经正确的帧"能否被 USRP 长时间、连续、稳定、无明显毛刺地发射出来。**

P5 的新增复杂性只有一个：**连续循环发射与稳定性验证**。

## 3. 本阶段目标

本阶段只引入一个新的复杂性维度：**连续循环发射与长时间稳定性验证**。具体目标如下：

+ **G1**：建立单周期帧 `s_frame[n]` 的周期性重复连续发射离散模型；
+ **G2**：在 MATLAB 侧分析帧边界过渡点（`n = N_frame-1` → 下一周期 `n = 0`）的信号连续性，识别并记录可能存在的突变或毛刺；
+ **G3**：设计并实施 IQ 循环缓冲区方案，缓冲区至少覆盖 2 个完整周期（≥ 576000 点），以降低 underflow 风险；
+ **G4**：在 USRP 上实施连续循环发射，持续时间至少 5 分钟（约 37500 个周期），记录 underflow 事件次数；
+ **G5**：在频谱仪上进行长时间稳定性观测，验证 BOC(15,3) 双峰结构在整个测试期间保持稳定，无频率漂移、无功率闪烁、无周期性毛刺；
+ **G6**：记录最终可用于后续阶段的发射配置参数（采样率、中心频率、增益、缓冲区大小等）；
+ **G7**：形成完整的长时间稳定性测试记录集（underflow 日志、频谱仪截图时间序列、帧边界分析图），作为进入 P6 的可追溯基线。

## 6. 连续循环发射的理论模型

### 6.1 为什么单次发射和连续循环发射是两个不同层次的问题

P4 阶段验证的是"一帧之内"的正确性——288000 个样点的帧结构、时隙索引、频谱特征。P4 的 USRP 验证采用"短时重复播放"（10～100 个周期），其目的仅是让频谱仪在平均/保持模式下稳定出谱形，此时：

+ 播放器可能在两次帧播放之间插入短暂的"死区"或"归零段"（取决于实现方式），而短时观测的频谱仪不一定能捕捉到这种间歇；
+ 主机侧的数据供给仅需持续数百毫秒到几秒，远未触及操作系统调度抖动和 USB/PCIe 带宽饱和的边界；
+ 帧与帧之间的拼接点是否存在相位突变或幅度毛刺，在 P4 的频谱仪观测中无法辨识。

**连续循环发射是根本不同的工程场景**：

+ 要求 USRP 的发射流（TX stream）在整个测试期间（≥ 5 分钟）**不中断**——即不允许出现任何 underflow（欠载）事件；
+ 帧边界必须**无缝**——从第 $k$ 个周期的第 287999 号样点到第 $k+1$ 个周期的第 0 号样点，时序上没有任何间隙、跳变或额外插入；
+ 主机必须以恒定速率 144 MB/s 持续向 USRP 供数，持续数分钟。

### 6.2 周期性重复的离散模型

定义连续循环发射信号 $s_\text{TX}[m]$ 为 `s_frame[n]` 的无限周期性重复：

$$ \boxed{\; s_\text{TX}[m] = s_\text{frame}[\,m \bmod N_\text{frame}\,], \quad m = 0, 1, 2, \ldots \;} $$

其中：

+ $N_\text{frame} = 288000$；
+ $m$ 为全局连续发射样点索引，从 0 开始单调递增；
+ $m \bmod N_\text{frame}$ 保证了周期性重复。

等价地，若将 $m$ 拆解为周期编号 $k$ 和帧内索引 $n$：

$$ k = \left\lfloor \frac{m}{N_\text{frame}} \right\rfloor, \quad n = m - k \cdot N_\text{frame} $$

则：

$$ s_\text{TX}[m] = s_\text{frame}[n] $$

> **关键工程约束**：上式成立的前提是**所有周期使用同一个 `s_frame[n]` 模板**。在当前 P5 阶段中，`s_frame[n]` 是常量模板（P4 锁定），不随周期 $k$ 变化。未来若数据 bit 逐周期更新（`b_m` 随 $k$ 变化），属于更高层级的系统联调，不在 P5 范畴。

### 6.3 帧边界拼接的精确描述

连续循环发射中最关键的拼接点发生在：

$$ m = k \cdot N_\text{frame} - 1 \longrightarrow m = k \cdot N_\text{frame} $$

即第 $k-1$ 个周期的**最后一个样点**到第 $k$ 个周期的**第一个样点**之间的过渡。在帧内索引上，这对应于：

$$ s_\text{frame}[N_\text{frame} - 1] \longrightarrow s_\text{frame}[0] $$

即从 $n = 287999$ 到 $n = 0$ 的拼接。

由 P4 的 `slot_config` 可知：

+ $n = 287999$ 属于 SLOT-32（数学 `i = 31`），在当前工程假设下为**静默时隙**，因此 $s_\text{frame}[287999] = 0$；
+ $n = 0$ 属于 SLOT-1（数学 `i = 0`），在当前工程假设下为**发送时隙**，因此 $s_\text{frame}[0] = s_\text{slot}[0] \in \{-1, +1\}$。

这意味着帧边界处存在一个**从 0 到 ±1 的突变**，与帧内的"静默→发送"边界（例如 SLOT-2 末尾→SLOT-3 起始）在性质上完全一致——它不是新现象，而是 P4 已分析的时隙门控效应在帧边界的又一次出现。

## 7. 帧边界连续性分析

### 7.1 为什么帧边界是 P5 必须单独验证的重点

P4 阶段的时隙边界分析（V3、F3）覆盖了**帧内**的发送→静默和静默→发送过渡，但**帧尾到下一帧帧头**的过渡从未被真正测试。在连续循环发射中，这个过渡每 8 ms 发生一次、5 分钟内发生约 37500 次——任何该点处的异常都会被频谱仪积累为"周期性毛刺"。

### 7.2 当前 `slot_config` 下的帧边界状态

根据 P4 锁定的当前工程假设（`P4_working_assumption_v1`）：

| 位置 | 时隙 | 发送/静默 | 样点值 |
| --- | --- | --- | --- |
| 帧尾：$n = 287999$（SLOT-32） | `i = 31` | 静默 | 0 |
| 帧头：$n = 0$（SLOT-1） | `i = 0` | 发送 | $s_\text{slot}[0]$（$\in \{-1, +1\}$） |

过渡性质：**从静默到发送**，幅度从 0 跳变到 ±1。

这与帧内的静默→发送过渡（例如 SLOT-2 末尾→SLOT-3 起始）完全同类。因此：

+ 帧边界不会引入 P4 未见过的"新型突变"；
+ 但帧边界突变的**周期性**为 $T_\text{frame} = 8\,\text{ms}$（频率 125 Hz），因此在频谱仪上可能以 125 Hz 间隔的离散谱线形式叠加在底噪上。

### 7.3 若帧边界拼接不当会出现什么

若 USRP 发射程序在帧与帧之间**存在间隙**（gap），即在第 $k$ 个周期末尾到第 $k+1$ 个周期开头之间插入了额外的零样点或不确定样点：

1. **帧周期失真**：实际重复周期变为 $N_\text{frame} + N_\text{gap}$ 点，不再等于 288000 / 8 ms，导致频谱谱线间距偏离 125 Hz；
2. **额外宽带毛刺**：间隙前后的突变引入两次额外的阶跃响应，频谱上表现为宽带能量抬升；
3. **可闻/可测的功率闪烁**：频谱仪的 VBW 积分窗口内功率出现周期性起伏，Max Hold 与 Average 之间差异明显增大。

若帧边界拼接正确但**缓冲区供数出现一次 underflow**：

1. USRP 将在 DAC 输出上插入一段不确定数据（通常为归零或重复上次缓冲区残余）；
2. 影响的时长取决于 underflow 恢复延迟（通常为微秒到毫秒级），可能跨越数个样点到数个时隙；
3. 在频谱仪上表现为瞬态谱线异常——若使用"Normal"检波模式可能难以捕获，建议同时使用"Max Hold"。

### 7.4 帧边界毛刺的来源诊断

若在 USRP 连续发射过程中观察到**周期性毛刺**，需按以下优先级排查：

| 优先级 | 可能来源 | 诊断方法 |
| --- | --- | --- |
| 1 | USRP 帧间存在间隙（实现错误） | 检查发送程序是否在循环内插入了额外间隔；用 UHD async metadata 检查 underflow 事件 |
| 2 | 缓冲区耗尽导致 underflow | 检查 UHD 日志 / async metadata 中的 underflow 计数 |
| 3 | 帧模板本身的帧尾-帧头数值突变（已知现象） | 回 MATLAB 检查 `s_frame[end]` 和 `s_frame[1]` 的数值 |
| 4 | 操作系统调度导致间歇性供数中断 | 降低主机其他负载后复测 |

### 7.5 MATLAB 侧先行检查

在上 USRP 之前，必须在 MATLAB 中完成帧边界拼接的**软件模拟**。具体验证内容见 §9 的 V1–V3。

## 8. IQ 循环缓冲区与 USRP 连续供数问题

### 8.1 为什么 P5 会出现 underflow 风险而 P4 不会

P4 的 USRP 验证是"短时重复播放"，典型时长为数秒，主机需持续供数的时间极短。在此条件下：

+ 操作系统的瞬态调度抖动（数十微秒级）不足以耗尽 USRP 内部 FIFO；
+ 主机内存中缓存的 IQ 数据完全可以覆盖短时播放需求；
+ 即使偶尔出现微秒级的供数延迟，频谱仪在"平均"模式下不一定能观测到。

P5 要求连续发射至少 5 分钟（300 秒），这彻底改变了工程条件：

+ 主机必须以 $36\,\text{MSPS} \times 4\,\text{B/sample} = 144\,\text{MB/s}$ 的恒定速率向 USRP 供数，持续 300 秒；
+ 在 300 秒内，操作系统的进程调度、磁盘 I/O 突发、USB 控制器中断聚合、内存页面回收等事件**必然发生**；
+ 任何导致供数速率瞬时低于 144 MB/s 的事件，若持续时间超过 USRP 内部 FIFO 的缓冲深度，即触发 underflow。

### 8.2 带宽需求计算

| 参数 | 数值 | 推导 |
| --- | --- | --- |
| 采样率 | 36 MSPS | 系统规划 |
| 每样点字节数（sc16 格式） | 4 B（I: int16 + Q: int16） | P0 锁定 |
| 持续数据传输带宽 | **144 MB/s** | $36 \times 10^6 \times 4$ |
| 5 分钟总数据量 | **43.2 GB** | $144 \times 300$ |
| 单周期帧数据量 | 1.152 MB | $288000 \times 4$ |
| 每秒帧数 | 125 | $1 / 8\,\text{ms}$ |

USB 3.0 理论峰值带宽约 625 MB/s，但实测可用带宽通常为 300～400 MB/s，原则上满足 144 MB/s 需求。但若主机有其他高带宽 USB 设备竞争、或操作系统 USB 调度抖动较大，有效带宽可能降到 144 MB/s 的临界区间。

> **工程建议**：P5 测试期间，确保 USRP 所在 USB 3.0 控制器上不挂载其他高带宽设备（如 USB 硬盘、摄像头等），并关闭主机上不必要的后台服务。

### 8.3 循环缓冲区设计

**为什么建议循环缓冲区至少 ≥ 2 个周期**：

+ 最小循环单元 = 1 个 `s_frame[n]`（288000 点 = 1.152 MB），但如果主机-USRP 链路只缓冲 1 帧，则缓冲区供尽到下一帧注入之间没有任何余量；
+ 2 个周期（576000 点 = 2.304 MB）允许"一帧正在被 USRP 消费的同时，主机可以补充下一帧"，实现双缓冲（ping-pong buffer）流水线；
+ 对于循环播放同一帧模板的场景，缓冲区中放置 2 份相同的 `s_frame[n]` 拼接体即可，USRP 循环读取时帧边界自动对齐。

**缓冲区大小建议**：

| 方案 | 缓冲区大小（点） | 缓冲区大小（MB） | 余量描述 |
| --- | --- | --- | --- |
| 最小（1 周期） | 288000 | 1.152 | 无余量，不推荐 |
| **推荐（2 周期）** | **576000** | **2.304** | 双缓冲，工程推荐最低值 |
| 安全（4 周期） | 1152000 | 4.608 | 4 倍余量，适用于 USB 抖动较大的平台 |

> **注**：若使用 UHD 的 `tx_streamer` API，实际缓冲深度由 `send_buffer_size`（或 `spp` × 帧数）决定。若使用 GNU Radio 的 `File Source (Repeat=True)` → `UHD Sink`，缓冲深度由 GNU Radio 的输出缓冲区决定。无论哪种方式，建议显式配置缓冲区深度并在记录表中写明。

### 8.4 underflow 的监控方法

**UHD C++ / Python API**：

```cpp
// C++ 伪代码
uhd::async_metadata_t async_md;
while (tx_stream->recv_async_msg(async_md, 0.0)) {
    if (async_md.event_code == uhd::async_metadata_t::EVENT_CODE_UNDERFLOW) {
        underflow_count++;
        log_timestamp(async_md.time_spec);
    }
}
```

**GNU Radio**：

+ UHD Sink 会在终端输出 `U` 字符表示 underflow 事件；
+ 建议将终端输出重定向到日志文件（`tee -a p5_uhd.log`），测试结束后统计 `U` 出现次数。

**MATLAB UHD 接口**（若使用 MATLAB Communications Toolbox / SDR 支持）：

+ 检查 `underflow` 属性或回调函数。

无论使用哪种方式，P5 必须在整个 5 分钟测试期间**持续监控并记录** underflow 事件，并在测试结束后输出：

+ underflow 总次数；
+ 每次 underflow 的时间戳（若可获取）。

### 8.5 主机负载与接口带宽对稳定性的影响

| 影响因素 | 影响方式 | 建议对策 |
| --- | --- | --- |
| CPU 占用率过高 | 数据供给线程被抢占，导致供数延迟 | P5 测试期间保持 CPU 空闲率 > 50%，关闭不必要进程 |
| 内存不足 / swap 频繁 | 数据缓冲区被页面回收，供数中断 | 确保物理内存充足（建议空余 ≥ 2 GB） |
| USB 控制器共享带宽 | 其他 USB 设备竞争带宽 | USRP 独占 USB 3.0 控制器 |
| 网络中断 / 磁盘 I/O 突发 | 间接占用 CPU / 总线带宽 | 断开不必要的网络连接，暂停磁盘备份任务 |
| Ubuntu 内核电源管理 | USB 自动挂起导致短暂断连 | 禁用 USB autosuspend（`echo -1 > /sys/module/usbcore/parameters/autosuspend`） |
| USRP 主时钟抖动 | 长时间运行后时钟漂移 | 记录发射前后频偏（若频谱仪可测量），作为基线数据 |

## 9. MATLAB 侧验证路径

**在将 `s_frame[n]` 加载到 USRP 进行长时间连续发射之前**，必须先在 MATLAB 中完成以下验证，确认帧边界拼接在数学上无异常。

### V1：载入 `s_frame[n]` 并构造多周期重复序列

```matlab
load('s_frame_template.mat', 's_frame');      % P4 锁定模板
N_frame = 288000;
assert(length(s_frame) == N_frame, 'P5 ERROR: s_frame length mismatch');

% 构造 K 个周期的重复序列（K >= 3 即可做边界分析）
K = 3;

assert(length(s_multi) == K * N_frame);
```

验收项：

+ `s_frame` 长度严格为 288000；
+ `s_multi` 长度严格为 `K × 288000`。

### V2：检查周期拼接点的样点连续性

```matlab
% 帧边界拼接点检查
for p = 1:(K-1)
    boundary_idx = p * N_frame;                % s_multi 中该周期末尾（MATLAB 1-based）
    val_tail = s_multi(boundary_idx);          % 第 p 周期最后一点
    val_head = s_multi(boundary_idx + 1);      % 第 p+1 周期第一点
    
    fprintf('Boundary %d→%d: tail = %.4f, head = %.4f, jump = %.4f\n', ...
            p, p+1, val_tail, val_head, val_head - val_tail);
end
```

验收项：

+ 在当前 `slot_config` 下，`val_tail` 应为 0（SLOT-32 静默），`val_head` 应为 `s_slot[1]`（MATLAB 1-based，SLOT-1 发送）——与 P4 帧内的静默→发送边界同质；
+ 跳变幅度 `|val_head - val_tail|` 应等于 `|s_slot[1]|`（通常为 1）；
+ 该跳变与帧内静默→发送边界处的跳变一致，不引入额外的异常。

### V3：绘制帧尾 + 下一帧帧头的局部时域图

```matlab
% 放大帧边界：取第 1→2 周期拼接点前后各 200 个样点
center = N_frame;                              % 拼接点位置
win    = 200;
idx    = (center - win + 1) : (center + win);  % 共 400 点

figure;
plot(t_us, s_multi(idx), 'b-', 'LineWidth', 0.8);

grid on;
```

验收项：

+ 帧尾段（SLOT-32 静默）应为持续的 0；
+ 帧头段（SLOT-1 发送）应与 `s_slot[n]` 起始段一致；
+ 两者之间存在一个阶跃跳变——该跳变的幅度和形态与帧内静默→发送边界（P4 F3 图）一致，不应有额外的异常过冲或振荡。

### V4：多周期 FFT 与时频观察

```matlab
% 对 3 周期重复序列做 FFT
Nfft    = 2^20;                                % 1048576，足以覆盖 3 * 288000 = 864000
S_multi = fftshift(fft(s_multi, Nfft));
f_axis  = (-Nfft/2 : Nfft/2-1) * (fs / Nfft);
P_dB    = 20 * log10(abs(S_multi) + eps);

figure;
plot(f_axis / 1e6, P_dB, 'b-');
xlabel('频率 (MHz)');
ylabel('幅度 (dB)');
title('P5-F2: 3 周期重复序列 FFT');
xlim([-18 18]);
grid on;
```

验收项：

+ BOC 双峰骨架仍在 ±15 MHz，形态与 P4 的 F7 一致；
+ 频谱细结构应呈现 $1/T_\text{frame} = 125\,\text{Hz}$ 间隔的谱线簇——对 3 周期窗口，FFT 分辨率为 $f_s / N_\text{fft} \approx 34.3\,\text{Hz}$，可部分分辨 125 Hz 谱线；
+ 不应出现 P4 未见的额外谱峰（如因帧间间隙引入的附加频率分量）。

### V5：帧边界毛刺能量分析

```matlab
% 提取所有帧边界前后各 Nw 点作为"边界区"
Nw = 50;                                       % 前后各 50 点
boundary_energy = [];
for p = 1:(K-1)
    center = p * N_frame;
    seg    = s_multi(center - Nw + 1 : center + Nw);
    boundary_energy(end+1) = sum(abs(seg).^2);
end

% 提取等量的帧中部区域作为对照
mid_energy = [];
for p = 1:K
    mid_start = (p-1) * N_frame + round(N_frame / 2) - Nw + 1;
    seg       = s_multi(mid_start : mid_start + 2*Nw - 1);
    mid_energy(end+1) = sum(abs(seg).^2);
end

fprintf('边界区平均能量: %.4f\n', mean(boundary_energy));
fprintf('帧中部平均能量: %.4f\n', mean(mid_energy));
```

验收项：

+ 帧边界区域的能量应与帧内同类过渡（静默→发送边界）的能量一致；
+ 不应出现异常的能量集中，否则提示帧模板末尾或首部有非预期的数值异常。

### V6：导出用于 USRP 长时间循环发射的文件模板

```matlab
% 导出 2 周期拼接的 IQ 文件（供 USRP 循环播放）
s_tx_buf = repmat(s_frame, 1, 2);              % 576000 点，双缓冲

% 转换为 sc16 格式（I + jQ，Q = 0）
I = int16(round(s_tx_buf * 32767));            % 归一化至 int16 满量程
Q = int16(zeros(size(I)));

% 交织为 [I0 Q0 I1 Q1 ...]
iq_interleaved = zeros(1, 2 * length(I), 'int16');
iq_interleaved(1:2:end) = I;
iq_interleaved(2:2:end) = Q;

% 写入二进制文件
fid = fopen('p5_tx_buffer_2frames.bin', 'w');
fwrite(fid, iq_interleaved, 'int16');
fclose(fid);

% 验证文件大小
file_info = dir('p5_tx_buffer_2frames.bin');
expected_bytes = 576000 * 4;                   % 2 周期 × 288000 点 × 4 B
assert(file_info.bytes == expected_bytes, 'P5 ERROR: IQ file size mismatch');

fprintf('IQ 文件导出完成: %d bytes (expected %d)\n', file_info.bytes, expected_bytes);
```

验收项：

+ 文件大小严格等于 $2 \times 288000 \times 4 = 2304000\,\text{bytes} = 2.304\,\text{MB}$；
+ 重新读回并与 `s_tx_buf` 对比，误差应为 0（int16 量化后精确一致）；
+ IQ 格式（字节序、交织方式）严格继承 P0 锁定方案。

### 10.5 重点观测现象

P5 频谱仪观测需重点关注以下现象：

| 观测项 | 预期现象 | 异常判据 |
| --- | --- | --- |
| **BOC 双峰稳定性** | 双峰中心始终在 ±15 MHz，11 次截图间无漂移 | 任何一次截图中双峰中心偏移 > 0.5 MHz |
| **功率稳定性** | 双峰峰值电平在 11 次截图间波动 ≤ 1 dB | 波动 > 2 dB 或出现单调趋势 |
| **功率闪烁** | RMS Average 模式下谱形平滑，无可见闪烁 | 肉眼可见的周期性或随机闪烁 |
| **频率漂移** | 双峰中心位置无随时间的单调偏移 | 峰值频率随时间单调移动 |
| **帧边界周期性毛刺** | Max Hold 与 RMS Average 差异不超过 3 dB | Max Hold 上出现远超 Average 的宽带尖峰 |
| **间歇性异常** | 无间歇性的频谱突变或消失 | 某次截图中双峰消失或形变 |
| **底噪抬升** | 底噪在测试期间稳定 | 底噪持续抬升 |

### 10.6 若频谱异常，故障树

| 现象 | 可能原因 | 定位方向 |
| --- | --- | --- |
| 双峰在某次截图中消失 | underflow 导致发射中断 | 检查 UHD 日志 underflow 计数 |
| 功率周期性闪烁 | 帧边界存在间隙 | 检查发射程序循环逻辑 |
| 双峰频率漂移 | USRP 参考时钟漂移 | 记录频谱仪中心频率读数时间序列 |
| Max Hold 大幅超出 Average | 间歇性 underflow 产生瞬态宽带能量 | 检查 UHD 日志对应时间戳 |
| 底噪持续抬升 | 长时间运行下硬件温度升高，噪声增加 | 监控 USRP 温度（若可获取） |
| 双峰位置与 P4 不一致 | IQ 文件加载错误 / 采样率配置错误 | 回到 P0 / V6 核对 |

## 12. 常见误判与风险

1. **将平均功率波动误判为算法错误**——频谱仪的 RMS Average 在不同截图间可能有 ±0.5 dB 的读数波动（仪器本身的测量不确定性），这不是信号问题。只有持续的单调趋势或突发性的 > 2 dB 跳变才需要排查。

2. **将频谱闪烁误判为 BOC 参数错误**——若使用"Normal"或"Peak"检波模式而非"RMS Average"，频谱仪上可能看到快速闪烁。这是帧结构（8 ms 周期性开关）在频谱仪扫描速率下的采样混叠效应，**不代表信号体制有误**。务必使用 RMS Average 模式做稳定性判定。

3. **忽略 underflow 日志**——有时 underflow 仅导致几个样点的异常，在频谱仪 RMS Average 模式下几乎不可见。但这些 underflow 会在接收端造成帧同步丢失。P5 必须以"0 次 underflow"为硬性通过标准，不能仅凭频谱仪"看上去正常"就放行。

4. **帧边界毛刺未做单独分析**——若仅观察频谱仪整体谱形，帧边界毛刺可能被淹没在底噪中。必须在 MATLAB 侧（V3/V5）先确认帧边界的数值连续性，再在频谱仪上用 Max Hold 模式对比 Average 模式来排查。

5. **在稳定性测试过程中同时改采样率 / 增益 / 中心频率**——这将导致问题无法定位。P5 的正式轮测试必须在**全部参数固定**的条件下完成。任何参数变更必须重新开始一轮独立的 5 分钟测试。

6. **将 P4 已记录的"占空比 -9 dB 下降"和"sinc 旁瓣"再次误判为硬件故障**——这些现象在 P4 已充分分析和记录，P5 不应重复排查。P5 只需验证这些现象在长时间发射中**保持稳定**即可。

7. **使用过小的循环缓冲区**——若仅用 1 个周期（288000 点）作为缓冲区，在 USB 带宽抖动时极易触发 underflow。务必使用 ≥ 2 个周期的缓冲区。

8. **在 P5 阶段重新生成 `s_frame[n]` 或修改 `slot_config`**——这将引入 P4 未经验证的新版本，违反"一次锁定，继承到底"的原则。P5 必须只 `load`，不重新生成。

9. **将 USRP 温度上升导致的轻微功率变化误判为信号体制问题**——长时间运行后 USRP 射频前端温度升高，输出功率可能有 ±0.5 dB 的漂移。这属于硬件热效应，记录即可，不需要回溯 P1–P4。

10. **在 P5 阶段引入接收端内容**——如"尝试用另一台 USRP 接收并做初步解调"，属于越界行为。P5 只做发端稳定性验证，接收端问题属于后续阶段。

## 16. 应锁定的参数与遗留待确认项

### 16.1 P5 结束后必须锁定的内容

以下内容在 P5 验收通过后**必须锁定**，P6 及之后阶段不得在未重新验证的情况下修改：

1. **连续循环发射可行性确认**：36 MSPS、288000 点/帧、8 ms 周期的帧模板可在 USRP 上连续循环发射至少 5 分钟且零 underflow——这一结论即为 P5 的核心锁定输出；
2. **帧边界连续性结论**：帧尾（SLOT-32 静默，值 = 0）到帧头（SLOT-1 发送，值 = `s_slot[1]`）的过渡与帧内同类过渡一致，无额外异常——P5 锁定；
3. **USRP 配置参数表**（§14.4）：采样率、中心频率、增益、缓冲区大小、发射程序类型——作为 P6 的基线配置，P6 若修改其中任何一项必须在独立轮次中重新测试稳定性；
4. **频谱长期稳定性基线**：BOC 双峰在 5 分钟测试中保持稳定（电平波动 ≤ 1 dB、频率无漂移）——P6 的参数扫描结果需以此为对照基准；
5. **循环缓冲区大小**：经 P5 验证可行的最小缓冲区大小（推荐 ≥ 2 周期）——后续阶段不应低于此值；
6. **IQ 文件模板**：`p5_tx_buffer_2frames.bin` 作为 P6 的默认发射文件基线；
7. **继承锁定**：P0–P4 所有已锁定内容在 P5 中保持不变——`s_frame[n]`（P4）、`s_slot[n]`（P3）、`sc[n]`（P1）、`c_up[n]`（P2）、IQ 格式（P0）。

### 16.2 仍然【待确认】但不阻碍 P5 → P6 推进的内容

1. **32 时隙三类（固定 / 保护 / 动态）的完整编号映射**——继承自 P4 遗留项，当前工程假设足以完成 P5 验证；

4. **USRP 最终中心频率 / 发射增益**——若 P5 使用的值与最终系统部署值不同，P6 中修改后需重做稳定性验证；
5. **bit 映射方向**——不影响 P5 稳定性验证结论；
