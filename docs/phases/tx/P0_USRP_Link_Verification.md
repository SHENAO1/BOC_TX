## 1. 文档目的与适用范围

### 1.1 文档目的

本文件仅服务于 **P0：USRP 基础链路打通** 阶段。其核心目标是：**在引入 BOC(15,3) 子载波、Gold 码扩频、TDMA 帧结构等任何信号体制复杂性之前，确认 USRP 发射硬件链路本身是连通的、可控的、可复现的。**

P0 产出的硬件基线配置和测试方法论，将作为后续所有阶段（P1–P6）的可复用基础。

### 1.2 适用范围

本文件覆盖以下工作：

- USRP 设备识别、主机通信链路确认
- 采样率 / 主时钟可行性验证
- 已知简单基带信号（单音 CW）的生成、导出、发射与频谱仪观测
- IQ 数据格式、字节序、归一化的端到端确认
- 硬件配置参数的锁定与记录

本文件 **不覆盖** 以下工作：

- BOC 子载波离散化分析及验证（属 P1）
- Gold 码生成、截短与扩频验证（属 P2）
- 单时隙 BOC 基带组合（属 P3）
- TDMA 帧结构拼接（属 P4）
- 连续循环发射稳定性（属 P5）
- 参数扫描与性能优化（属 P6）

### 1.3 信息来源标注约定

本文件中所有技术陈述均按以下三级标注：

- **【总规划已明确】**：直接引自总规划或系统规划文档中的明确条目。
- **【工程上合理推断】**：基于已知参数和通用工程实践推导，未在上位文档中显式写明。
- **【待确认】**：上位文档中明确标注为"待确认"的问题，或两份文档间存在表述差异的条目。

## 2. 本阶段在总工程中的位置

### 2.1 为什么 P0 必须先完成

【总规划已明确】总规划在 §4.0.1 中明确指出："在做任何信号体制验证之前，必须先确认'硬件链路本身是通的'。"其理由包括：

- 如果跳过此阶段，后续所有"信号不对"的问题都无法区分是算法问题还是硬件链路问题。
- IQ 字节序错误会导致后续所有信号频谱镜像翻转。
- 采样率配置错误会导致所有频率关系系统性偏移。
- DC offset 若未在简单信号阶段暴露，会干扰 P3 阶段 BOC 双峰结构的观测。
- 主机与 USRP 的带宽不足若不在此阶段发现，会在后续 36 MSPS 连续发射时频繁触发 underflow。

【总规划已明确】总规划在 §1.2 中明确了分阶段的核心思想："每一阶段只引入一个新的复杂性维度，其余维度保持已验证状态。"P0 引入的唯一新复杂性维度是 **USRP 硬件配置与 IQ 文件格式**。

### 2.2 P0 与 P1 / P2 的关系

【总规划已明确】总规划在 §1.3 中给出了阶段依赖关系：

```
Phase 0 ──► Phase 1（BOC 子载波验证）
Phase 0 ──► Phase 2（Gold 码扩频验证）
```

P1 与 P2 均以 P0 为前提，但彼此无依赖，可并行推进。原则为"前一阶段不通过，绝不进入下一阶段"。

【当前工程执行顺序补充】系统规划 §6 阶段 A 中定义的 `matlab/check_params.m`，在当前项目中正式落位为 **P0-A：参数级自洽检查**。它不是 P0 的硬件验收项，但属于**当前推荐开工顺序中的第一步**：先在纯 MATLAB 环境中确认参数、派生量和非整数采样风险的解释一致，再进入本文件覆盖的 USRP 硬件链路验证（可视作 P0-B）。

### 2.3 P0 完成后可作为后续阶段输入的结论

P0 完成后，应沉淀以下可复用结论供后续阶段直接使用：

- 已确认的 USRP 型号、接口类型、主时钟频率（为 P1–P6 的硬件环境提供基准）
- 已确认可工作的采样率 36 MSPS 配置方式（为 P1 的 BOC 离散化分析提供确定性输入）
- 已确认的 IQ 文件格式（sc16、字节序、交织方式、归一化系数），为后续所有阶段的 MATLAB→USRP 数据流提供模板
- 已确认的频谱仪观测方法论与正常基线（为后续阶段的频谱仪对比测试提供参考）
- 已确认的 DC offset 和镜像分量基线水平

## 3. 本阶段目标

以下目标直接继承自总规划 §2（阶段划分总表）和 §4.0（Phase 0 详细说明），并结合系统规划 §7.3（USRP 参数配置）做了工程化展开。

### 3.1 目标 G1：确认 USRP 能被主机识别并正常通信

【总规划已明确】总规划 §4.0.1 明确列出此项。

具体含义：主机执行 UHD 设备探测命令后，能返回正确的设备标识信息（型号、序列号、固件版本），且无通信超时或驱动错误。

### 3.2 目标 G2：确认 USRP 能以 36 MSPS 发射已知基带信号

【总规划已明确】总规划 §2 阶段划分总表中 P0 的核心目标为"确认 USRP 能以 36 MSPS 发射已知基带信号"。

具体含义：USRP 主时钟频率应能被配置为与 36 MSPS 兼容的值（如 36 MHz 的整数倍），且在该采样率下发射时不产生 underflow 告警（至少短时发射稳定）。

【工程上合理推断】总规划 §4.0.2 和系统规划 §7.3 均提到需确认"主时钟频率配置"和"接口带宽是否满足 36 MSPS × 4 bytes = 144 MB/s"。对于 USRP B210（系统规划 §7.3 提及其支持最高约 56 MSPS 单通道），36 MSPS 在硬件能力范围内，但主时钟与插值因子的具体组合需实测确认。

### 3.3 目标 G3：确认 IQ 文件格式、采样率、主时钟、中心频率、增益配置链路正确

【总规划已明确】总规划 §4.0.2 任务 3 要求"验证 I、Q 字节序、交织方式（先 I 后 Q）"以及"归一化系数（int16 满量程 = 32767）"。系统规划 §E-2 / E-3 详细给出了 sc16 格式导出的方法。

具体含义：从 MATLAB 生成 IQ 数据 → 导出为二进制文件 → USRP 加载发射 → 频谱仪观测的完整链路中，不存在格式转换错误。

### 3.4 目标 G4：确认频谱仪能够稳定观测到预期结果

【总规划已明确】总规划 §2 阶段划分总表中 P0 的完成判据为"频谱仪上看到预期单频峰，功率稳定"。

具体含义：频谱仪在合适的设置下（RBW、Span、参考电平），能清晰显示 CW 信号单峰，且功率读数在短时间内无明显波动。

## 5. 本阶段不做什么

以下事项明确排除出 P0 的工作范围。

### 5.1 不做 BOC 子载波验证

【总规划已明确】总规划 §4.0.3 明确写出"不要试图发射 BOC 信号"。BOC(15,3) 子载波的离散化分析（包括 `sign(0)` 过零点处理、非整数采样 $f_s/f_{sc}=2.4$ 的影响量化）属于 P1 的专属工作。

### 5.2 不做 Gold 码扩频验证

【总规划已明确】总规划 §4.0.3 明确写出"不要试图验证任何扩频参数"。Gold 码生成多项式、截短方式、自相关性能分析均属于 P2。

### 5.3 不做单时隙组合

单时隙 BOC 基带信号 $s[n] = b_m \cdot c[n] \cdot sc[n]$ 的生成与验证属于 P3。P0 仅发射简单已知信号。

### 5.4 不做 TDMA 整帧拼接

32 时隙帧结构（288000 点）的生成、时隙分配管理、占空比效应分析属于 P4。

### 5.5 不做长期连续发射优化

【总规划已明确】总规划 §7.3（风险详析）指出："Underflow 问题在 P0 阶段（发射 CW）不会暴露，因为 CW 的数据量极小。"P0 仅要求短时发射稳定（总规划 §4.0.6 给出的门槛为连续发射 2 分钟无 underflow），长时间（5 分钟以上）稳定性测试属于 P5。

### 5.6 不做多参数同时调整

【总规划已明确】总规划 §4.0.3 明确写出"不要同时调整多个 USRP 参数"。P0 中每次测试仅变更一个参数，以便在出现异常时可定位原因。

## 6. 建议的实施路径

以下路径按工程推荐顺序排列。每一步完成后，其结论作为下一步的输入前提。

### 步骤 6.0：参数级自洽检查（P0-A，推荐先做）

**做什么：** 运行 `matlab/check_params.m`，在不连接 USRP、不过频谱仪的情况下，先验证锁定参数和派生参数是否自洽。

**为什么：** 这一步能把“参数理解错误”和“硬件链路错误”提前分层。若 `Tc`、`Nc`、`Nslot`、`Tslot`、`Nframe`、`Tframe`、处理增益或数据率有任一项不对，后续硬件测试没有解释价值。

**预期现象：** 脚本输出中除 `fs/fsc = 2.4` 这一个预期警告外，其余检查项全部为 `√`，`FAIL = 0`，并明确写出“当前仍不触碰硬件”。

**若异常意味着什么：**

| 异常现象 | 可能原因 |
|---|---|
| `Nslot` 或 `Tslot` 不匹配 | 基础参数录入错误，或码片点数理解错误 |
| `Nframe` 或 `Tframe` 不匹配 | 时隙数、时隙时长或采样率关系理解错误 |
| 将 `fs/fsc = 2.4` 判成失败 | 把预期的非整数采样风险误判为参数错误 |

**建议记录项：** `PASS/WARN/FAIL` 计数、`fs/fsc` 的警告解释、是否允许进入后续 MATLAB/硬件阶段。

### 步骤 6.1：USRP 识别与基础信息确认

**做什么：** 在主机上运行 UHD 设备探测工具，确认 USRP 能被发现并报告正确的型号信息、固件版本、接口类型。

**为什么：** 这是所有后续操作的前提。若设备不可识别，任何信号发射都无从谈起。

**推荐输入信号：** 无。本步骤不涉及信号发射。

**预期现象：** 设备探测命令返回 USRP 的型号标识（如 B210）、序列号、连接接口（USB3 / GigE），无错误信息。

**若异常意味着什么：**

| 异常现象 | 可能原因 |
|---|---|
| 设备未找到 | USB 线缆故障、驱动未安装、固件未烧写 |
| 设备找到但报错 | UHD 版本与固件不匹配、权限不足 |
| 超时 | 接口带宽不足（如 USB2 口误当 USB3 用） |

**建议记录项：** USRP 型号、序列号、UHD 驱动版本、固件版本、接口类型。

### 步骤 6.2：采样率 / 主时钟可行性确认

**做什么：** 通过 UHD API 或命令行工具设置 USRP 的发射采样率为 36 MSPS，确认设备是否接受该配置，并记录实际生效的主时钟频率和插值因子。

**为什么：** 【总规划已明确】36 MSPS 是本系统的核心参数（系统规划 §3.3）。USRP 的采样率由主时钟除以插值因子得到，主时钟必须是采样率的整数倍，否则设备可能拒绝配置或自动舍入到最近的可用值。

**推荐输入信号：** 无。本步骤验证配置能力，不需要实际发射信号。

**预期现象：** 设备接受 36 MSPS 配置请求，返回的实际采样率精确等于 36,000,000 Hz（或在 Hz 级精度内一致）。同时记录主时钟频率（如 36 MHz 或其整数倍如 72 MHz、144 MHz）。

**若异常意味着什么：**

| 异常现象 | 可能原因 |
|---|---|
| 请求被拒绝 | 主时钟与 36 MSPS 不兼容（无法整除） |
| 实际采样率与请求值有偏差 | 设备自动舍入，需检查主时钟配置 |
| 配置成功但后续发射时 underflow | 接口带宽实际不足 |

**建议记录项：** 请求采样率、实际生效采样率、主时钟频率、插值因子。

### 步骤 6.3：已知单音基带生成与发射

**做什么：** 在 MATLAB 中生成一个纯单频余弦波的 IQ 数据，导出为 sc16 格式二进制文件，通过 USRP 发射，在频谱仪上观测。

**为什么：** 【总规划已明确】总规划 §4.0.2 任务 2 明确要求"发射已知 CW（连续波）信号"。单音信号是频谱上最简单、最易识别的测试信号：应在中心频率偏移 $f_0$ 处出现一个且仅一个功率峰。

**推荐输入信号：** 单频复指数信号 $x[n] = \cos(2\pi f_0 n / f_s) + j \sin(2\pi f_0 n / f_s)$，$f_0$ 建议取 1 MHz。详见第 7 章。

**预期现象：** 频谱仪在中心频率 + 1 MHz 处显示一个清晰、窄的功率峰；峰宽度由频谱仪 RBW 决定；峰两侧底噪平坦；无其他明显杂散峰。

**若异常意味着什么：**

| 异常现象 | 可能原因 |
|---|---|
| 无信号 | IQ 文件未正确加载、USRP 发射未启动、SMA 线缆未接、增益为零 |
| 峰在错误频率 | 采样率配置错误导致频率缩放、中心频率设置错误 |
| 出现两个对称峰 | IQ 数据仅有实部（缺少虚部），发出的是双边带信号而非单边带信号 |
| 峰旁有明显裙摆 | IQ 文件长度非信号周期整数倍，导致截断泄漏 |

**建议记录项：** 测试信号频率 $f_0$、频谱仪截图（标注中心频率、RBW、参考电平）、峰值频率读数、峰值功率读数。

### 步骤 6.4：IQ 数据格式检查

**做什么：** 在 MATLAB 中将导出的 IQ 二进制文件重新读回，逐样本与原始数据比对，确认无精度损失和格式错误。同时检查文件大小是否与预期一致。

**为什么：** 【总规划已明确】总规划 §4.0.2 任务 3 明确要求"验证 I、Q 字节序、交织方式（先 I 后 Q）"以及"归一化系数"。IQ 格式错误是 P0 中最常见也最隐蔽的问题来源。

**推荐输入信号：** 沿用步骤 6.3 的 CW 信号。

**预期现象：** 重新读回的 IQ 数据与原始 MATLAB 数据在 int16 量化精度范围内一致。文件大小符合计算值（样本数 × 2 通道 × 2 字节 / 样本）。

**若异常意味着什么：**

| 异常现象 | 可能原因 |
|---|---|
| 读回数据与原始不符 | 字节序错误（大端 / 小端混淆）、I/Q 交织顺序颠倒 |
| 文件大小不对 | 写入时数据类型错误（如写了 float32 而非 int16） |
| 数值溢出 | 归一化系数计算错误，超出 int16 范围 |

**建议记录项：** 导出文件名、文件大小（字节）、预期文件大小、读回误差最大值。

### 步骤 6.5：频谱仪观测与记录

**做什么：** 在 USRP 稳定发射步骤 6.3 的 CW 信号时，系统地记录频谱仪上的关键现象，建立正常基线。

**为什么：** 频谱仪观测结果将作为后续所有阶段的对比基线。P0 阶段发射的是最简单的信号，任何异常都可以归因于硬件链路而非信号体制。

**推荐输入信号：** 同步骤 6.3。

**预期现象：** 单一功率峰在正确频率位置；底噪平坦；无明显 DC 分量（零频偏处无突出的窄峰）；无镜像分量。详见第 10 章。

**若异常意味着什么：** 详见第 10 章频谱仪测试方案。

**建议记录项：** 完整的频谱仪参数设置（Center Freq、Span、RBW、VBW、Ref Level、衰减设置）、截图、峰值参数。

### 步骤 6.6：增益与中心频率基本联调

**做什么：** 在 CW 信号持续发射的同时，分别独立调整 USRP 发射增益和中心频率，观察频谱仪上的变化是否符合预期。

**为什么：** 【总规划已明确】总规划 §4.0.4 要求"信号功率是否随 USRP 发射增益设置线性变化"。增益联调可验证 USRP DAC/PA 链路是否正常工作；中心频率调整可验证 LO 锁定是否正确。

**推荐输入信号：** 同步骤 6.3。

**预期现象：**

- 增益调整：增益每增加 N dB，频谱仪峰值功率增加约 N dB（在线性工作区内）。
- 中心频率调整：频谱仪上信号峰随中心频率同步移动，峰偏移量 = 0（因为基带频率 $f_0$ 不变）。

**若异常意味着什么：**

| 异常现象 | 可能原因 |
|---|---|
| 增益变化但功率不变 | USRP 软件增益控制链路有问题，或频谱仪量程未更新 |
| 增益变化引起频率偏移 | 不应出现，若出现则硬件可能异常 |
| 中心频率变化后信号消失 | LO 未锁定、中心频率超出硬件范围 |
| 高增益时波形失真 | PA 压缩，进入非线性区 |

**建议记录项：** 增益设置序列及对应峰值功率、中心频率设置序列及对应观测频率。

### 步骤 6.7：稳定性短时验证

**做什么：** 在最终确定的配置下（采样率 36 MSPS、选定中心频率、选定增益），让 USRP 连续发射 CW 信号至少 2 分钟，监控 underflow 告警，并在此期间定期观察频谱仪。

**为什么：** 【总规划已明确】总规划 §4.0.6 给出了进入 P1/P2 的条件之一为"主机连续发射 2 分钟无 underflow 报告"。

**推荐输入信号：** 同步骤 6.3。

**预期现象：** 在 2 分钟内，UHD 驱动控制台无 underflow 告警输出；频谱仪上 CW 峰值功率和频率位置无可见漂移。

**若异常意味着什么：**

| 异常现象 | 可能原因 |
|---|---|
| 偶发 underflow | 主机负载过高、USB3 控制器性能不足、后台进程争抢带宽 |
| 频繁 underflow | 接口带宽根本不足，需换接口或降低采样率排查 |
| 功率漂移 | USRP 热稳定性问题、LO 漂移 |
| 频率漂移 | 时钟源精度问题 |

**建议记录项：** 发射持续时间、underflow 事件次数与时间戳、起止时刻频谱仪截图对比。

## 7. 建议采用的测试信号

### 7.1 为什么 P0 优先推荐单音信号

P0 的核心任务是验证硬件链路，而非验证信号体制。理想的 P0 测试信号应满足：

- **频域表征简单**：在频谱仪上只有一个峰，便于判断峰位置是否正确。
- **异常容易识别**：DC offset 表现为零频处额外的峰；镜像分量表现为对称位置的额外峰；IQ 格式错误表现为双边带。
- **与体制无关**：信号本身不包含任何 BOC、扩频或 TDMA 特征，使得一切异常均可归因于硬件链路。

单音（CW）信号完美满足上述所有条件。

### 7.2 推荐的单音频率选择

**建议 $f_0 = 1\,\text{MHz}$**。

选择理由（【工程上合理推断】）：

- **远离 DC**：1 MHz 距零频有足够距离，DC offset 产生的零频尖峰不会与信号峰混淆。
- **远离奈奎斯特边界**：1 MHz 远小于 $f_s/2 = 18\,\text{MHz}$，不会受到抗混叠滤波器滚降影响。
- **频谱仪分辨容易**：1 MHz 偏移在 ±10 MHz Span 设置下清晰可见。
- **与后续信号特征频率有区分度**：BOC 子载波基频为 15 MHz，扩频谱主瓣宽约 6 MHz，P0 的 1 MHz 不会与这些特征混淆。

【总规划已明确】总规划 §4.0.2 任务 2 也建议"选取 $f_0$ 为一个容易在频谱仪上识别的频率（例如 1 MHz）"。

### 7.3 可选的补充测试信号

若 P0 进度允许，可在 CW 验证通过后追加以下信号作为交叉验证。这些信号 **不是 P0 必须项**。

**双音信号（可选）：** 两个频率（如 $f_1 = 0.5\,\text{MHz}$，$f_2 = 1.5\,\text{MHz}$）叠加。可用于验证 USRP DAC 线性度——若两信号功率之和在频谱仪上的表现正确，说明 DAC 在该信号幅度下工作在线性区。

**简单双极性方波（可选）：** 周期为整数采样点数的方波（如周期 36 点 = 1 MHz 基频）。其频谱应包含基频及奇次谐波。可用于验证 USRP 对宽带信号的传输保真度。

### 7.4 为什么此阶段不适合直接发 BOC 信号

【总规划已明确】总规划 §4.0.3 明确禁止在 P0 发射 BOC 信号。

工程理由如下：

- BOC(15,3) 子载波频率 15 MHz 接近奈奎斯特频率 18 MHz（$f_{sc}/f_s = 15/36 = 0.833$），频谱仪上的表现受 USRP 内部滤波器影响大，不适合用于基础链路验证。
- BOC 信号具有双峰功率谱，正常情况下频谱仪就应显示双峰，难以区分"双峰是 BOC 导致的正常现象"和"双峰是 IQ 不平衡导致的镜像分量"。
- BOC 子载波的非整数采样问题（$f_s/f_{sc}=2.4$）本身是 P1 需要独立分析的技术风险，在硬件链路未确认的情况下引入此变量毫无益处。

## 8. MATLAB 侧在本阶段应完成的工作

P0 中 MATLAB 的角色是 **生成最简单的已知基带数据，并确保 MATLAB→IQ 文件这一数据流正确**。P0 不要求 MATLAB 做任何复杂通信体制仿真。

### 8.1 生成简单已知基带

MATLAB 需生成一个单频复指数信号的 IQ 数据。建议变量命名如下（仅为建议，非强制）：

- `fs_tx`：发射采样率，应设为 36e6
- `f_tone`：单音频率，建议 1e6
- `N_samples`：样本总数（建议为 $f_s / \gcd(f_s, f_0)$ 的整数倍，使得信号恰好包含整数个周期，避免截断泄漏；1 MHz 单音在 36 MSPS 下的最小整周期长度为 36 个样本）
- `t_vec`：时间向量，`(0:N_samples-1) / fs_tx`
- `iq_baseband`：复基带信号，`cos(2*pi*f_tone*t_vec) + 1j*sin(2*pi*f_tone*t_vec)`

### 8.2 检查点数、采样率、自洽性

MATLAB 脚本应包含以下检查逻辑（不要求写成可运行代码，但应在操作流程中执行）：

- 确认 `N_samples` 是信号周期的整数倍
- 确认 `length(iq_baseband) == N_samples`
- 确认 `max(abs(real(iq_baseband))) <= 1` 且 `max(abs(imag(iq_baseband))) <= 1`
- 确认信号实部和虚部的峰值相等（归一化正弦/余弦对）

### 8.3 验证导出 IQ 数据格式前的数值范围

在将浮点数据量化为 int16 之前，应确认：

- 归一化后信号最大绝对值 = 1.0（对应 int16 的 32767）
- 无 NaN 或 Inf 值
- 量化后的 int16 数据重新转回浮点后，与原始数据的绝对误差小于 $1/32768 \approx 3 \times 10^{-5}$

### 8.4 本阶段不做的事

- 不生成 Gold 码序列
- 不生成 BOC 子载波
- 不做自相关 / 互相关分析
- 不做 FFT 频谱对比（但可做简单 FFT 验证生成信号的频谱是否为单峰）
- 不把 `matlab/check_params.m` 当作硬件验收项；它属于推荐优先完成的 **P0-A 热身关卡**，但不替代本文件的 USRP 链路验收

## 10. 频谱仪测试方案

### 10.1 为什么本阶段频谱仪是必须的

【总规划已明确】总规划 §2 阶段划分总表中 P0 的"频谱仪"列为"✅ 必须"。§6.1 进一步说明 P0 阶段频谱仪的主要观测内容为"CW 单峰位置、DC offset、镜像"。

频谱仪在 P0 中的不可替代性在于：它是独立于 USRP 和 MATLAB 的第三方观测手段。仅靠 UHD 驱动报告"发射成功"不能证明射频端实际发出了正确信号。

### 10.2 推荐观测现象

在 USRP 发射 1 MHz 单音 CW 信号时，频谱仪上应逐项确认以下现象：

**10.2.1 单峰位置是否正确**

预期：在中心频率 + 1 MHz 处出现一个窄峰。偏差应小于频谱仪频率分辨率（与 RBW 相关）。

若峰在错误位置：采样率配置可能不正确，导致基带频率 $f_0$ 在射频上映射到了错误位置。

**10.2.2 功率是否稳定**

预期：峰值功率在观测期间（至少数十秒）波动 < 1 dB。

若功率不稳定：可能存在 underflow（间歇性数据中断）或 LO 锁定问题。

**10.2.3 是否有明显 DC offset**

预期：在中心频率处（零频偏位置）不应有突出的窄峰。若有，表示 USRP 模拟前端存在 DC 偏置泄漏。

【总规划已明确】总规划 §4.0.4 将"是否存在 DC 分量（零频处的尖峰，USRP 模拟前端 DC offset 问题）"列为 P0 必须观察的现象。

DC offset 的严重程度评估：若 DC 峰低于信号峰 30 dB 以上，对后续 BOC 双峰观测影响较小；若接近信号峰水平，需在 P0 阶段设法抑制（如 UHD DC offset 校正功能）。

**10.2.4 是否有镜像分量**

预期：在中心频率 − 1 MHz 处（与信号峰关于中心频率对称的位置）不应有明显的功率峰。

【总规划已明确】总规划 §4.0.4 将"是否有明显的镜像频率分量（IQ 不平衡）"列为 P0 必须观察的现象。

若出现镜像峰：表示 IQ 信号存在幅度不平衡或相位正交偏差。若镜像峰低于信号峰 30–35 dB 以上（对应 USRP B210 AD9361 的典型镜像抑制水平），属正常硬件性能。若镜像峰与信号峰幅度接近，则可能是 IQ 数据本身的问题（如 Q 分量全零时发射的是实信号，会产生等幅镜像——此为信号设计问题而非硬件问题）。

**10.2.5 增益变化是否能反映到频谱仪上**

预期：USRP 发射增益每变化 N dB，频谱仪上峰值功率同步变化约 N dB（在线性区内）。

### 10.3 推荐记录参数

每次频谱仪测试应记录以下信息：

- 频谱仪型号
- 中心频率（Center Frequency）
- 频率跨度（Span）
- 分辨率带宽（RBW）
- 视频带宽（VBW）
- 参考电平（Reference Level）
- 检波器类型（Detector：Peak / RMS / Sample）
- 输入衰减设置
- 信号峰值频率
- 信号峰值功率
- DC 分量相对信号峰的电平差
- 镜像分量相对信号峰的电平差
- 截图文件名及时间戳

### 10.4 推荐的频谱仪设置

以下设置建议适用于 1 MHz CW 单音在 36 MSPS 采样率下的 P0 测试。

| 参数 | 建议值 | 说明 |
|---|---|---|
| Center Frequency | 与 USRP 中心频率一致 | 使基带信号围绕屏幕中心显示 |
| Span | ±10 MHz（共 20 MHz）| 【总规划已明确】§6.1 建议 P0 Span 为"中心频率 ±10 MHz" |
| RBW | 100 kHz | 【总规划已明确】§6.1 建议 P0 RBW 为 100 kHz |
| VBW | ≥ RBW（如 100 kHz 或 Auto） | 【工程上合理推断】VBW ≥ RBW 可减少扫描时间 |
| Reference Level | 根据信号功率调整，使信号峰在屏幕上方约 1/3 处 | 【工程上合理推断】避免 ADC 过载同时保证动态范围 |
| Detector | Peak（初始观测）/ RMS（功率测量） | 【工程上合理推断】 |
| 衰减 | 根据 USRP 输出功率设置，确保频谱仪输入不超过最大允许功率 | 【工程上合理推断】直连时尤其重要 |

### 10.5 正常与异常现象判断表

| 观测项 | 正常 | 异常 |
|---|---|---|
| 信号峰位置 | 精确在 $f_c + f_0$ 处 | 偏移 > RBW |
| 信号峰宽度 | 约等于 RBW（窄峰） | 明显宽于 RBW |
| 信号峰功率稳定性 | 波动 < 1 dB | 波动 > 3 dB 或间歇消失 |
| DC 分量（中心频率处） | 低于信号峰 30 dB 以上 | 与信号峰幅度接近 |
| 镜像分量（$f_c - f_0$ 处） | 低于信号峰 30 dB 以上 | 与信号峰幅度接近 |
| 底噪形态 | 平坦 | 有周期性纹波或突起 |
| 杂散峰 | 无或远低于信号峰 | 有不可解释的额外峰 |

## 12. 常见风险与误判

### 12.1 IQ 字节序 / IQ 顺序错误

**表现：** 频谱仪上信号频谱呈镜像翻转，或完全不可辨认。

**根因：** MATLAB 导出时采用了与 USRP 期望不同的字节序（如 MATLAB 默认大端但 UHD sc16 期望小端），或 I/Q 分量交织顺序颠倒。

**排查方法：** 用十六进制编辑器查看 IQ 文件头部若干字节，与 MATLAB 中原始 int16 数值逐一比对。

**总规划依据：** 总规划 §4.0.5 明确指出"IQ 字节序错误 → 导致后续所有信号频谱镜像翻转"。

### 12.2 采样率设置错误

**表现：** 频谱仪上信号峰偏离预期位置，偏移量与采样率误差成比例。例如，若实际采样率为 32 MSPS 但脚本以 36 MSPS 计算，则 1 MHz 基带信号在射频上将出现在 $f_c + 1 \times (36/32) = f_c + 1.125\,\text{MHz}$。

**根因：** USRP 主时钟频率与采样率不匹配，设备自动舍入到了另一个采样率。

**排查方法：** 检查 UHD 初始化时的控制台输出，确认"Actual TX Rate"与请求值是否一致。

**总规划依据：** 总规划 §4.0.5 明确指出"采样率配置错误 → 导致所有频率关系系统性偏移"。

### 12.3 主时钟与采样率不匹配

**表现：** UHD 驱动报告的实际采样率与请求的 36 MSPS 不一致（如被舍入到 35.714... MSPS 或 36.363... MSPS）。

**根因：** USRP 的主时钟频率无法被 36 MSPS 整除。例如，B210 默认主时钟为某些值时，可能无法精确产生 36 MSPS。

**排查方法：** 显式设置主时钟频率为 36 MHz 的整数倍（如 36 MHz、72 MHz、144 MHz），然后重新配置采样率。

**系统规划依据：** 系统规划 §7.3 建议"主时钟频率需支持 36 MSPS（建议 144 MHz 或 36 MHz 时钟）"。

### 12.4 DC offset

**表现：** 频谱仪在精确的中心频率处（零频偏）显示一个与信号峰可比较的窄峰。

**根因：** USRP 零中频架构下，模拟前端的 I/Q 混频器存在直流泄漏（LO leakage）。

**排查方法：** 启用 UHD 的 DC offset 校正功能；或在基带数据中预补偿一个小的 DC 偏置；或更换为非零中频方案。

**总规划依据：** 总规划 §4.0.5 明确指出"DC offset 严重 → 干扰 BOC 双峰结构观测"。

### 12.5 镜像分量

**表现：** 频谱仪上在信号峰关于中心频率的对称位置出现一个额外峰，幅度可能低于主峰 20–40 dB。

**根因：** USRP I/Q 模拟前端的幅度不平衡和相位正交误差。

**排查方法：** 确认 IQ 数据本身是复信号（非纯实信号）；若数据正确，则残留镜像是硬件固有性能，记录电平差即可。

**注意区分：** 若 IQ 数据的 Q 分量全为零（纯实信号），则频谱仪上 **必然** 出现等幅镜像——这不是硬件问题，而是信号设计导致的双边带效应。P0 测试信号应使用复指数信号以避免此混淆。

### 12.6 USRP 与主机带宽问题

**表现：** 发射过程中频繁出现 underflow 告警（UHD 控制台输出 "U" 字符），频谱仪上观测到信号间歇性中断。

**根因：** USB3 / GigE 接口的实际可用带宽低于 36 MSPS × 4 bytes = 144 MB/s 的需求。

**排查方法：** 关闭主机上的其他高带宽应用；确认使用的是 USB3（而非 USB2）端口；测试降低采样率时是否消除 underflow。

**总规划依据：** 总规划 §4.0.5 明确指出"USRP 与主机带宽不足 → 后续 36 MSPS 连续发射时频繁 underflow"。

### 12.7 误把硬件链路问题当成信号体制问题

**表现：** 在跳过 P0 直接进入 P1 或更高阶段后，频谱仪上看到异常频谱（如不对称、额外峰、频率偏移），被误认为是 BOC 离散化问题、Gold 码问题或 TDMA 拼接问题，浪费大量时间排查算法。

**根因：** 硬件链路的基础性问题（IQ 格式错误、采样率不对、DC offset 严重）从未被隔离验证。

**排查方法：** 这正是 P0 存在的意义——用最简单的信号先排除硬件链路因素。

**总规划依据：** 总规划 §1.1（风险四）明确指出"若 USRP 基础配置存在问题，会被误认为是信号生成算法问题"。

### 13.2 关于 30 dB 门限的说明

【工程上合理推断】总规划原文使用的表述是"无明显 DC offset 和镜像分量"，未给出具体 dB 门限。本文件建议采用 30 dB 作为工程判据，理由如下：

- USRP B210（AD9361）的典型 LO suppression 约为 35–36 dBc，典型镜像抑制约为 30–40 dBc。30 dB 是一个合理的最低期望。
- 后续 P3 阶段需要在频谱仪上识别 BOC 双峰结构，DC/镜像分量若高于 −30 dBc 可能干扰双峰识别。

若 30 dB 门限无法满足，应在 P0 报告中明确记录实测值，并评估对后续阶段的影响。此门限为 **建议值，非总规划强制要求**，最终判断应结合实测情况由工程师决定。
