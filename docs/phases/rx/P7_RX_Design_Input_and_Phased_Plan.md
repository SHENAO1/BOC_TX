## 1. 文档目的与适用范围

### 1.1 文档目的

本文档是 P6 完成之后、接收端开发开始之前的**母文档 / 起点文档**。其核心定位是：

> **把 P0–P6 已经在发端侧锁定的全部参数、信号模型、帧结构、频谱基线与自相关基线，整理成接收端设计所需的确定性输入；并在此基础上，给出接收端自身的分阶段实施路线（PR0–PR7），作为后续每一份接收端阶段文档的上位规划。**

P7 不重新验证发端，不重新分析发端频谱，不重写 `sc[n]` / `c_up[n]` / `s_slot[n]` / `s_frame[n]` 的构造过程。P7 只做两件事：

1. **对上承接**：把 P6 输出的"发端参数规格书"翻译为接收端可直接引用的输入条件；
2. **对下规划**：为接收端后续若干阶段（捕获、跟踪、帧同步、判决、优化）建立与 P0–P6 同构的分阶段开发框架。

### 1.2 适用范围

| 适用对象 | 适用章节 |
|---|---|
| 接收端算法设计人员 | 第 3、4、5、6 章（参数继承、问题分解、阶段规划） |
| 接收端硬件 / USRP 接收链路实施人员 | 第 3、5、6、8 章 |
| 系统架构与方案评审人员 | 第 2、4、7、10 章 |
| 后续各接收端阶段文档（PR0–PR7）撰写者 | 全文作为上位依据 |

### 1.3 本文件不覆盖的内容

- 具体的捕获 / 跟踪 / 解调算法实现代码；
- 接收端射频前端（LNA、混频器、滤波器）硬件选型；
- 多用户 MAC 层解复用逻辑；
- 任何发端参数的再次更改——发端参数在 P6 完成时已锁定，本阶段及之后**不得自行改动**。

## 4. 接收端面临的核心问题分解

本章从工程角度分解接收端必须解决的核心技术问题。每一条都将映射到第 5 章的接收端阶段。

### 4.1 TDMA 时隙边界与周期同步问题

接收端面对的是一个长度为 288000 点、以 8 ms 为周期循环出现的帧信号，其中仅少数时隙（对应各载荷发送时隙）有能量，其余时隙静默。主要工程问题包括：

- **时隙边界对齐**：必须找到每个 250 μs（9000 点）时隙的起点；
- **周期对齐**：必须在时隙对齐基础上进一步找到 8 ms 周期的起点；
- **时隙类型识别**：根据 `slot_config`，识别当前时隙属于哪个载荷，或属于静默 / 保护时隙；
- **能量不连续性**：由于占空比较低（发送时隙约 4/32 = 12.5%），基于总能量的同步方法（如滑动能量窗）不适用，必须基于相关能量。

### 4.2 载波频偏 / 多普勒处理问题

接收端必定存在以下频率偏差：

- 发端 USRP 与接收端 USRP 的本振不严格同步，导致固定载波频偏；
- 若存在相对运动（如星地、机载），叠加多普勒频偏；
- 采样时钟偏差将同时引入频率刻度偏差与码片时间刻度漂移。

工程要求：

- 在捕获阶段必须做**载波频偏二维搜索**（频偏 × 码相位）；
- 跟踪阶段必须有载波环路（FLL / PLL）；
- 【待确认】频偏搜索范围上限（取决于应用场景，例如 ±kHz 级 vs ±十 kHz 级）。

### 4.3 带 BOC 子载波的扩频信号捕获问题

与常规 DSSS 捕获相比，本体制的捕获有两点差异：

- 本地参考信号必须是 `c_up[n] · sc[n]`（即包含 BOC 子载波的完整基带模板），而非仅 `c_up[n]`；
- 扩频增益虽为 28.75 dB，但 BOC 双峰结构会将相关函数从单峰变为 M 形（见 4.4）。

捕获搜索维度：

- 码相位（0 ~ 8999 点）
- 载波频偏（范围【待确认】）
- 时隙编号（1 ~ 32）或至少对应某一已知发送时隙

### 4.4 BOC 自相关 M 形带来的峰模糊问题

这是**本接收体制最核心的技术风险之一**。

P3 已确认：`s_slot[n]` 的自相关呈 M 形双峰结构，两峰间距约 1.2 采样点（33.3 ns）。工程后果：

- 相关峰不再是单一尖峰，捕获阶段容易**误锁到边峰而非主峰**，产生**半码片甚至更小尺度的固定偏差**；
- 跟踪阶段若使用传统早 / 晚门鉴别器（E–L 结构），鉴别曲线会出现**多个过零点**，环路可能锁定到错误的稳定点；
- 该问题在整个 GNSS BOC 文献体系中被称为**"BOC 跟踪模糊性"**，已有成熟对策（如 BPSK-like 处理、双边带独立相关、Bump-Jumping 等），但在本项目中【待确认】采用哪种对策。

### 4.5 跟踪环设计问题

一旦捕获完成，必须进入稳定跟踪状态。跟踪环需要同时维持：

- **码跟踪环（DLL）**：跟踪码相位，输出时隙边界；
- **载波跟踪环（FLL / PLL）**：跟踪剩余频偏与相位；
- **环路带宽、阶数、鉴别器形式**：均属于接收端设计选择；
- **BOC 条件下 DLL 鉴别器的选择**：见 4.4；
- 【待确认】环路更新周期（每时隙一次？每 bit 一次？每周期一次？）。

### 4.6 单 bit 判决与误码统计问题

系统规划与 `../../assets/BOC_TX.png` 已明确：每个载荷每个周期传输 1 bit，信息速率 125 bps。接收端的 bit 判决对应以下问题：

- **积分时长**：每 bit 的积分长度等于发送时隙长度（9000 点 = 250 μs），而不是一个周期；
- **判决方式**：同相累积后硬判；
- **bit 映射方向**：$b_m \in \{+1, -1\}$ 对应原始比特 $\{0, 1\}$ 的映射方向【待确认】；
- **误码统计**：需要已知发端 bit 序列作为真值，【待确认】是否使用固定测试序列（如全 1 / PN 序列）；
- **多载荷场景**：4 个载荷的 bit 在同一周期内通过不同时隙区分，接收端需按 `slot_config` 分别解出 4 路 bit。

### 4.7 非发送时隙策略的影响

P4 / P6 锁定了 `slot_config`，但：

- 非发送时隙当前为**零填充（静默）**；
- 若后续将非发送时隙改为其他载荷或噪声填充，接收端同步策略需要重新审视。

该项在本阶段标注为【待确认】项目级策略的稳定性。

## 5. 接收端推荐分阶段实施路线（PR0–PR7）

### 5.1 阶段划分原则

沿用总规划的同一原则：**每一阶段只引入一个新的复杂性维度，其余维度保持已验证状态**。

接收端从"简到繁"的复杂性维度依次为：

```
接收硬件链路与采样
    └── 本地参考（Gold 码 / BOC 模板）
         └── 单时隙捕获（已知时隙、已知频偏）
              └── 频偏 × 码相位联合搜索
                   └── 跟踪环（DLL + FLL/PLL）
                        └── 帧同步 / 时隙提取
                             └── bit 判决 / 误码统计
                                  └── 鲁棒性 / 参数优化
```

### 5.2 阶段划分总表

| 编号 | 阶段名称 | 核心目标 | 本阶段新增复杂性 | 是否建议上硬件 |
|---|---|---|---|---|
| **PR0** | 接收硬件链路与采样数据检查 | 确认 USRP 接收端能以 36 MSPS 正确采集发端信号 | USRP 接收硬件配置、采样数据读入与频谱一致性 | ✅ 必须 |

## 6. 每一阶段的详细说明

以下各小节的描述风格与 P0–P6 单阶段文档一致。每一阶段在后续仍需展开为独立 md 文档，本章仅作母文档级的规划。

### 6.1 PR0：接收硬件链路与采样数据检查

**核心目标：** 在接收 USRP 侧，能以 36 MSPS 稳定采集到发端的 BOC(15,3) 信号，并确认采集文件的 IQ 格式、采样率、频谱特征与发端一致。

**新增复杂性：** 接收 USRP 硬件配置、采集文件读入、基带频谱一致性比对。

**输入条件：**
- 发端（P5）可连续稳定循环发射；
- 发端参数规格书（P6）已锁定；
- 本文件 §3 参数总表。

**主要输出：**
- 接收 USRP 配置记录（采样率、中心频率、增益、IQ 格式）；
- 采集样本文件（至少 1 个完整周期 8 ms，建议 ≥ 10 个周期）；
- 接收端实测频谱图（与 P3 / P4 频谱基线对比）。

**验证方式：**
- MATLAB 读入采集文件，计算 FFT，对比 BOC 双峰结构；
- 与 P4 单周期帧频谱的形状、双峰间距、占空比下降量进行吻合度检查。

**是否建议上硬件：** ✅ 必须。

**完成判据：**
- 采集文件在 MATLAB 中可被正确读入（IQ 格式、字节序、归一化与发端一致）；
- 实测基带频谱呈 BOC(15,3) 双峰结构，双峰中心约 ±15 MHz，与 P3 / P4 基线定性一致；
- 至少连续采集 10 个周期（80 ms）无丢帧 / 无 overflow。

---

### 6.2 PR1：本地 Gold 码 / BOC 模板生成验证

**核心目标：** 在接收端独立复现一份与发端**比特级一致**的本地参考信号：`c_up[n]`、`sc[n]`、`s_slot[n]`。

**新增复杂性：** 严格复现 P1 / P2 / P3 锁定的生成规则（包括 `sign(0)` 处理、Gold 码截短规则、初相）。

**输入条件：**
- P1 / P2 / P3 / P6 已锁定的全部参数与实现细节；
- 发端留存的 `s_slot[n]` 参考文件（若有），作为逐点比对用。

**主要输出：**
- 接收端版本的 `c_up[n]`、`sc[n]`、`s_slot[n]`；
- 与发端对应序列的逐点一致性对比报告；
- 本地 `s_slot[n]` 自相关图（应与 P3 M 形基线一致）。

**验证方式：**
- 逐点比对（接收端本地 `s_slot[n]` 与发端保存的 `s_slot[n]`）；
- 自相关函数与 P3 基线对比。

**是否建议上硬件：** ❌ 纯 MATLAB。

**完成判据：**
- 本地 `s_slot[n]` 与发端 `s_slot[n]` 逐点完全一致（零误差）；
- 本地自相关 M 形结构与 P3 基线吻合；
- `sign(0)` 处理方案与 P1 / P6 锁定版本完全一致。

---

### 6.3 PR2：单时隙捕获验证

**核心目标：** 在**理想条件**（无频偏、已知当前时隙属于某已知载荷、SNR 充足）下，使用 PR1 生成的本地 `s_slot[n]` 对 PR0 采集的数据做相关捕获，找到正确的码相位。

**新增复杂性：** 滑动相关 / 频域快速相关实现，BOC 自相关 M 形在实测数据下的初次暴露。

**输入条件：**
- PR0 采集样本；
- PR1 本地 `s_slot[n]`。

**主要输出：**
- 捕获搜索输出（相关峰位置、峰值、次峰值）；
- 单时隙相关函数实测图（与 P3 M 形基线对比）；
- 主峰与边峰的幅度比记录。

**验证方式：**
- 捕获到的码相位与 `slot_config` 给出的时隙起点对齐；
- 相关函数主峰结构与 P3 自相关基线在形状上一致。

**是否建议上硬件：** ❌ 纯 MATLAB（基于 PR0 采集数据）。

**完成判据：**
- 能稳定检测到与已知时隙起点一致的相关主峰；
- 明确观察到 BOC M 形结构（不要求此阶段解决，但要求能**观测到并记录**峰模糊现象）。

---

### 6.4 PR3：频偏与码相位联合搜索验证

**核心目标：** 在人为注入或实际存在的载波频偏条件下，完成二维（频偏 × 码相位）捕获搜索，并**正面处理 BOC 峰模糊问题**。

**新增复杂性：**
- 频偏网格搜索；
- BOC 峰模糊对策方案的选择与验证（例如 BPSK-like、双边带独立相关等）【待确认】；
- 搜索计算量的工程可行性。

**输入条件：**
- PR0 采集样本（可在其上人为加频偏做受控实验）；
- PR1 本地模板；
- PR2 已验证的单时隙相关基础。

**主要输出：**
- 二维捕获结果（频偏 × 码相位相关矩阵）；
- 峰模糊对策前后的对比结果；
- 频偏搜索范围与步长的推荐值。

**验证方式：**
- 在注入已知频偏条件下，搜索结果应在正确频偏处达到最大峰；
- 主峰与最近边峰的幅度比在选定对策下满足【待确认】门限。

**是否建议上硬件：** ⚠️ 可选。建议先在 MATLAB 中通过人为加频偏的方式完成方法验证，再考虑在真实接收链路上（PR7 或 PR5 / PR6 阶段）验证。

**完成判据：**
- 在【待确认】的频偏范围内，二维搜索能稳定找到正确峰；
- BOC 峰模糊对策方案确定并锁定，后续 PR4 / PR5 沿用同一对策。

---

### 6.5 PR4：跟踪环验证

**核心目标：** 在 PR3 捕获完成后，建立 DLL（码跟踪）与 FLL/PLL（载波跟踪），维持稳定跟踪状态。

**新增复杂性：**
- 鉴别器设计（特别是 BOC 条件下 DLL 鉴别器的选择）；
- 环路滤波器阶数与带宽；
- 环路更新周期【待确认】。

**输入条件：**
- PR3 输出的初始码相位与频偏估计；
- PR1 本地模板。

**主要输出：**
- 跟踪环结构定义（鉴别器形式、环路阶数、带宽）；
- 跟踪过程中的码相位、频偏、相位误差随时间曲线；
- 环路锁定与失锁判据。

**验证方式：**
- 在至少若干个周期内，跟踪环输出的码相位误差、频偏误差收敛并稳定在零均值小方差范围内；
- 跟踪状态下相关峰值保持稳定。

**是否建议上硬件：** ⚠️ 先 MATLAB 仿真，再接入真实采集数据。

**完成判据：**
- 在 PR3 的捕获输出基础上，跟踪环能持续锁定至少 10 个周期（80 ms）；
- DLL 不发生 BOC 边峰跳锁（与 4.4 对策保持一致）。

---

### 6.6 PR5：TDMA 帧级同步与时隙提取

**核心目标：** 由"单时隙对齐"扩展到"32 时隙周期对齐"，识别帧内每一个时隙的编号，按 `slot_config` 将各载荷时隙分路输出。

**新增复杂性：**
- 帧头 / 周期起点识别（基于已知发送时隙位置或帧同步标志【待确认】）；
- `slot_config` 在接收端的完整映射；
- 多载荷（4 个载荷）时隙的并行分路。

**输入条件：**
- PR4 稳定跟踪输出；
- P4 / P6 锁定的 `slot_config`。

**主要输出：**
- 周期起点时间戳序列；
- 每载荷分路后的时隙采样数据（每周期 1 段，共 4 路）；
- 帧同步可靠性统计。

**验证方式：**
- 按 `slot_config` 分路出来的每一路时隙，其相关主峰应与 PR1 本地 `s_slot[n]` 对齐；
- 连续多个周期帧同步稳定，不发生周期滑动。

**是否建议上硬件：** ✅ 推荐。

**完成判据：**
- 至少连续 100 个周期（800 ms）内帧同步稳定；
- 4 路载荷时隙均能被正确分路。

---

### 6.7 PR6：bit 判决与误码验证

**核心目标：** 对 PR5 分路输出的每个载荷时隙，完成同相累积、bit 判决，并在已知发端 bit 序列条件下统计误码率。

**新增复杂性：**
- bit 映射方向确认【待确认】；
- 已知 bit 测试序列的选择【待确认】；
- 误码率统计与置信区间。

**输入条件：**
- PR5 分路输出；
- 发端 bit 序列真值（由项目方提供）【待确认】。

**主要输出：**
- 每载荷每周期的 bit 判决输出；
- 误码率统计表；
- bit 映射方向锁定结论。

**验证方式：**
- 在 SNR 充足条件下，误码率应接近 0；
- 随 SNR 下降，误码率变化趋势应符合理论扩频增益下的 BPSK 性能曲线（定性吻合即可）。

**是否建议上硬件：** ✅ 推荐。

**完成判据：**
- 在良好 SNR 条件下连续 1000 bit 无误码（或达到【待确认】的工程门限）；
- bit 映射方向被正式锁定并写入接收端参数规格书。

---

### 6.8 PR7：参数优化与鲁棒性测试

**核心目标：** 评估接收端性能对 SNR、频偏、时序抖动、采样时钟偏差等关键变量的敏感性，给出接收端参数的最终锁定与鲁棒性边界。

**新增复杂性：** 多维度参数扫描与决策，类似 P6 在发端的角色。

**输入条件：** PR0–PR6 全部完成。

**主要输出：**
- 接收端参数扫描报告；
- 接收端最终参数规格书；
- 鲁棒性工作边界（可用 SNR 下限、可用频偏范围等）。

**验证方式：** 与 P6 同构的参数扫描评价体系。

**是否建议上硬件：** ✅ 推荐。

**完成判据：**
- 关键参数全部锁定；
- 工作边界有明确量化结论。

## 7. 接收端的主要风险

| 风险编号 | 内容 | 风险等级 | 说明 |
|---|---|---|---|
| **R1** | BOC 自相关 M 形峰模糊导致误捕获 / 误跟踪 | 🔴🔴🔴 极高 | 见 §4.4，影响 PR3 / PR4 / PR5 全链条 |
| **R2** | Gold 码截短至 750 chips 导致自相关旁瓣上升 | 🔴🔴 高 | 发端已接受此旁瓣水平，接收端需要在捕获门限设置上相应放宽 |
| **R3** | 时隙静默导致能量不连续，基于能量的同步方法不可用 | 🔴🔴 高 | 必须强制基于相关而非能量做同步 |
| **R4** | 多载荷 / 多码分配策略尚未完全锁定 | 🟡 中 | 4 个载荷当前共用同一 Gold 码还是分别使用不同 Gold 码属于【待确认】，直接影响 PR5 / PR6 的分路逻辑 |
| **R5** | bit 映射方向（$b_m \in \{+1,-1\}$ ↔ $\{0,1\}$）尚未最终锁定 | 🟡 中 | 影响 PR6 误码统计的真值比对 |
| **R6** | 非发送时隙策略可能从"静默"变为其他填充 | 🟡 中 | 若后续变更，PR5 同步策略需要重审 |
| **R7** | 频偏搜索范围 / 跟踪环带宽等接收端工程参数尚无确定依据 | 🟡 中 | 需在 PR3 / PR4 根据实际应用场景定版 |
| **R8** | 接收 USRP 与发端 USRP 之间的时钟 / 本振差异 | 🟡 中 | PR0 阶段需明确是否共参考时钟，若无，必须在 PR3 / PR4 承担全部频偏与采样率偏差 |

## 8. MATLAB 与硬件接收的分工原则

| 阶段 | MATLAB 的角色 | 硬件（USRP 接收 + 采集文件）的角色 |
|---|---|---|

| PR1 | 全程；不上硬件 | — |
| PR2 | 基于 PR0 采集数据做相关 | 采集数据来源 |
| PR3 | 方法验证；可人为加频偏 | 真实频偏验证（可放到后期） |
| PR4 | 环路仿真 | 真实跟踪验证 |
| PR5 | 分路逻辑实现 | 长时间帧同步验证 |
| PR6 | bit 判决逻辑 | 真实误码统计 |
| PR7 | 参数扫描主体 | 硬件侧鲁棒性验证 |

## 11. 总结

P7 是发端工作（P0–P6）完成之后、接收端开发（PR0–PR7）正式启动之前的**母文档**。本文件完成了两项核心工作：

1. 将 P0–P6 已锁定的发端参数、信号模型（`sc[n]`、`c_up[n]`、`s_slot[n]`、`s_frame[n]`）、`slot_config`、频谱基线与自相关基线，整理为接收端的确定性输入（第 3 章）；
2. 在此基础上，将接收端工程任务按复杂性从低到高分解为 PR0–PR7 八个阶段，每一阶段均明确了核心目标、新增复杂性、输入、主要输出、验证方式、是否上硬件、完成判据（第 5、6 章），并建立了与发端 P0–P6 同构的阶段边界原则。

接收端的核心技术风险集中在 **BOC 自相关 M 形带来的峰模糊问题**（R1），这是 PR3 / PR4 必须正面解决的工程难题，也是本接收体制与常规 DSSS 接收的最大区别。其余遗留项（多载荷 Gold 码策略、bit 映射方向、频偏搜索范围、帧同步机制等）已在第 10 章以【待确认】标注，由后续各 PR 阶段文档或系统设计方依序闭环。
