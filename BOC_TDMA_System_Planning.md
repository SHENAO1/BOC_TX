## 2. 系统总体结构梳理

### 2.1 整体层次结构

【**图中明确给出**】系统采用 TDMA 帧结构，层次如下：

```
周期（帧）层
 └── 时隙层（32 个时隙 / 周期）
      └── 码片层（750 码片 / 时隙）
           └── 采样点层（12 点 / 码片）
```

### 2.2 周期结构

【**图中明确给出**】

- 1 个周期共 **32 个时隙**
- 每个时隙时长 **250 μs**
- 1 个周期总时长：$32 \times 250\,\mu\text{s} = 8\,\text{ms}$
- 每个载荷在 1 个周期内传输 **1 bit 信息**，数据率 = **125 bps**

### 2.3 时隙分类

【**图中明确给出**】图中将 32 个时隙分为三类：

| 时隙类型 | 图中标注位置 | 说明 |
|---|---|---|
| **固定时隙**（Fixed Slot） | SLOT-1 附近（粉色） | 特定载荷固定占用的发送时隙 |
| **保护时隙**（Guard Slot） | 中间区域（绿色） | 用于保护间隔，【待确认】是否静默 |
| **动态时隙**（Dynamic Slot） | SLOT-8 附近（黄色/灰色） | 动态分配，【待确认】分配规则 |

> ⚠️ **待确认**：固定时隙、保护时隙、动态时隙的具体编号分配（即哪些 SLOT 编号属于哪类）在图中仅示意性标注，未给出完整映射表。

### 2.4 多载荷发送机制

【**图中明确给出**】图中展示了 4 个载荷（载荷1～载荷4）分别在不同时隙发送：

- 载荷1 → SLOT-1（固定时隙）
- 载荷2 → SLOT-3（固定时隙）
- 载荷3 → SLOT-5（固定时隙）
- 载荷4 → SLOT-7（固定时隙）

> 【**根据图和通信原理推断**】：每个载荷各自占用自己的发送时隙，在该时隙内发送当前周期的 1 bit 数据扩频后的波形；在非本载荷时隙内，该载荷不发射信号（或静默）。

> 【**当前尚未明确**】：图中 4 个载荷共用同一组 Gold 码还是各自使用独立 Gold 码序列，对接收端区分载荷至关重要，需后续明确。

### 2.5 每周期 1 bit 的数据组织

【**图中明确给出**】

- 每个载荷在 1 个周期（8 ms）内发送 **1 bit 信息**
- 该 1 bit 信息对应 1 个时隙（250 μs），通过 **750 个码片**的扩频序列承载
- 图中顶部黄色栏明确标注：每个载荷在 1 个周期内传输 1 bit 信息

## 4. 单个发送时隙内的信号结构

### 4.1 连续时间信号模型

#### 4.1.1 符号映射

【**根据图和通信原理推断**】，1 bit 信息 $b \in \{0, 1\}$ 经 BPSK 映射后：

$$b_m = 2b - 1 \in \{-1, +1\}$$

> 【**待确认**】：bit 到符号的映射方向（即 0→+1 还是 0→-1）尚未明确，需与系统设计方确认。

#### 4.1.2 扩频码序列

【**图中明确给出**】使用 10 级 Gold 码，截短为 750 个码片：

$$\mathbf{c} = [c_0, c_1, \ldots, c_{749}], \quad c_k \in \{-1, +1\}$$

> 【**待确认**】：Gold 码的初始状态（两个 m 序列的初相）尚未明确，不同载荷是否使用不同初相也未明确。

#### 4.1.3 扩频信号（调制前）

单个码片的扩频基带信号（矩形成形）：

$$s_\text{spread}(t) = A \cdot b_m \cdot \sum_{k=0}^{749} c_k \cdot p\!\left(t - k T_c\right), \quad 0 \leq t < T_\text{slot}$$

其中：
- $A$：发射幅度（归一化时 $A=1$）
- $T_c = 1/3\,\text{MHz} \approx 333.3\,\text{ns}$：单个码片时长
- $T_\text{slot} = 250\,\mu\text{s}$：时隙时长
- $p(t)$：矩形脉冲，$p(t) = 1,\; 0 \leq t < T_c$，否则为 0

#### 4.1.4 BOC 子载波

【**图中明确给出**】BOC 子载波定义为：

$$sc(t) = \text{sign}\!\left(\sin\!\left(2\pi \cdot 15\,\text{MHz} \cdot t\right)\right)$$

其中 $\text{sign}(\cdot)$ 为符号函数，$sc(t) \in \{-1, +1\}$，为方波。

#### 4.1.5 单时隙 BOC 基带信号（连续时间）

$$\boxed{s_\text{BOC}(t) = A \cdot b_m \cdot \sum_{k=0}^{749} c_k \cdot p(t - kT_c) \cdot sc(t), \quad 0 \leq t < T_\text{slot}}$$

**各变量物理含义：**

| 符号 | 含义 | 取值范围 |
|---|---|---|
| $A$ | 发射幅度 | 归一化为 1 |
| $b_m$ | 当前周期数据 bit 映射后的符号 | $\{-1, +1\}$ |
| $c_k$ | 第 $k$ 个码片值 | $\{-1, +1\}$ |
| $p(t - kT_c)$ | 第 $k$ 码片的矩形时域窗 | $\{0, 1\}$ |
| $sc(t)$ | BOC 方波子载波 | $\{-1, +1\}$ |
| $s_\text{BOC}(t)$ | 单时隙基带发送信号 | $\{-A, +A\}$ |

### 4.2 离散时间信号模型

#### 4.2.1 采样时刻

$$t_n = \frac{n}{f_s}, \quad n = 0, 1, \ldots, N_\text{slot}-1, \quad f_s = 36\,\text{MSPS}, \quad N_\text{slot} = 9000$$

#### 4.2.2 扩频码上采样序列

第 $n$ 个采样点对应的码片索引：

$$k(n) = \left\lfloor \frac{n}{N_c} \right\rfloor, \quad N_c = 12\,\text{（每码片采样点数）}$$

扩频码上采样序列：

$$c[n] = c_{k(n)}, \quad n = 0, 1, \ldots, 8999$$

即每个码片值重复 12 次后得到长度为 9000 的序列。

#### 4.2.3 BOC 子载波离散序列

$$sc[n] = \text{sign}\!\left(\sin\!\left(2\pi \cdot \frac{15\,\text{MHz}}{36\,\text{MSPS}} \cdot n\right)\right) = \text{sign}\!\left(\sin\!\left(\frac{5\pi}{6} n\right)\right)$$

> ⚠️ **重要注意**：$15/36 = 5/12$，相位增量为 $5\pi/6$ rad/sample，**子载波并非整数倍关系**，方波在采样点上的符号取值不构成严格对称的整数周期序列。详细分析见第 8.1 节。

#### 4.2.4 单时隙 BOC 基带离散信号

$$\boxed{s[n] = A \cdot b_m \cdot c[n] \cdot sc[n], \quad n = 0, 1, \ldots, 8999}$$

这是最终在 MATLAB 中生成和在 USRP 中发射的**基带实信号序列**。

#### 4.2.5 伪代码示意（MATLAB 风格）

```matlab
% 参数定义
fs      = 36e6;          % 采样率
Tc      = 1/3e6;         % 码片时长
Nc      = 12;            % 每码片采样点数
N_slot  = 9000;          % 每时隙采样点数
f_sc    = 15e6;          % BOC子载波频率

% 时间向量
t = (0:N_slot-1) / fs;   % 长度 9000

% 扩频码上采样（gold_code 为 750×1 的 {-1,+1} 向量）
c_upsampled = repelem(gold_code, Nc);  % 长度 9000

% BOC子载波
sc = sign(sin(2*pi * f_sc * t));       % 长度 9000

% 单时隙基带信号
s_slot = A * b_m * c_upsampled .* sc; % 长度 9000
```

## 5. 时隙拼接后的完整发端波形结构

### 5.1 单周期波形构成

【**根据图和通信原理推断**】完整的单周期基带波形由 32 个时隙拼接而成，总长度为：

$$N_\text{frame} = 32 \times 9000 = 288000\,\text{points}$$

单周期波形向量 $\mathbf{S}_\text{frame}$ 构造如下：

$$S_\text{frame}[n + i \cdot N_\text{slot}] = s_i[n], \quad i = 0, 1, \ldots, 31, \quad n = 0, 1, \ldots, 8999$$

其中 $s_i[n]$ 为第 $i$ 个时隙的基带信号，定义为：

$$s_i[n] = \begin{cases} A \cdot b_m^{(\text{payload}_j)} \cdot c^{(\text{payload}_j)}[n] \cdot sc[n] & \text{若第 } i \text{ 时隙为载荷 } j \text{ 的发送时隙} \\ 0 & \text{若第 } i \text{ 时隙非发送时隙} \end{cases}$$

### 5.2 非发送时隙的处理

【**待确认**】图中未明确说明非发送时隙（保护时隙、动态时隙中未分配的时隙）是否必须发送全零。

**可能的处理方式：**

| 方式 | 描述 | 影响 |
|---|---|---|
| 全零静默 | 非发送时隙输出 0 | 降低平均发射功率；USRP 需正确处理突发发射 |
| 继续发载波（无数据） | 仅发子载波无扩频 | 不推荐，会引入干扰 |
| 发送已知参考序列 | 如导频 | 【图中无此说明，不推荐假设】 |

**建议**：在参数确认前，MATLAB 仿真中先按**全零静默**处理非发送时隙。

### 5.3 多载荷时隙排布

【**图中明确给出**】图中示意了 4 个载荷分别在 SLOT-1、SLOT-3、SLOT-5、SLOT-7 发送（均为固定时隙区域）。

【**待确认**】：
- 4 个载荷是否共享同一个 USRP 平台发射（即单一发射机需在不同时隙发不同载荷信号）？
- 还是每个载荷对应独立的发射机？

**若单一 USRP 同时承载 4 个载荷，则：**

$$S_\text{frame}[n + 0 \cdot 9000] = s_{\text{slot-1}}[n] = A \cdot b_m^{(1)} \cdot c^{(1)}[n] \cdot sc[n]$$
$$S_\text{frame}[n + 2 \cdot 9000] = s_{\text{slot-3}}[n] = A \cdot b_m^{(2)} \cdot c^{(2)}[n] \cdot sc[n]$$
$$\vdots$$

### 5.4 周期波形循环发射

【**根据通信原理推断**】发端持续循环发送，每 8 ms 更新数据 bit 并生成下一个周期波形：

$$S_\text{TX}[n + k \cdot N_\text{frame}] = S_\text{frame}^{(k)}[n], \quad k = 0, 1, 2, \ldots$$

其中上标 $(k)$ 表示第 $k$ 个周期，不同周期中 $b_m$ 可能不同。

## 6. MATLAB 仿真规划

### 阶段 A：参数级自洽性验证

**目标：** 在写任何信号生成代码之前，用数值计算验证所有参数的内在一致性，排除设计错误。

**输入：** 第 3 章参数汇总表。

**输出：** 一个 MATLAB 脚本 `check_params.m`，打印所有派生参数并与预期值比对。

**关键检查项：**

| 检查项 | 计算 | 预期结果 |
|---|---|---|
| 码片时长 | $T_c = 1/f_c = 1/3\,\text{MHz}$ | $333.3\,\text{ns}$ |
| 每码片点数 | $N_c = f_s / f_c = 36/3$ | $12$（整数 ✓） |
| 每时隙点数 | $N_\text{slot} = N_\text{chips} \times N_c = 750 \times 12$ | $9000$ ✓ |
| 时隙时长验证 | $N_\text{slot} / f_s = 9000 / 36\,\text{MSPS}$ | $250\,\mu\text{s}$ ✓ |
| 周期时长 | $32 \times 250\,\mu\text{s}$ | $8\,\text{ms}$ ✓ |
| 数据率 | $1\,\text{bit} / 8\,\text{ms}$ | $125\,\text{bps}$ ✓ |

| BOC 子载波每时隙周期数 | $f_{sc} \times T_\text{slot} = 15\,\text{MHz} \times 250\,\mu\text{s}$ | $3750$（整数 ✓） |
| 处理增益 | $10\log_{10}(750)$ | $28.75\,\text{dB}$ ✓ |

**可能踩坑：**
- 注意 MATLAB 中浮点时间向量的精度：`t = (0:N-1)/fs` 与 `t = 0:1/fs:(N-1)/fs` 结果相同，但后者可能有末端精度问题，统一使用前者。

### 阶段 B：单时隙基带信号生成

**目标：** 生成一个完整的单时隙 BOC(15,3) 基带信号，长度 9000 点。

**输入：** 1 bit 数据（取 $b=1$ 和 $b=0$ 各测一次），Gold 码参数。

**输出：** `s_slot`（9000×1 实数向量），对应连续信号 $s[n]$。

**步骤：**

**B-1：bit 生成与符号映射**
```matlab
b = 1;            % 测试 bit
b_m = 2*b - 1;   % BPSK 映射：0→-1, 1→+1  （待确认映射方向）
```

**B-2：Gold 码生成与截短**
```matlab
% 使用 comm.GoldSequence 或手动构造 10 级 Gold 码
% 生成长度 1023 的序列，取前 750 个码片
gold_full = generate_gold_code(init1, init2);  % 返回 1023 个 {0,1}
gold_750  = gold_full(1:750);
gold_bpsk = 2*gold_750 - 1;  % 映射到 {-1,+1}
```
> 【**待确认**】`init1`, `init2`（两个移位寄存器初始值）尚未确定。

**B-3：码片序列上采样**
```matlab
Nc = 12;
c_up = repelem(gold_bpsk, Nc);  % 9000×1
```

**B-4：BOC 子载波生成**
```matlab
fs   = 36e6;
f_sc = 15e6;
t    = (0:8999) / fs;
sc   = sign(sin(2*pi * f_sc * t));  % 9000×1，{-1,+1}
```

**B-5：相乘得到单时隙信号**
```matlab
A      = 1;  % 归一化幅度
s_slot = A * b_m * c_up .* sc;  % 9000×1
```

**关键检查点：**
- `s_slot` 的取值应严格在 $\{-1, +1\}$ 中（实数双极性信号）。
- `mean(s_slot)` 应接近 0（无 DC 分量）。
- `var(s_slot)` 应为 1（单位方差，归一化）。

**可能踩坑：**
- `sign(0)` 返回 0 而非 ±1。由于 $5\pi/6 \cdot n$ 在某些 $n$ 处恰好等于 $k\pi$，`sin` 值为 0，`sign` 返回 0，导致信号出现"空洞"。处理方法见第 8.1 节。

### 阶段 C：单周期帧信号生成

**目标：** 将单个时隙信号按 TDMA 时隙结构拼接为完整的一个周期波形，长度 288000 点。

**输入：** 各载荷的 `s_slot`，时隙分配表。

**输出：** `s_frame`（288000×1 实数向量）。

**步骤：**

**C-1：初始化帧缓冲区**
```matlab
N_frame  = 32 * 9000;  % 288000
s_frame  = zeros(N_frame, 1);
```

**C-2：填充各发送时隙**
```matlab
% 示例：载荷1 占用 SLOT-1（索引从0起）
slot_idx = 0;  % SLOT-1 对应索引 0
s_frame(slot_idx*9000+1 : (slot_idx+1)*9000) = s_slot_payload1;

% 载荷2 占用 SLOT-3
slot_idx = 2;
s_frame(slot_idx*9000+1 : (slot_idx+1)*9000) = s_slot_payload2;
% ... 依此类推
```

**C-3：非发送时隙保持为零**（已由初始化保证）

**C-4：多周期重复**
```matlab
N_periods = 10;  % 仿真 10 个周期
s_tx = repmat(s_frame, N_periods, 1);  % 静态数据时可复制
% 注意：若每周期 bit 不同，需逐周期生成
```

**关键检查点：**
- 检查各时隙边界处信号的不连续性（见第 8.4 节）。
- 统计非零时隙比例是否与预期一致（4/32 = 12.5%）。

**可能踩坑：**
- MATLAB 索引从 1 开始，注意时隙边界的偏移计算。
- 多周期数据 bit 变化时，不能简单 `repmat`，需重新生成。

### 阶段 D：频谱与时域特性验证

**目标：** 可视化验证信号的时频域特性是否符合 BOC(15,3) 理论预期。

**输入：** `s_frame`（或 `s_slot`）。

**输出：** 一套验证图组（建议保存为 PDF 或 PNG）。

**D-1：时域波形图**
- 绘制单时隙 9000 点的时域波形
- 横轴：时间（μs），纵轴：幅度
- 预期：双极性 ±1 随机方波

**D-2：扩频码时域图**
- 绘制 `c_up` 的前 3 个码片（36 点），验证码片形状为矩形

**D-3：BOC 子载波时域图**
- 绘制 `sc` 的前 200 ns（约 7 点），观察方波形状
- ⚠️ 由于采样非整数倍，应能看到**近似方波**而非理想方波

**D-4：调制后基带时域图**
- 对比扩频码与 BOC 基带信号，验证乘法关系

**D-5：FFT 频谱图**
- 对单时隙信号做 FFT，与 BOC(15,3) 理论功率谱比对
- 理论 BOC(15,3) 功率谱有两个主瓣，中心分别在 $\pm f_{sc} = \pm 15\,\text{MHz}$（相对基带），间距与码片速率有关
- 验证谱零点、主瓣位置

**D-6：自相关分析**
- 计算 `xcorr(s_slot, s_slot)`
- BOC(15,3) 自相关函数应呈现"M 形"双峰特征（主峰 + 两侧**负副峰/valleys**；不同于标准 BPSK-DSSS 的单峰三角形）
- 相邻主副极值间距约为 $T_{sc}/2 = 1/(2 f_{sc}) = 33.3\,\text{ns}$，对应 $\approx 1.2$ 个采样点（注：`1/f_sc = 66.7 ns` 是子载波周期，非 M 形副峰间距）

**关键检查点：**
- FFT 主瓣位置是否在 $\pm 15\,\text{MHz}$ 附近？
- 自相关旁瓣电平是否合理（扩频增益约 28.75 dB）？
- 非发送时隙是否静默（对应频谱是否在发送时隙外无功率）？

**可能踩坑：**
- `fftshift` + 正确的频率轴计算：`f = (-N/2:N/2-1) * (fs/N)`
- 对短时序列做 FFT 时需考虑频率分辨率（$\Delta f = f_s / N = 36\,\text{MHz}/9000 = 4\,\text{kHz}$）

### 阶段 E：为 USRP 发射做准备

**目标：** 将 MATLAB 生成的实数基带信号转化为 USRP 可接受的复数 IQ 格式，验证格式正确性。

**输入：** `s_frame`（实数基带，288000×1）。

**输出：** `iq_frame`（复数基带，288000×1，或实部/虚部分离文件）。

**E-1：复数基带转换**

【**当前方案**】：本信号体制为**零中频实基带**信号（非复调制，仅有 I 分量），直接令：

$$\text{I}[n] = s[n], \quad \text{Q}[n] = 0$$

因此 IQ 向量：
```matlab
iq_frame = complex(s_frame, zeros(size(s_frame)));
% 或直接：iq_frame = s_frame + 1j*0;
```

> 【**待确认**】：是否需要上变频到某个中频后再作为 USRP 基带输入？若需要，则 Q 分量不为零，需在本步骤完成数字上变频。

**E-2：幅度归一化**
```matlab
iq_norm = iq_frame / max(abs(iq_frame));  % 归一化到 [-1, 1]
% USRP 通常要求 int16 格式，缩放因子：
iq_int16 = int16(iq_norm * 32767);
```

**E-3：IQ 文件导出（离线发射模式）**
```matlab
% 以二进制交织方式导出 I/Q
fid = fopen('boc_tdma_frame.bin', 'wb');
iq_interleaved = [real(iq_int16); imag(iq_int16)];
iq_interleaved = iq_interleaved(:)';
fwrite(fid, iq_interleaved, 'int16');
fclose(fid);
```

**关键检查点：**
- 导出文件大小：$288000 \times 2 \times 2\,\text{bytes} = 1.152\,\text{MB}$（单周期）。
- 在 MATLAB 中重新读回并与原始 `s_frame` 比对，确认无精度损失。

**可能踩坑：**
- USRP UHD 的 IQ 格式通常为 `sc16`（复数 16-bit 整型），需确认字节序（通常小端）。
- 归一化后的峰均比（PAPR）：本信号为恒包络（BPSK），PAPR = 0 dB，不需要预失真。

## 7. USRP 发端信号源设计规划

### 7.1 总体架构

推荐采用**离线生成 + 在线循环播放**的架构：

```
[MATLAB 离线生成] ──── IQ 二进制文件 ────► [USRP 离线发射模式]
                                                  │
                        ┌─────────────────────────┘
                        │
                   [UHD tx_waveforms 或
                    自定义 UHD 程序循环播放]
```

**理由：**

| 对比维度 | 实时生成 | 离线生成 |
|---|---|---|
| 开发难度 | 高（需实时 Gold 码 + BOC 生成） | 低（MATLAB 预计算） |
| 时序稳定性 | 受计算负载影响 | 高（纯内存拷贝） |
| 数据更新灵活性 | 高 | 低（需重新生成文件） |
| 适合当前阶段 | ✗ | ✓ |

**结论：当前阶段推荐离线生成，待体制验证完成后再考虑实时生成。**

### 7.2 发端链路模块划分

```
┌──────────────────────────────────────────────────────────┐
│                  USRP 发端链路                            │
│                                                          │
│  [IQ 文件]                                               │
│      │                                                   │
│      ▼                                                   │
│  [缓冲区加载]  ──── 将一个完整周期（288000点）加载入内存   │
│      │                                                   │
│      ▼                                                   │
│  [循环发送控制]  ─── 帧边界对齐，周期循环                  │
│      │                                                   │
│      ▼                                                   │
│  [USRP 驱动层]  ─── UHD stream → USRP 硬件               │
│      │                                                   │
│      ▼                                                   │
│  [DAC / 数字上变频]  ─── USRP 内部完成                    │
│      │                                                   │
│      ▼                                                   │
│  [射频前端]  ─── 中心频率、增益（后续配置项）               │
└──────────────────────────────────────────────────────────┘
```

### 7.3 关键 USRP 参数配置

以下参数需在实施时确认，**本文档不假设具体数值**：

| 参数 | 当前状态 | 说明 |
|---|---|---|
| 采样率（tx_rate） | **已确定：36 MSPS** | 与仿真一致 |
| 中心频率（center_freq） | 【**待确认**】 | 取决于射频规划 |
| 发射增益（tx_gain） | 【**待确认**】 | 需功率预算后确定 |
| 天线端口 | 【**待确认**】 | 取决于 USRP 型号 |
| USRP 型号 | 【**待确认**】 | 影响带宽与采样率上限 |
| 主时钟频率 | 【**待确认**】 | 需支持 36 MSPS（建议 144 MHz 或 36 MHz 时钟） |

### 7.4 多周期循环发送实现思路

**方案一：UHD 内置循环播放（推荐）**

使用 `uhd_siggen` 或自定义 C++ UHD 程序，将 IQ 文件以循环方式连续发送：

```
// 伪代码
while (running) {
    tx_stream->send(iq_buffer, num_samps_per_frame, md);
    // 自动对齐帧边界
}
```

**方案二：GNU Radio + File Source 循环**

使用 `File Source`（勾选 "Repeat"）→ `UHD: USRP Sink`，快速验证用。

**帧边界保持关键要求：**
- 每次循环发送必须保证整帧（288000 点）连续，不能有中断或截断。
- 建议使用 `timed` 发送模式（`start_of_burst` / `end_of_burst` 标志），确保时序稳定。

### 7.5 带宽需求估算

BOC(15,3) 信号的主要功率集中在：

$$B_\text{主瓣} \approx 2(f_{sc} + f_c) = 2(15 + 3)\,\text{MHz} = 36\,\text{MHz}$$

因此 USRP 射频带宽需求约为 36 MHz，与采样率匹配。需确认所选 USRP 的模拟低通滤波器带宽满足此需求。

## 8. 关键工程难点与风险

### 8.1 【高风险】15 MHz 子载波在 36 MSPS 下的非整数采样问题

**问题描述：**

$$\frac{f_s}{f_{sc}} = \frac{36\,\text{MHz}}{15\,\text{MHz}} = 2.4$$

BOC 子载波的半周期内仅有 **1.2 个采样点**，不是整数。

**深层影响分析：**

| 影响层面 | 具体表现 |
|---|---|
| 理想连续方波 | 每 $1/f_{sc}$ = 66.7 ns 翻转一次，而采样间隔 = 27.8 ns |
| 离散采样结果 | 每个"半周期"不是恒定长度，有时 1 点有时 2 点 |
| `sign(sin(...))` 过零点 | 当 $5\pi n/6$ 恰为 $k\pi$ 时，`sin = 0`，`sign = 0`，出现非 ±1 值 |
| 功率谱形状 | 与理论 BOC(15,3) 谱有偏差，副瓣结构会有差异 |
| 自相关函数 | 相关峰形状会有轻微失真 |

**对 MATLAB 仿真的影响：**
- 仿真结果是真实离散系统的准确模型（而非理想 BOC 的近似），这**正是** USRP 硬件发出的实际波形。
- 建议在仿真报告中明确说明这是"基于 36 MSPS 离散化实现的 BOC(15,3)"，而非理想连续 BOC(15,3)。

**对 USRP 发端的影响：**
- USRP 发出的实际信号与 MATLAB 仿真完全一致（因为两者采样率相同）。
- 接收端设计必须基于此离散化 BOC 子载波设计相关模板，而非理想连续子载波。

**建议处理方案：**

```matlab
% 方案1：直接 sign(sin(...))，接受 sign(0)=0 的"空洞"
sc = sign(sin(2*pi * f_sc * t));

% 方案2：添加微小偏移避免过零点（推荐）
epsilon = 1e-10;
sc = sign(sin(2*pi * f_sc * t + epsilon));

% 方案3：直接生成周期为 2.4 点的近似方波
% 对 (n mod 2.4) 判断符号，但会引入周期性误差
```

**推荐：方案 2**，物理意义最清晰，且与实际 DAC 输出最接近（DAC 建立时间远大于 `epsilon`）。

---

### 8.2 截短 Gold 码对相关性能的影响

**问题描述：**

Gold 码的最优自相关和互相关性质是针对完整周期（1023 码片）定义的。截短为 750 码片后：

- 自相关旁瓣电平会有所升高（理论最优值不再保证）
- 不同载荷之间的互相关性能可能下降
- 截短位置（前750 vs 后273 vs 其他）对性能有影响

**建议在仿真中：**
- 对截短 Gold 码做完整的自相关和互相关分析（阶段 D-6）
- 与完整 1023 码片 Gold 码的相关性能做对比

---

### 8.3 TDMA 时隙边界处的过渡问题

**问题描述：**

在时隙边界处，发送信号从 ±1（当前时隙）突变到 0（静默时隙），或从 0 突变到 ±1（下一时隙开始）。这种突变会：

- 产生宽带瞬态频谱（旁瓣泄漏）
- 在 USRP 发射时造成短暂的射频毛刺

**MATLAB 阶段验证：**
- 在时隙边界处绘制细粒度时域波形，观察过渡情况
- 计算突变带来的额外带外辐射功率

**USRP 实现建议：**
- 可考虑在时隙边界处加短暂的幅度渐变窗（cosine taper），但需与系统规范确认是否允许。

---

### 8.4 USRP 连续发送缓存与时序稳定性

**问题描述：**

- USRP 通过 USB/PCIe/网络传输 IQ 数据，若主机计算机无法及时补充数据，会产生**underflow**（欠载），导致发射信号中断。
- 288000 点 × 4 bytes（int16 I+Q）= 1.152 MB/周期，在 36 MSPS 下每 8 ms 需消耗一个周期。
- 数据传输带宽需求：$1.152\,\text{MB} / 8\,\text{ms} = 144\,\text{MB/s}$，USB3 理论带宽约 400 MB/s，PCIe 更高，原则上满足需求，但实际可用带宽受系统负载影响。

**建议：**
- 采用至少 2 个周期（576000 点）的循环缓冲区，降低 underflow 风险。
- 监控 UHD 驱动的 `async_metadata`，实时检测 underflow 事件。

---

### 8.5 MATLAB 理想仿真与 USRP 实际发射的差距

| 差异来源 | MATLAB 仿真 | USRP 实际发射 |
|---|---|---|
| DAC 量化 | 双精度浮点 | 16-bit 整型（约 96 dB 动态范围） |
| 滤波器 | 无（理想矩形脉冲） | USRP 内部 CIC + 补偿滤波器 |
| 相位噪声 | 无 | 本振相噪影响（待确认 USRP 型号） |
| 时钟抖动 | 无 | 采样时钟抖动（影响 BOC 子载波相位） |
| 直流偏置 | 无 | USRP 模拟前端 DC offset（影响零中频） |

## 附录 A：关键公式速查

| 公式名称 | 表达式 |
|---|---|
| 码片时长 | $T_c = 1 / f_c = 1/3\,\text{MHz} \approx 333.3\,\text{ns}$ |
| 时隙时长 | $T_\text{slot} = N_\text{chips} \times T_c = 750 \times T_c = 250\,\mu\text{s}$ |
| 周期时长 | $T_\text{frame} = 32 \times T_\text{slot} = 8\,\text{ms}$ |
| 数据率 | $R_b = 1 / T_\text{frame} = 125\,\text{bps}$ |
| 每码片采样点数 | $N_c = f_s / f_c = 36/3 = 12$ |
| 每时隙采样点数 | $N_\text{slot} = N_\text{chips} \times N_c = 750 \times 12 = 9000$ |
| 每周期采样点数 | $N_\text{frame} = 32 \times N_\text{slot} = 288000$ |
| BOC 子载波 | $sc(t) = \text{sign}(\sin(2\pi \times 15\,\text{MHz} \times t))$ |
| 离散 BOC 子载波 | $sc[n] = \text{sign}(\sin(5\pi n / 6))$ |
| 单时隙基带（离散） | $s[n] = A \cdot b_m \cdot c[n] \cdot sc[n]$ |
| 处理增益 | $G = 10\log_{10}(750) \approx 28.75\,\text{dB}$ |
| 子载波采样比 | $f_s / f_{sc} = 36/15 = 2.4$（**非整数**） |
