# P4：单周期 TDMA 帧结构生成与验证——技术实施文档
---

## 1. 文档目的与适用范围

本文档是 BOC-TDMA 系统分阶段实施计划中的 **P4 阶段技术实施文档**，承接 P0（USRP 链路验证）、P1（BOC 子载波分析）、P2（Gold 码扩频验证）、P3（单时隙 BOC 基带组合验证），并为 P5（USRP 连续循环发射验证）提供可直接继承的“单周期 TDMA 帧模板”。

P4 阶段的核心定位是：**第一次将 P3 锁定的单时隙基带 `s_slot[n]` 按照 32 时隙的 TDMA 帧结构进行拼接，生成一个完整的、长度严格为 288000 点的单周期帧 `s_frame[n]`，并对其时隙索引、时域结构、频域结构和占空比效应进行定性与定量验证。**

P4 必须独立存在，原因如下：

+ P3 仅验证了**单时隙、单 bit** 的 BOC 基带信号 `s_slot[n]`，其频谱是“持续发射假设下的连续 BOC 双峰谱”；
+ P4 是工程链路上**第一次真正出现 TDMA 帧结构的阶段**，也是**第一次出现“时隙门控（slot gating）”效应的阶段**——部分时隙发射、部分时隙静默；
+ 时隙门控会在 P3 的双峰谱骨架上叠加**占空比带来的平均功率下降**与**时隙边界突变带来的附加频谱泄漏**，这两种效应若不在 P4 单独验证，直接进入 P5 的长时间循环发射时将**无法快速定位**问题是"算法帧结构错"还是"硬件长时稳定性错"；
+ P4 是从“单时隙信号验证”走向“完整周期帧验证”的关键阶段，也是**第一次真正看到“完整单周期 TDMA 帧”的阶段**。

> **阶段边界提醒**：P4 只验证“1 个周期、32 个时隙、288000 点”这一单周期帧，**不涉及连续循环发射、不涉及长时间稳定性、不涉及 underflow 统计**。这些均为 P5 的范畴。

## 2. 本阶段在总工程中的位置

依据【总规划已明确】的分阶段路线：

```plain
P0  USRP 链路基线验证
        │
        ├──> P1  BOC 子载波分析（锁定 sc[n]）
        │
        └──> P2  Gold 码扩频验证（锁定 c_up[n]）
                                │
                                ▼
                        P3  单时隙 BOC 基带组合验证（锁定 s_slot[n]）
                                │
                                ▼
                        P4  单周期 TDMA 帧生成验证   ← 本阶段
                                │
                                ▼
                        P5  USRP 连续循环发射验证
                                │
                                ▼
                        P6  ……
```

依赖关系明确：

+ **P4 必须以 P3 通过验收为前提**，不允许在 P3 仍存在“双峰谱 / M 形自相关”等任何未锁定项的情况下启动 P4；
+ **P4 的输出 `s_frame[n]` 将直接成为 P5 循环发射的帧模板**，P5 不会重新生成帧，而是直接对 `s_frame[n]` 做周期性重复播放；
+ P4 是工程链路中**第一次把单时隙放进帧里**的阶段，也是**时隙索引管理、时隙分配表、静默时隙处理**第一次真正落地的阶段。

> **关键原则**：P4 的新增复杂性不再是“信号体制”，而是“**帧结构拼接与索引管理**”。P3 已经把 BOC 本身的所有疑点解决完毕，P4 只需专注于“把正确的单时隙信号放到正确的位置，再把其他位置按约定处理”。

## 3. 本阶段目标

本阶段只引入一个新的复杂性维度：**32 时隙单周期 TDMA 帧结构的拼接与门控**。具体目标如下：

+ **G1**：建立单周期 TDMA 帧的离散时间模型，明确时隙索引 `i`、帧内样点索引的双层映射关系；
+ **G2**：建立 `slot_config` 时隙分配表，明确 32 个时隙中哪些是“发送时隙”、哪些是“静默时隙”；
+ **G3**：将 P3 锁定的 `s_slot[n]` 正确填入 `slot_config` 指定的发送时隙位置；
+ **G4**：生成长度严格为 288000 点的单周期帧 `s_frame[n]`，覆盖 8 ms 时长；
+ **G5**：验证非发送时隙的**静默（零填充）**处理正确、无越界、无漏填、无覆盖；
+ **G6**：验证完整帧的时域结构（时隙边界、发送/静默分布）与频域结构（BOC 双峰骨架是否保留）；
+ **G7**：量化**占空比（duty cycle）** 对平均功率谱与平均功率电平的影响，给出 `10·log10(duty)` 级别的理论-实测对比；
+ **G8**：形成一个稳定、可复用的“单周期 TDMA 帧模板”，供 P5 阶段直接继承用于循环发射。

## 5. 本阶段不做什么

为避免范围蔓延，本阶段**明确排除**以下内容：

1. **不做连续循环发射与长时间稳定性验证**——属于 P5；
2. **不做 USRP underflow / overflow 长时统计**——属于 P5；
3. **不做多周期运行优化**（缓冲区策略、流控、采样率抖动等）——属于 P5；
4. **不做参数扫描优化**（时隙分配扫描、幅度扫描、增益扫描）；
5. **不做接收端捕获、跟踪、解调实现**——属于 P5/P6；
6. **不重新生成 `s_slot[n]`**，一律继承 P3；
7. **不解决 Gold 码多项式 / 初始状态等 P2 遗留的【待确认】项**，不阻碍 P4 的帧结构验证；
8. **不做多载荷独立 Gold 码的完整建模**——P4 暂按“单一 `s_slot[n]` 模板填入 4 个发送时隙”验证帧骨架；

10. **不最终敲定 USRP 中心频率 / 发射增益**——P4 若涉及硬件验证仅为定性观测；
11. **不引入帧同步序列 / 前导码**——当前系统规划中未确认是否存在，本阶段不擅自添加。

## 6. 单周期 TDMA 帧理论模型

### 6.1 帧层与时隙层的双层索引关系

系统具有两层时间层次结构：

+ **帧层**：一个周期为 `T_frame = 8 ms`，对应 `N_frame = 288000` 个样点；
+ **时隙层**：每帧含 `N_s = 32` 个时隙，每时隙 `T_slot = 250 μs`，对应 `N_slot = 9000` 个样点；
+ 满足：`N_frame = N_s × N_slot = 32 × 9000 = 288000`。

**双层索引的精确定义**（统一采用 0-based 数学索引，与 P1/P2/P3 一致；MATLAB 实现时再平移至 1-based，详见 §6.3）：

+ 时隙编号：`i = 0, 1, ..., 31`（共 32 个时隙）；
+ 时隙内样点编号：`n = 0, 1, ..., 8999`（共 9000 点）；
+ 第 `i` 个时隙在帧中对应的样点索引区间：`[i·N_slot, (i+1)·N_slot - 1]`，即 `[9000·i, 9000·i + 8999]`。

例如：

| 时隙编号 `i` (0-based) | 对应图中 SLOT 编号 (1-based) | 帧内起始索引 | 帧内结束索引 |
| --- | --- | --- | --- |
| 0 | SLOT-1 | 0 | 8999 |
| 1 | SLOT-2 | 9000 | 17999 |
| 2 | SLOT-3 | 18000 | 26999 |
| ... | ... | ... | ... |
| 6 | SLOT-7 | 54000 | 62999 |
| ... | ... | ... | ... |
| 31 | SLOT-32 | 279000 | 287999 |

### 6.2 单周期帧的离散模型

定义 `s_i[n]` 为第 `i` 个时隙的 9000 点基带信号：

$$ s_i[n] = \begin{cases} s_\text{slot}[n], & i \in \mathcal{T}_\text{tx} \\ 0, & i \notin \mathcal{T}_\text{tx} \end{cases}, \quad n = 0, 1, \ldots, 8999 $$

其中：

+ $\mathcal{T}_\text{tx}$：发送时隙编号集合（当前工程假设为 $\{0, 2, 4, 6\}$，对应图中 SLOT-1/3/5/7）；
+ `s_slot[n]`：来自 P3 的单时隙 BOC 基带模板；
+ 非发送时隙按“全零静默”处理【当前工程假设，最终方案待确认】。

则单周期帧的完整离散模型为：

$$ \boxed{\; s_\text{frame}[\,i \cdot N_\text{slot} + n\,] = s_i[n], \quad i = 0, 1, \ldots, 31,\ \ n = 0, 1, \ldots, 8999 \;} $$

等价地：

$$ s_\text{frame}[m] = \sum_{i=0}^{31} s_i[m - i \cdot N_\text{slot}] \cdot \mathbb{1}\!\left[i \cdot N_\text{slot} \le m < (i+1) \cdot N_\text{slot}\right], \quad m = 0, 1, \ldots, 287999 $$

变量说明：

+ `m`：帧内绝对样点索引（0-based），`m = 0, 1, ..., 287999`；
+ `i`：时隙编号（0-based），`i = floor(m / N_slot)`；
+ `n`：时隙内相对样点索引（0-based），`n = m - i · N_slot = mod(m, N_slot)`。

**工程约束**：

1. **`s_frame[n]` 严格长度为 288000**，不允许任何偏差；
2. 若第 `i` 时隙是发送时隙，`s_frame` 在 `[i·9000, i·9000+8999]` 区间应逐点等于 `s_slot[n]`；
3. 若第 `i` 时隙是静默时隙，`s_frame` 在对应区间严格为 0（当前工程假设下）；
4. 时隙之间**不加任何保护间隔、不加任何过渡窗**（若未来需要加，属于新版 `slot_config` 的修改，不改变 P4 验证框架）。

### 6.3 时隙配置表的工程意义与索引管理

P4 的最大工程风险来自**索引管理错误**，不是信号体制错误。必须把以下三件事分别处理清楚：

**（1）图示 SLOT 编号（1-based，业务语义）**

+ 系统结构图中使用 SLOT-1 ～ SLOT-32；
+ 这是**业务层语义**，所有沟通、评审、记录表均应使用这一编号；
+ 例如：“载荷1 的固定时隙是 SLOT-1”。

**（2）时隙索引 `i`（0-based，数学语义）**

+ 数学公式中使用 `i = 0, 1, ..., 31`；
+ 便于公式推导与帧内样点索引换算（`start = i · N_slot`）。

**（3）MATLAB 代码索引（1-based，实现语义）**

+ MATLAB 向量索引从 1 开始；
+ 第 `i` 个时隙（0-based）在 MATLAB 中对应 `s_frame(i*N_slot+1 : (i+1)*N_slot)`；
+ 例如 SLOT-1（业务）= `i=0`（数学）= MATLAB 索引 `1:9000`；
+ SLOT-7（业务）= `i=6`（数学）= MATLAB 索引 `54001:63000`。

**换算表（核心，必须严格遵守）**：

| 业务 SLOT 编号 | 数学 `i` | MATLAB 起始索引 | MATLAB 结束索引 |
| --- | --- | --- | --- |
| SLOT-1 | 0 | 1 | 9000 |
| SLOT-3 | 2 | 18001 | 27000 |
| SLOT-5 | 4 | 36001 | 45000 |
| SLOT-7 | 6 | 54001 | 63000 |
| SLOT-32 | 31 | 279001 | 288000 |

> **工程戒律**：P4 阶段至少 50% 的 bug 来自这三种索引的混用。**任何一条代码在处理时隙索引时，必须在注释中明确使用的是哪一种编号体系**。在 V3（索引与边界检查）中必须逐条显式验证。

## 7. 时隙占空比与频谱变化分析

**本节是 P4 的核心分析之一。** P3 的频谱是“持续发射假设”下的 BOC 双峰谱，而 P4 的频谱将叠加时隙门控效应。若不先在 MATLAB 中定量理解这两种效应，P5 硬件观测到频谱异常时将无法区分“帧结构本身的特性”与“硬件或软件缺陷”。

### 7.1 从 P3 到 P4 的信号变化

+ **P3 阶段 `s_slot[n]`**：9000 点，全部样点为有效 BOC 基带信号（±1），对应“持续发射”假设，频谱为典型 BOC(15,3) 双峰；
+ **P4 阶段 `s_frame[n]`**：288000 点，其中仅有 `|T_tx| · N_slot = 4 · 9000 = 36000` 个样点为有效基带信号，其余 252000 个样点为零；
+ 时域上，`s_frame[n]` 可理解为 `s_slot_repeat[n]`（假想的连续重复）被**时隙门控窗** `g[n]` 逐点相乘：

$$ s_\text{frame}[n] = s_\text{slot\_repeat}[n] \cdot g[n] $$

其中 `g[n]` 为取值 ∈ {0, 1} 的时隙门控序列：发送时隙对应段为 1，静默时隙对应段为 0。

### 7.2 占空比（Duty Cycle）与平均功率

**占空比定义**：

$$ D = \frac{N_\text{tx}}{N_s} = \frac{|\mathcal{T}_\text{tx}|}{32} $$

**当前工程假设下**：$N_\text{tx} = 4$，$D = 4/32 = 12.5\%$。

**平均功率理论值**（相对 P3 持续发射情况）：

$$ \bar{P}_\text{frame} = D \cdot \bar{P}_\text{slot} $$

转换为 dB：

$$ \boxed{\; \Delta P_\text{dB} = 10 \log_{10}(D) \;} $$

在当前工程假设下：

$$ \Delta P_\text{dB} = 10 \log_{10}(0.125) \approx -9.03\,\text{dB} $$

即 `s_frame` 的平均功率应比 `s_slot`（持续发射）低约 **9 dB**。

> **工程要点**：这 9 dB 的下降**不是故障**，而是帧结构的必然结果。P5 硬件观测到频谱仪读数比 P3 阶段低 9 dB 左右，属于预期行为。

若未来 `slot_config` 更新为其他配置，占空比与 dB 下降按下表计算：

| 发送时隙数 `N_tx` | 占空比 `D` | 理论功率差 (dB) |
| --- | --- | --- |
| 1 | 3.125% | -15.05 |
| 2 | 6.25% | -12.04 |
| 4 | 12.5% | -9.03 |
| 8 | 25% | -6.02 |
| 16 | 50% | -3.01 |
| 32 | 100% | 0.00 |

### 7.3 平均功率谱：骨架保留，幅度下降

时隙门控的效应在频域等价于**与门控序列 `g[n]` 的频谱做卷积**。关键结论：

1. **BOC 双峰骨架保留**：只要发送时隙内填的是 `s_slot[n]`，双峰中心仍位于 ±15 MHz 附近；
2. **平均功率谱密度整体下降约 `10·log10(D)` dB**；
3. **谱线精细结构叠加**：门控序列 `g[n]` 是周期为 8 ms 的周期函数，其频谱是间隔 `1/T_frame = 125 Hz` 的谱线。与 `s_slot` 的连续谱卷积后，理论上 `s_frame` 的频谱是一个**密集谱线簇**，包络仍近似 BOC 双峰；
4. **在 P4 的 FFT 分辨率下**（Nfft = 2^19 = 524288，频率分辨率约 68.7 Hz），是否能清晰分辨 125 Hz 谱线视 FFT 长度而定；当 FFT 分辨率粗于 125 Hz 时，观测到的即是“连续包络”；

### 7.4 时隙边界突变与附加频谱泄漏

**时隙边界是 P4 必须单独观察的现象**：

+ 在 `i` 是发送时隙、`i+1` 是静默时隙的交界处，信号从 ±1（BOC 电平）瞬间跳变到 0；
+ 在 `i` 是静默时隙、`i+1` 是发送时隙的交界处，信号从 0 瞬间跳变到 ±1；
+ 这种矩形门控引入的突变等价于**用矩形窗去截取**连续 BOC 信号，会在双峰主瓣外侧引入 **sinc 型旁瓣**；
+ 旁瓣的衰减包络大致为 `1/f` 级别（矩形窗在频域的衰减速率）；
+ **这是帧结构的固有特性，不是缺陷**，但必须在 P4 阶段记录其基线幅度，供 P5 阶段对比。

> **工程识别要点**：若 P5 阶段在频谱仪上发现双峰主瓣外侧有 sinc 型旁瓣抬起约 `10·log10(D)` dB 的底噪，**不要误判为硬件泄漏**，它就是 P4 阶段已记录的时隙门控效应。

### 7.5 若本阶段不先验证，会给 P5 带来什么定位困难

若跳过 P4 直接进入 P5 的连续循环发射：

1. 一旦频谱仪上看到平均功率下降 ~9 dB，无法判断是“占空比效应（预期）”还是“USRP 输出幅度异常（故障）”；
2. 一旦看到双峰外侧旁瓣抬起，无法判断是“时隙门控矩形窗效应（预期）”还是“硬件泄漏（故障）”；
3. 一旦看到频谱细结构，无法判断是“8 ms 周期性引入的 125 Hz 谱线（预期）”还是“采样时钟抖动（故障）”；
4. 所有帧级别的现象都会和 P5 特有的“长时稳定性”现象混在一起，**几乎无法快速定位**。

**结论：P4 是 P5 之前不可绕过的最后一道帧结构级验证闸门。**

## 9. USRP + 频谱仪验证建议

**前提：MATLAB 侧 V1–V7 必须全部通过，`s_frame[n]` 已导出为 USRP 可读的 IQ 文件。** USRP 硬件验证仅作为定性确认手段，不作为算法正确性的判据。

### 9.1 发射配置

+ 发射内容：`I[n] = s_frame[n]`，`Q[n] = 0`；
+ 发射方式：**短时重复播放**单周期帧（例如连续播放 10～100 个周期用于频谱仪观察）；
  - **本阶段的重复播放不等同于 P5 的长时间连续循环发射**；
  - 目的仅是让频谱仪能平均/稳定出谱形，不涉及长时 underflow 统计；
+ 采样率：`fs = 36 MSPS`（继承 P0）；
+ 中心频率 / 发射增益：继承 P0 基线配置【待确认最终值】；
+ IQ 文件格式：继承 P0 锁定格式。

### 9.2 频谱仪观测参数（参考总规划）

+ Span：中心频率 ±40 MHz（覆盖 BOC 双峰）；
+ RBW：300 kHz（参考总规划）；
+ VBW：≤ RBW；
+ 平均模式：**RMS Average 或 Video Average**（用于观察稳态谱形，不用 Max Hold）；
+ 参考电平：根据 P0 记录调整，避免过载。

### 9.3 关键观测现象（P4 专属）

| 观测项 | 预期现象 | P3 → P4 变化 |
| --- | --- | --- |
| BOC 双峰结构 | 仍在 ±15 MHz 出现 | 保留 |
| 中心频率凹陷 | 仍为 null | 保留 |
| 双峰峰值电平 | 比 P3 低约 9 dB | **核心变化** |
| 双峰主瓣外侧旁瓣 | sinc 型旁瓣抬起 | **核心变化** |
| 频谱细结构（若 RBW 足够窄） | 125 Hz 间隔谱线簇 | **P4 新现象** |

**P4 硬件验证的重点不是“长时间稳定”，而是“帧结构带来的频谱变化是否符合 MATLAB 仿真预期”。**

### 9.4 若频谱异常，故障树

| 现象 | 可能原因 | 定位方向 |
| --- | --- | --- |
| 双峰完全消失 | `s_frame` 未正确填入 `s_slot` | **算法问题**，回查 V2/V3 |
| 双峰功率下降不是 ~9 dB | 占空比计算错 / 发送时隙数错 | **算法问题**，回查 `slot_config` |
| 双峰中心偏离 ±15 MHz | 继承的 `s_slot` 错误 | 回到 P3 |
| 双峰不对称（> 3 dB） | DC offset、IQ 不平衡 | **硬件问题**，回到 P0 |
| 未看到占空比效应（功率不降） | 非发送时隙未做零填 | **算法问题**，回查 V2 |
| 频谱形状与 MATLAB 完全不符 | IQ 文件格式错 / 采样率错 | **硬件问题**，回到 P0 |
| 间歇性谱线抖动 | USRP 缓冲 / 时钟问题 | 属于 P5 范畴，P4 不深入 |

> **工程识别原则**：P4 阶段若频谱仪上的现象与 MATLAB 仿真**定性一致但有量化差异**（几个 dB 以内），允许进入 P5；若**定性不一致**（双峰消失、中心变主峰等），**必须回查 `slot_config` 与 V3 索引检查**，不得带伤进入 P5。

## 11. 常见误判与风险

1. **把“平均功率下降 ~9 dB”误判为 USRP 输出幅度异常**——实际上这是 4/32 占空比的必然结果，P4 已量化。
2. **把“双峰外侧 sinc 旁瓣”误判为 BOC 频谱异常或硬件泄漏**——它是矩形门控的固有特性，不是缺陷。
3. **MATLAB 1-based 与数学 0-based 索引混用**——将导致整个时隙偏移一位，最典型的表现是“本应填入 SLOT-1 的内容实际出现在 SLOT-2”，而时域包络看上去“差不多”，极其隐蔽。必须通过 V3 索引检查表逐条手工核对。
4. **在 `slot_config` 未最终固定就开始生成帧**——后续文档、代码、记录表将不一致，且 P5 会直接继承错误版本。务必先存盘 `slot_config`，再生成 `s_frame`。
5. **非发送时隙处理方式在代码中前后不一致**——例如 V2 用 `zeros(1, N_frame)` 初始化，但后续某段又在静默时隙写入非零值。必须在 V3 中用 `sum(s_frame ~= 0)` 严格核对非零样点数。
6. **重新生成 `s_slot[n]` 而不是 load P3 模板**——这将引入 P3 未经验证的新版本，违反“一次锁定，继承到底”的原则。P4 代码必须只 `load`，不重新生成。
7. **误在 P4 阶段引入“每个发送时隙使用不同 Gold 码”**——当前工程假设明确是“同一 `s_slot` 模板复用”，多 Gold 码属于后续阶段内容。若现在就引入，P4 的频谱/功率结论将与 `slot_config` 记录不匹配。
8. **将帧频谱中的 125 Hz 谱线簇误判为异常**——它来自 8 ms 周期性，是预期行为，在 FFT 分辨率足够时自然出现。
9. **在 P4 阶段做长时间发射测试**——属于越界行为，会把 P5 的 underflow / overflow 问题混入 P4 验收，导致两个阶段都无法清晰收敛。**严格守住 P4 只做短时重复观测的边界。**
10. **若不在 P4 单独验证就进入 P5**——几乎所有上述现象都会在 P5 阶段一次性涌现，且与 USRP 长时稳定性问题交织，定位成本极高。

## 12. 完成判据 / 进入下一阶段条件

P4 验收必须**同时满足**以下条件方可进入 P5：

1. **`slot_config` 已明确记录**，包含版本标注、tx_slots、静默策略；
2. **`s_frame[n]` 已稳定生成**，长度严格为 288000 点，采样率 36 MSPS；
3. **索引与边界检查全部通过**：V3 索引检查表中每一行均通过（手工核对 + `assert`）；
4. **时域图上可清晰识别发送时隙与静默时隙**：F1、F2 通过；
5. **时隙边界、发送时隙内部、多发送时隙一致性已目视确认**：F3、F4、F5 通过；
6. **FFT 中完整帧仍保留可解释的 BOC 双峰特征**：
   - 主峰中心位于 ±15 MHz（±1 bin）；
   - 双峰对称性 ≤ 1 dB；
   - 中心频率处为凹陷；
   - 与 P3 相比包络形状一致；
7. **占空比对平均功率的影响已量化**：
   - 实测 `D` 与理论 `12.5%` 一致；
   - 实测 `10·log10(P_frame/P_slot)` 与 `-9.03 dB` 偏差 ≤ 0.3 dB；
8. **时隙门控引入的附加频谱效应（sinc 旁瓣）已记录并作为 P5 基线**；
9. **（若执行 USRP 验证）频谱仪上已确认双峰下降约 9 dB，且形态与 MATLAB 仿真趋势一致**；
10. **单周期帧模板已导出为可复用数据文件（如 `s_frame_template.mat`）**，供 P5 直接 load。

只有上述全部完成，才允许进入 P5 的 USRP 连续循环发射验证。

## 15. 应锁定的参数与遗留待确认项

### 15.1 P4 结束后必须锁定的内容

以下内容在 P4 验收通过后**必须锁定**，P5 及之后阶段不得再行修改：

1. **单周期帧长度**：严格 288000 点，采样率 36 MSPS；
2. **帧内双层索引规则**：`frame_idx = i · N_slot + n`，其中 `i ∈ [0,31]`（数学）、`n ∈ [0,8999]`（数学），MATLAB 实现需平移至 1-based；
3. **`s_frame[n]` 生成公式**：`s_frame[i·N_slot + n] = s_slot[n]` 若 `i ∈ T_tx`，否则为 0；
4. **当前工程版本 `slot_config` 快照**：存盘为 `p4_slot_config_v1.mat`，供 P5 唯一继承；
5. **非发送时隙处理方式**：全零静默【当前工程版本，最终方案仍属待确认项但不阻碍 P5 推进】；
6. **单周期帧 BOC 双峰基线**：中心 ±15 MHz，形态与 P3 一致；
7. **占空比与平均功率基线**：`D = 12.5%`，`10·log10(D) ≈ -9.03 dB`；
8. **时隙门控附加频谱效应基线**：sinc 型旁瓣形态与相对幅度已记录；
9. **单周期帧模板文件**：作为 P5 唯一可信输入，严禁在 P5 阶段重新生成。

### 15.2 仍然【待确认】但不阻碍 P4 → P5 推进的内容

1. **32 时隙三类（固定 / 保护 / 动态）的完整编号映射**——当前工程假设下仅用到 SLOT-1/3/5/7 作为发送时隙，其余均静默，足以完成 P4 帧结构验收；
2. **非发送时隙是否必须严格全零**——当前工程采用全零，若未来确认为其他处理（如训练序列、前导码等），需回到 `slot_config` 修改后重跑 P4 流程；
3. **4 个载荷是否共用同一发射机**——若共用则 P4 当前模型直接成立；若独立发射则属于多机同步问题，超出 P4 范畴；
4. **每个发送时隙具体对应哪个载荷 / 是否使用独立 Gold 码**——P4 当前假设同一 `s_slot` 模板，属于结构性验证；多 Gold 码版本可以在 `slot_config` 升级后重跑，**P4 的验证框架本身不变**；
5. **是否存在额外帧同步序列 / 前导码**——当前规划未明确，P4 不擅自添加；若后续引入，属于新版 `slot_config` 的扩展；
6. **最终 USRP 中心频率 / 发射增益**——P4 若做硬件验证采用 P0 基线，最终值待系统联调敲定；
7. **bit=0 / bit=1 → ±1 的具体映射方向**——继承自 P3 的【待确认】项，P4 阶段统一 `b_m = +1`，不依赖映射方向；
8. **接收端如何利用帧结构进行时隙同步**——属于 P5/P6 范畴，P4 仅提供帧模板作为接收端设计输入。

**阶段定位复述**：P4 是工程链路中**第一次真正看到完整单周期 TDMA 帧的阶段**，是**第一次把单时隙信号放进帧里的阶段**，也是 P5 之前**必须独立吃透的帧结构级验证闸门**。P4 的新增复杂性是“帧结构拼接与索引管理”，而非信号体制本身。P4 通过后，单周期帧模板 `s_frame[n]` 将作为 P5 唯一可信的循环发射输入。
