# USRP B200 / B210 中文参数整理

原始手册：`../B200_B210/b200-b210_spec_sheet.pdf`  
文档类型：2 页规格页（Spec Sheet）

## 1. 平台定位

USRP B200 和 B210 是面向通用 SDR 实验的平台，覆盖宽频段 RF、USB 3.0 高速主机连接，并提供 `GNU Radio`、`C++`、`Python` API 支持。规格页强调它们适合 FM、电视广播、蜂窝通信、Wi-Fi 等各类信号实验。

## 2. 两个型号的主要差异

### USRP B200

- 通道能力：`1 TX + 1 RX`
- 工作方式：支持半双工或全双工
- FPGA：`Xilinx Spartan-6 XC6SLX75`
- 瞬时带宽：最高 `56 MHz`
- 供电方式：`USB` 总线供电

### USRP B210

- 通道能力：`2 TX + 2 RX`
- 工作方式：支持半双工或全双工
- 支持 `2x2 MIMO` 相干能力
- FPGA：`Xilinx Spartan-6 XC6SLX150`
- 瞬时带宽：
  - `1x1` 模式最高 `56 MHz`
  - `2x2` 模式最高 `30.72 MHz`
- 附带直流电源适配器
- 支持 `GPIO`

## 3. 共有关键参数

- RF 频率覆盖：`70 MHz ~ 6 GHz`
- 主机接口：`USB 3.0 SuperSpeed`
- 连接器：`Standard-B USB 3.0`
- ADC 采样率最高：`61.44 MS/s`
- DAC 采样率最高：`61.44 MS/s`
- ADC / DAC 分辨率：`12-bit`
- Host Sample Rate（16-bit）：`61.44 MS/s`
- ADC 宽带 `SFDR`：`78 dBc`
- 频率精度：`±2.0 ppm`
- 带 GPS 未锁定 `TCXO`：`±75 ppb`
- GPS 锁定 `TCXO`：`< 1 ppb`
- 典型输出功率：`> 10 dBm`
- 接收噪声系数：`< 8 dB`
- 典型 `IIP3`（在典型噪声系数条件下）：`-20 dBm`
- `SSB/LO Suppression`：`-35 / -50 dBc`

## 4. 规格页中的物理参数

- 直流输入：`6 V`
- 尺寸：`9.7 x 15.5 x 1.5 cm`
- 重量：`350 g`
- 机身有接地安装孔

## 5. 软件与生态

- 驱动：`UHD (USRP Hardware Driver)`
- API：支持 `GNU Radio`、`C++`、`Python`
- 平台兼容性：`Windows`、`Linux`、`macOS`
- 规格页强调：应用可较平滑迁移到更高性能或嵌入式 USRP 平台

## 6. 工程解读

### 参数解读：`ADC 采样率最高：61.44 MS/s` 的实际含义

- 直观含义：`61.44 MS/s` 表示 ADC 每秒最多进行约 `6144 万次采样`，是模拟信号数字化的最高采样节拍上限。
- 频域理解（奈奎斯特）：若按实信号采样理解，理论上可无混叠表示到约 `30.72 MHz`（`Fs/2`）的基带带宽。
- 在 SDR 的 I/Q 语境下：B200/B210 常输出复基带（I/Q）样本流，配置到高采样率时，链路能否长期稳定运行还取决于 USB 吞吐、主机 CPU、内存和磁盘写入能力。
- 不是“任何场景都能跑满”：规格页给的是硬件上限，工程上可持续速率通常低于峰值，尤其在长时录制或多通道场景。
- 对 GNSS 实验的意义：
  - L1 C/A 常用采样率通常远低于 `61.44 MS/s`，多数情况下瓶颈不在 ADC 上限。
  - 提高采样率有助于频谱观察和滤波余量，但会显著增加数据率与后处理成本。
  - 采样率应结合目标信号带宽、算法需求和主机链路预算综合折中，而不是单纯追求最高值。

### 对 B200 的理解

- 适合单收单发实验、成本与供电约束较强的场景
- `USB` 直接供电，便于快速搭建，但高负载外设扩展能力相对有限

### 对 B210 的理解

- 更适合做同步双通道接收、双发射、阵列和 `MIMO` 类实验
- `2x2` 模式带宽会下降到 `30.72 MHz`，做多通道实验时要提前规划采样率与主机链路预算
- 带 `GPIO` 和独立电源，更适合与外部时序控制、前端模块或实验平台联动

### 使用上的关键点

- 虽然 RF 覆盖达到 `70 MHz ~ 6 GHz`，但不同频段的输出功率、线性度和噪声表现并不是完全一致的，做链路预算时不能只看覆盖范围
- `USB 3.0` 是发挥大带宽能力的前提；若退回 `USB 2.0`，实际可用吞吐会明显受限
- 规格页中很多指标是典型值，系统设计时建议为主机吞吐、缓存、时钟稳定度和散热留出余量
