# USRP X300 / X310 中文参数整理与迁移分析

原始手册：`../X300_X310/X300_X310_Spec_Sheet.pdf`  
文档类型：2 页规格页（Spec Sheet）  
分析范围：规格摘要 + 结合当前 BOC-TDMA 项目的 B210 -> X310 迁移判断

## 1. 平台定位

USRP X300 和 X310 是 Ettus 面向高性能、可扩展 SDR 原型验证的平台。和 B200/B210 这种“一体化 RF 实验板”不同，X300/X310 更像是：

- 高性能主机接口平台
- 大型 FPGA 处理平台
- 外接 RF daughterboard 的机箱平台
- 支持外部参考时钟和相干运行的实验底座

规格页给出的典型应用包括：

- 高级无线原型验证
- 大规模 MIMO 测试平台
- 被动雷达
- 信号情报类系统

对本项目最重要的意义是：它不是为了“先把一个 36 MSPS 单通道波形发出来”而生，而是为了在更高带宽、更多通道、更强同步和更大 FPGA 余量的条件下做系统级扩展。

## 2. X300 与 X310 的主要差异

### X300

- FPGA：`Xilinx Kintex-7 XC7K325T`

### X310

- FPGA：`Xilinx Kintex-7 XC7K410T`

工程上可以把两者理解为同一平台族，X310 主要是在 FPGA 资源上比 X300 更大，更适合做复杂自定义逻辑、更多缓存、更重的板上处理。

## 3. 规格页中的关键能力

## 3.1 射频与带宽框架

- RF daughterboard 插槽：`2` 个
- 每通道基带带宽：最高 `120 MHz`
- 频率覆盖：取决于所选 daughterboard，规格页写为“覆盖 `DC ~ 6 GHz` 的板卡可选”

这里有一个非常关键的工程点：

> X310 本体并不等于“直接覆盖 70 MHz ~ 6 GHz 的完整射频机”。  
> 实际可用频段、发射功率、噪声、线性度，取决于你装的 RF daughterboard。

规格页里的部分 RF 指标明确标注为 **with SBX-120**，也就是说这些数值是“装了 SBX-120 板卡时”的表现，不是 X310 机箱本体的统一指标。

## 3.2 数据转换与采样能力

- ADC 采样率最高：`200 MS/s`
- ADC 分辨率：`14-bit`
- DAC 采样率最高：`800 MS/s`
- DAC 分辨率：`16-bit`
- Host Sample Rate（16-bit）：`200 MS/s`

和 B210 相比，X310 在采样率、位宽和主机侧吞吐能力上都有明显提升。

## 3.3 主机接口

- `PCIe x4`
- 双 `SFP(+)` 端口，支持 `1/10 Gigabit Ethernet`

这点对你的项目很重要，因为当前系统规划已经把持续发射链路预算写成：

- `36 MSPS × 4 bytes = 144 MB/s`

因此对 X310 来说：

- `1 GigE` 理论上只有 `125 MB/s`，实际可用更低，**不应作为 36 MSPS 持续 I/Q 流的首选接口**
- 更合适的是：
  - `10 GigE`
  - 或 `PCIe x4`

## 3.4 时钟与同步

- 可配置采样时钟
- 可选 GPS 驯服 OCXO
- 支持 `10 MHz / 1 PPS` 外部参考与相干运行
- 内部参考精度：`2.5 ppm`
- 带 GPSDO 但未锁星时：典型 `20 ppb`

这说明 X310 在：

- 多机同步
- 长时间稳定性
- 与外部标准时钟联动

方面明显强于普通 USB SDR。

## 3.5 功耗与体积

- 直流输入：`12 V`
- 典型功耗：`45 W`（装 `2x SBX-120`）
- 尺寸：半宽 `1U`
- 重量：约 `1.7 kg`（装 `2x SBX-120`）

它显然不是 B210 那种“桌面随插随测”的轻量级实验板，而是更接近实验室/机架平台。

## 3.6 规格页中的典型 RF 指标（with SBX-120）

- `SSB/LO Suppression`：`-35 / -50 dBc`
- 输出功率：`> 10 dBm`
- 典型 `IIP3`：`0 dBm`
- 典型噪声系数：`8 dB`

因为这些是与具体板卡相关的指标，所以真正迁移到 X310 时，不能只写“目标平台 = X310”，还必须同时锁定：

- daughterboard 型号
- 工作频段
- 增益配置
- 接口方式（10GigE / PCIe）
- 时钟方案（内部 / GPSDO / 外部 10 MHz）

## 4. 与 B210 的工程差异

针对你当前项目，B210 和 X310 最重要的差异不是“谁更高级”，而是“它们解决的问题层级不同”。

| 维度 | B210 | X310 |
|---|---|---|
| 平台形态 | 一体化 RF SDR | 机箱 + daughterboard 平台 |
| 主机接口 | USB 3.0 | 1/10 GigE、PCIe x4 |
| ADC | 12-bit / 61.44 MS/s | 14-bit / 200 MS/s |
| DAC | 12-bit / 61.44 MS/s | 16-bit / 800 MS/s |
| Host Sample Rate | 61.44 MS/s | 200 MS/s |
| 时钟能力 | 常规 TCXO/GPS 级 | 外部 10 MHz / 1 PPS、GPSDO、相干运行 |
| RF 前端 | 集成 AD9361 | 取决于 daughterboard |
| 使用门槛 | 低 | 明显更高 |
| 适合当前 P0-P3 | 是 | 能做，但成本与复杂度偏高 |
| 适合后续扩展 | 一般 | 很强 |

再结合你已经整理的 AD9361 文档，可以把 B210 理解为：

- 集成度高
- 开工快
- 适合先把“采样率、波形、频谱、USRP 配置链路”打通

而 X310 更适合：

- 高带宽长时间流传输
- 更稳定的接口与时钟环境
- 更复杂的同步和扩展实验

## 5. 结合当前项目的判断

## 5.1 结论

**可以先用 B210 测，再过渡到 X310。**

而且对你当前项目阶段来说，这其实是一个比较合理的推进路径。

## 5.2 为什么当前阶段先上 B210 是合理的

你当前项目还处在：

- `P0-A` 刚通过
- `P0-B` 还没开始
- 后续马上要进入 `P1/P2/P3` 的波形与频谱基线验证

这一阶段最核心的问题不是“极限吞吐量”或“多通道同步”，而是：

- USRP 是否能稳定接受 `36 MSPS`
- 主时钟与采样率关系是否正确
- IQ 文件格式是否正确
- CW 单峰是否正确
- 后续 BOC 双峰、扩频谱、TDMA 帧结构是否与 MATLAB 基线一致

这些问题，**B210 单通道完全够用来做第一轮工程验证**。

尤其是系统规划里你的链路是单路发射为主，当前没有要求：

- 多路相干发射
- 大规模 MIMO
- 板上 FPGA 实时处理
- 外部统一授时

所以在这一阶段直接上 X310，不一定能带来同比例收益，反而会更早引入：

- 板卡选型
- 接口选型
- 机箱供电
- 时钟方案
- UHD 网络/PCIe 配置

这些额外复杂度。

## 5.3 为什么后续仍然值得迁移到 X310

当你走到更靠后的阶段，X310 的价值会明显出来：

- `P5/P6` 长时间持续发射时，对接口稳定性和吞吐余量更有帮助
- 如果后续接收端或系统联调需要统一参考时钟，X310 更合适
- 如果未来想做多通道、相干、阵列、同步实验，X310 的平台能力明显更强
- 如果后续要把更多逻辑推到 FPGA 或需要更强板上缓存，X310 余量更大

因此，**B210 更适合作为“先把链路和信号做对”的平台，X310 更适合作为“把系统做强、做稳、做大”的平台。**

## 6. 但这不是无缝切换

“先 B210、后 X310”虽然可行，但不能把它理解成“只换硬件名称，其他完全不变”。

迁移时至少要重新确认以下内容：

### 6.1 必须重新锁定的硬件项

- USRP 型号
- daughterboard 型号
- 主机接口类型（10GigE / PCIe）
- 主时钟频率
- 实际可生效的采样率
- TX 通道编号 / subdev 配置
- 增益范围与默认值
- 中心频率可配置范围

### 6.2 必须重新验证的测试项

- `P0-B` 的 CW 单峰验证
- `36 MSPS` 的实际可行性
- underflow 情况
- DC offset / LO leakage
- 镜像抑制
- 增益变化与功率变化关系
- 长时间连续发射稳定性

也就是说，迁移到 X310 后，**至少要把 P0-B 再完整跑一遍**，不能直接从 B210 的结果跳到 P3/P5。

### 6.3 代码层的迁移边界

若你希望后续切换平台尽量顺滑，当前就应该避免把代码写死成 B210 特有假设，例如：

- 写死设备名
- 写死主时钟值
- 写死 USB3 相关假设
- 写死通道 / 子设备字符串
- 写死增益范围

更稳妥的做法是把以下内容留成硬件配置层，而不是算法常量：

- device args
- subdev spec
- master clock rate
- tx gain
- center frequency
- clock source / time source

## 7. 对你当前项目的推荐推进顺序

基于当前文档状态，我建议这样走：

1. 用 **B210** 完成 `P0-B`
   - 先确认 `36 MSPS`
   - 再做 CW 单峰
   - 记录主时钟、接口、UHD、固件版本

2. 继续用 **B210** 做 `P1-P3`
   - 把离散子载波
   - 扩频谱
   - BOC 双峰
   这三类“信号本体问题”先全部打通

3. 到 `P4/P5` 前重新评估是否切换 **X310**
   - 如果只是单通道验证，B210 也可能继续够用
   - 如果要做更稳的长时持续发射、统一时钟或后续 RX 联调，X310 更值得切过去

4. 一旦切换到 **X310**
   - 先锁定 daughterboard
   - 接着重跑 `P0-B`
   - 再恢复后续阶段验证

## 8. 最终建议

针对你现在的项目状态，建议可以直接写成一句工程决策：

> **阶段性选型建议：当前先以 B210 作为 P0-B 到 P3/P4 的早期验证平台；待硬件链路、波形基线和后续同步/吞吐需求明确后，再迁移到 X310，并在迁移时重跑 P0-B 硬件基线验证。**

这个策略的优点是：

- 开工快
- 成本低
- 风险可控
- 不会过早把项目拖入 X310 的接口与板卡复杂度

同时也保留了未来升级空间：

- 更高吞吐
- 更强同步
- 更大 FPGA 余量
- 更适合系统级扩展
